# HLD: SILK Rate Control Loop Fix — V2

## Changes from V1
- **Fix 1 now explicitly includes nBitsExceeded update** at packet completion (was mentioned in risks, now promoted to required)
- **Added**: BITRESERVOIR_DECAY_TIME_MS constant definition
- **Clarified**: Exact insertion points for all changes
- **Noted**: For the first packet in isolation, target rate matches anyway — the fix primarily helps multi-packet (>50 frame) sessions

---

## Problem

All 20+ @16kbps encode tests fail because the SILK encoder's rate control loop converges differently. The `silk_gains_quant` function is correct; the issue is the loop iteration count differs because the nBits target/budget is computed differently.

## Root Cause

Three issues (in priority order):

**A. TargetRate_bps is raw bitrate.** Rust passes `enc_control.bit_rate` directly. C computes a 6-step per-frame target with LBRR subtraction, frame count division, bps conversion, bit reservoir decay, and intra-packet balance correction. For the first packet the adjustments are zero, but `nBitsExceeded` accumulates over packets, causing drift.

**B. Missing nBits re-read after damage control.** In the rate control loop's last-iteration fallback, C re-reads `nBits = ec_tell()` after zeroing pulses and re-encoding. Rust keeps the stale pre-damage-control value.

**C. gain_mult_lower/upper type mismatch.** C uses `opus_int32`; Rust uses `i16`. Values are typically 64–1024 (fits i16), but C semantics are wider.

## Design

### Fix 1: TargetRate_bps computation + nBitsExceeded update

**Location**: `src/silk/encoder.rs`, function `silk_encode`, replacing line ~7905.

```rust
// --- Compute per-frame target rate (matching C enc_API.c:422-454) ---

// Total bits for packet
let mut n_bits_target = silk_div32_16(
    silk_mul(enc_control.bit_rate, enc_control.payload_size_ms), 1000);

// Subtract LBRR bits (if not prefill)
if prefill_flag == 0 {
    n_bits_target -= enc.n_bits_used_lbrr;
}

// Divide by frames per packet
n_bits_target = silk_div32_16(n_bits_target, n_frames_per_packet);

// Convert to bps
let mut target_rate = if enc_control.payload_size_ms == 10 {
    silk_smulbb(n_bits_target, 100)
} else {
    silk_smulbb(n_bits_target, 50)
};

// Subtract bit reservoir decay
target_rate -= silk_div32_16(
    silk_mul(enc.n_bits_exceeded, 1000), BITRESERVOIR_DECAY_TIME_MS);

// Intra-packet balance for frame 2+
if prefill_flag == 0 && enc.state_fxx[0].s_cmn.n_frames_encoded > 0 {
    let bits_balance = range_enc.tell() as i32
        - enc.n_bits_used_lbrr
        - n_bits_target * enc.state_fxx[0].s_cmn.n_frames_encoded;
    target_rate -= silk_div32_16(
        silk_mul(bits_balance, 1000), BITRESERVOIR_DECAY_TIME_MS);
}

// Clamp to [5000, bitRate]
target_rate = target_rate.max(5000).min(enc_control.bit_rate);

silk_control_snr(&mut enc.state_fxx[0].s_cmn, target_rate);
```

**nBitsExceeded update at packet completion** — insert at ~line 7988, after `*n_bytes_out` is finalized and before the frame counter reset:

```rust
// Update bit reservoir (matching C enc_API.c:566-568)
enc.n_bits_exceeded += (*n_bytes_out as i32) * 8;
enc.n_bits_exceeded -= silk_div32_16(
    silk_mul(enc_control.bit_rate, enc_control.payload_size_ms), 1000);
enc.n_bits_exceeded = enc.n_bits_exceeded.max(0).min(10000);
```

**Constant definition** — add in `src/silk/encoder.rs` or `src/silk/common.rs`:

```rust
const BITRESERVOIR_DECAY_TIME_MS: i32 = 500;
```

### Fix 2: nBits re-read after damage control

**Location**: `src/silk/encoder.rs`, function `silk_encode_frame_fix`, after the damage control block (~line 7152).

The damage control block ends around line 7152. Add immediately after its closing brace:

```rust
n_bits = range_enc.tell() as i32;
```

Note: `n_bits` must be declared as `let mut n_bits` earlier in the loop (verify it's mutable).

### Fix 3: gain_mult type widening

**Location**: `src/silk/encoder.rs`, ~lines 7008-7009.

```rust
// Before:
let mut gain_mult_lower: i16 = 0;
let mut gain_mult_upper: i16 = 0;

// After:
let mut gain_mult_lower: i32 = 0;
let mut gain_mult_upper: i32 = 0;
```

Update any `as i32` casts on these variables that are now redundant, and ensure the assignment `gain_mult_lower = gain_mult_q8 as i32` (or similar) still compiles. The `gain_mult_q8` stays `i16`.

## Known Limitations

- **LBRR encode not implemented**: `n_bits_used_lbrr` is always 0. The subtraction is correct but has no effect until LBRR is ported.
- **Multi-frame packet splitting**: `tot_blocks`/`curr_block` maxBits splitting not implemented. Only affects 40ms+ payloads; tests use 20ms.
- **useCBR frame position**: CBR is applied to all frames, not just the last. Only affects multi-frame payloads.

## Risks

1. **silk_div32_16 / silk_mul / silk_smulbb availability**: Verified present in `src/silk/common.rs` at lines 932, 1045, 1111 with correct signatures.
2. **First packet equivalence**: For the very first packet, all adjustments are zero so target_rate == bit_rate. The fix primarily helps frames 2+ and multi-packet sessions where `nBitsExceeded` accumulates. Single-packet tests may not change immediately.
3. **n_bits mutability**: Ensure `n_bits` is declared `let mut` in the rate control loop scope.
