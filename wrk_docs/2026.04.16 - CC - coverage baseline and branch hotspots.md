## Analysis of Uncovered Branch Patterns

- **Warped / LTP quantization paths** (lines ~6283-6599): stereo LTP, per-subframe quantizer branches.

- **lpcnet.rs** decoder state continuation and quantization deltas.

## Root Cause Assessment

Lines/regions/functions are excellent (>98%). The gap is entirely in **taken-branch coverage**: conditional expressions with one side evaluated but not the other. This pattern indicates the tests reach the code but don't drive every boolean outcome.

Contributing factors, ordered by estimated impact:
1. **Parameter-space narrowness** in unit tests (fixed bitrate, fixed bandwidth, mono). Branches that only fire for stereo, specific bandwidths, or specific complexity levels are skipped.
2. **Assertion/error-return guards** (`if x.is_some() { … } else { return … }`) where test inputs are always well-formed.
3. **Config-toggle fan-out** (FEC, DTX, VBR, CBR, LBRR, force-channels) where we test one direction but not the other.
4. **Overflow / clamp guards** that need adversarial input.
5. **DNN feature paths** that compile but are rarely invoked without the `dnn` feature flag.
