# Workspace Coverage Push To 98

Date: 2026-04-08

Baseline from `cargo +nightly llvm-cov --workspace --all-features --branch`:

- lines: `91.45%`
- functions: `92.76%`
- regions: `92.09%`
- branches: `76.57%`

## Strategy

Do not chase raw test count. Chase uncovered execution in the files that dominate missed lines and missed branches.

This plan is ordered by expected return, not by module prestige.

## Stage 0: Keep The Runner Honest

Goal:

- preserve a repeatable nightly branch-aware coverage loop

Actions:

- keep using `cargo +nightly llvm-cov --branch`
- always regenerate both JSON summary and missing-line text output
- rerun `cargo test --all-features` or the touched test target before a full coverage pass
- treat any new test failure as a blocker and fix it immediately

Exit criteria:

- coverage run remains green and reproducible

## Stage 1: Low-Risk Deterministic Gains

Primary targets:

- `src\dnn\lpcnet.rs`
- `src\opus\multistream.rs`
- `src\opus\decoder.rs`

Why:

- these files still have large cold regions and can be exercised with deterministic unit tests
- they are much cheaper than `silk::encoder` for the next few points of improvement

Test themes:

- model load/reset/error-state branches
- packet parse and wrapper argument validation
- mapping/layout/projection branch matrices
- short-input and invalid-shape cases

Success signal:

- line coverage rises faster than test count
- branch coverage improves by at least a few tenths without touching the big encoders yet

## Stage 2: Packet And Decoder Branch Closure

Primary targets:

- `src\opus\repacketizer.rs`
- `src\silk\decoder.rs`
- `src\celt\decoder.rs`

Why:

- these files still have concentrated branch debt with manageable setup cost

Test themes:

- malformed packet shapes
- self-delimited / padding / extension edge cases
- PLC / FEC / decoder state-transition branches
- small-buffer and invalid frame-size handling

Success signal:

- branch coverage rises meaningfully even if line gains are smaller

## Stage 3: Attack The Big Encoder State Machines

Primary targets:

- `src\silk\encoder.rs`
- `src\opus\encoder.rs`

Why:

- 98% is not reachable without real movement here
- these two files alone account for `1,154` missed lines and `468` missed branches

Test themes:

- mode-switch and bandwidth-switch state machines
- low-bitrate, constrained-budget, DTX, FEC, and redundancy interactions
- initialization / reuse / reset paths that are still cold
- invalid combinations that force early returns through untested branches

Success signal:

- both files move out of the low-70s branch range

## Stage 4: Residual Sweep

Primary targets:

- whatever still tops the missed-line and missed-branch lists after Stage 3

Likely candidates:

- `src\celt\encoder.rs`
- `src\celt\bands.rs`
- `src\dnn\core.rs`

Rule:

- no broad smoke tests
- only add tests that are justified by fresh uncovered-line data

## Stage 5: Stop Condition Review

Decision rule:

- keep going while a wave materially moves a top hotspot
- if late-stage tests buy only trivial coverage or require brittle internal-state assertions, document that rather than pretending the target is still one more pass away

## First Implementation Wave

Start with disjoint worker scopes:

1. `src\dnn\lpcnet.rs`
2. `src\opus\multistream.rs`
3. `src\opus\decoder.rs`

Then rerun branch coverage and reprioritize from fresh JSON instead of following the original order blindly.
