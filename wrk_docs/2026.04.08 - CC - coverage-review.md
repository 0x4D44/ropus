# Coverage Review

Scope: Rust library coverage only, run with:

```powershell
$env:MDOPUS_SKIP_REFERENCE='1'; cargo +nightly llvm-cov --branch --lib --features "simd dnn"
```

The repo cannot currently run the C reference harness in this workspace because `reference/` does not contain the xiph/opus source tree. The build script now short-circuits cleanly under `MDOPUS_SKIP_REFERENCE=1`, which lets Rust tests and coverage run without inventing fake C symbols.

Latest nightly branch-coverage run:

- Functions: `91.15%`
- Regions: `89.37%`
- Branches: `71.64%`

The encoder file moved to:

- `src/opus/encoder.rs`
- Lines: `81.94%`

That is a real improvement from the initial baseline, but it is still nowhere near 98% overall.

## Baseline Blockers

Two issues prevented a clean baseline at the start:

- `cargo llvm-cov --branch` requires nightly.
- The package build script wanted the upstream C reference tree and failed when `reference/celt/bands.c` was absent.

The workspace now has a working escape hatch for the second issue, so the coverage loop can continue in this sandbox.

- `src/opus/multistream.rs`: `80.97%` lines, `72.49%` branches
- `src/celt/decoder.rs`: `86.01%` lines, `74.39%` branches

These are the files that still have enough density to move the total noticeably.

## What Changed In `src/opus/encoder.rs`

Added tests for:

- `frame_size_select` valid and invalid branches, including long-frame paths
- `compute_equiv_rate` mode-specific and low-complexity paths
- `decide_fec` bandwidth backoff and restore behavior
- `compute_redundancy_bytes` cap behavior
- `compute_frame_energy` and `compute_stereo_width`
- `update_delay_buffer` and `update_delay_buffer_from_pcm_buf`
- mode-transition prefill behavior across CELT and SILK handoffs

The intent was to hit deterministic helper logic first, then branchy encode-state transitions that are hard to reach from the public API.

## Interpretation

The encoder file is still branch-heavy, so the easy wins are gone. Further gains will come from carefully chosen scenario tests that force specific state transitions and packet-shape branches, not from more smoke tests.

The repo-wide 98% target is still a large distance away. Reaching it will require attacking the remaining high-miss files after the encoder pass, especially `dnn::lpcnet` and `silk::encoder`.
