# Build Time Benchmark

**Date:** 2026-05-02 (Machine A), 2026-05-03 (Machine B)

Both runs use the same protocol: cold builds with no prior `target/` artifacts. Debug build first (`cargo build`), then release (`cargo build --release`) — no artifacts shared between profiles.

---

## Machine A — 2017 laptop (i7-7820HQ)

**CPU:** Intel Core i7-7820HQ @ 2.90 GHz — 4 cores / 8 threads
**RAM:** 32 GB
**OS:** Windows 11 Pro (build 26100)
**Rust toolchain:** rustc 1.95.0 (59807616e 2026-04-14) / cargo 1.95.0

| Profile | Wall time | Notes |
|---------|-----------|-------|
| `debug` | **1m 27.7s** | Cold — full dependency graph |
| `release` | **2m 27.2s** | Cold — after debug run (no shared artifacts between profiles) |

Crates compiled: **121** (release pass; debug count expected to be comparable).

Artifact disk usage:

| Location | Size |
|----------|------|
| `target/debug` | 1.26 GB |
| `target/release` | 0.52 GB |
| `target/` total | 1.78 GB |

## Machine B — 2024 desktop (i7-14700KF)

**CPU:** Intel Core i7-14700KF — 20 cores (8P + 12E) / 28 threads, base 3.4 GHz P-core, boost up to 5.6 GHz
**RAM:** 64 GB
**OS:** WSL2 (Ubuntu 24.04.4 LTS, kernel 6.6.87.2-microsoft-standard) on Windows
**Rust toolchain:** rustc 1.92.0 (ded5c06cf 2025-12-08) / cargo 1.92.0

| Profile | Wall time | Notes |
|---------|-----------|-------|
| `debug` | **11.3s** | Cold — full dependency graph |
| `release` | **29.5s** | Cold — after debug run (no shared artifacts between profiles) |

Crates compiled: **123** (release pass).

Artifact disk usage:

| Location | Size |
|----------|------|
| `target/debug` | 1.5 GB |
| `target/release` | 0.58 GB |
| `target/` total | 2.1 GB |

CPU utilisation during release: ~371 s of user CPU over 29.5 s wall ⇒ ~12.6× parallelism, well under the 28-thread cap (LLVM codegen serialises per-crate).

## Comparison

| Metric | Machine A (i7-7820HQ) | Machine B (i7-14700KF) | Speed-up |
|--------|----------------------:|-----------------------:|---------:|
| Cold debug | 87.7 s | 11.3 s | **7.8×** |
| Cold release | 147.2 s | 29.5 s | **5.0×** |
| Threads | 8 | 28 | 3.5× |

---

## Observations

- Debug scales close to thread count (7.8× speed-up vs 3.5× more threads), suggesting the debug build is still fairly parallel-friendly across the dependency graph.
- Release scales worse (5.0×) because LLVM optimisation runs single-threaded per crate; once the parallel front of crates narrows late in the build, extra threads sit idle.
- The earlier prediction in this doc — "a modern 8-core/16-thread machine would likely cut cold release to ~50–60 s" — was conservative. A 14-core/28-thread part lands at 29.5 s, almost 2× faster than that estimate.
- Disk footprint is roughly stable across machines (debug ~1.3–1.5 GB, release ~0.5–0.6 GB); minor variance is from the extra two crates compiled and per-toolchain code-gen differences.
- Incremental builds after a single-file change remain sub-second for debug and a few seconds for release on both machines — cold-build time is the only meaningful axis where hardware matters.
