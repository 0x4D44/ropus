# Quick Smoke Stage 2 Timeout Diagnosis

Date: 2026-05-06

## Scope

Diagnose why `full-test --quick --release-preflight` still times out in Stage 2
after adding `--skip extensions`, without editing source code.

Observed timeout line from implementer validation:

```text
[tests] cargo test --workspace --lib --tests --no-fail-fast -- --test-threads=1 --skip extensions
```

## Root Cause

`--skip extensions` removes only the `tests/conformance/tests/extensions.rs`
test binary. The command is still a full workspace Stage 2 run, so it continues
to execute broad and slow test buckets:

- `ropus` library tests: 1850 tests, about 91 seconds.
- conformance `decode`: exceeds 60 seconds by itself.
- conformance `encode`: exceeds 60 seconds by itself.
- `ropus-harness-control` + `ropus-harness-deep-plc`: about 62 seconds and
  currently fails in `dnn_plc_tier2_snr_above_50db`.
- ordinary package tests such as `ropus-tools-core`, `ropus-fb2k`, CLI crates,
  `capi`, `full-test`, and `ropus-harness`.

The 300s timeout is therefore not caused by one remaining slow test after
`extensions`; it is caused by using a near-full workspace test command for a
smoke path.

## Design Decision

Do not add more global libtest `--skip` filters.

Reason: libtest `--skip` is a substring filter across every scheduled test
binary. Skipping names such as `decode`, `encode`, or `tier2` would be
fragile and would accidentally remove ordinary `ropus` unit tests and other
useful signal. More skip filters would also leave the command conceptually
wrong: it would still be a full workspace run with carve-outs.

Recommended smallest next design change:

For `--quick --release-preflight`, make Stage 2 use a narrower package profile
instead of the full workspace command. Keep the full workspace Stage 2 command
unchanged for non-quick validation.

Suggested quick-preflight Stage 2 shape:

```bash
cargo test --workspace \
  --exclude conformance \
  --exclude ropus-harness-control \
  --exclude ropus-harness-deep-plc \
  --lib --tests --no-fail-fast -- --test-threads=1
```

This should keep meaningful smoke signal from:

- `ropus` core library tests,
- `capi`,
- `ropus-harness` regular C-reference differential tests,
- CLI crates,
- `ropus-tools-core`,
- `ropus-fb2k`,
- `full-test` self-tests.

It removes the exhaustive conformance binaries and the deep/control PLC
harnesses from the quick smoke path. Those suites should remain part of full
validation.

If the target for `--quick --release-preflight` is sub-minute rather than
bounded-under-300s, use an explicit smoke list instead of a workspace package
profile. A package profile still includes `ropus --lib`, which is about 91s
serially on this machine.
