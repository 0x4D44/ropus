# mdopus — HLD: publish as opensource crate (`ropus`)

**Date:** 2026-04-18
**Status:** Approved
**Author:** Claude (with Arthur)

## Goal

Publish the codec to crates.io as a library named `ropus`. A downstream Rust project should be able to `cargo add ropus` and get a safe Opus encoder/decoder with no external C build step, no env vars, and no surprising default-unsafe behaviour.

## Non-goals

- Redesigning the public API. The `OpusEncoder` / `OpusDecoder` / multistream / repacketizer types in `src/opus/` ship as-is. Pre-1.0, no stability guarantee.
- Exhaustive `pub → pub(crate)` audit of all 300+ module-level symbols. We fence the *stable* API at the crate root and document the rest as unstable (see Public API below).
- Wiring DNN into the encode/decode pipeline.
- New functionality: surround (>2ch) validation, platform-specific SIMD, analysis/tonality port.
- CI-based release automation. Manual `cargo publish` for v0.3.0.
- Backwards compatibility with the current single-crate layout. We restructure cleanly; the old paths do not need to keep working.
- Renaming the on-disk repo (`mdopus`) or the project docs. Only the published crate takes the new name.

## Blockers today

1. Root `Cargo.toml` sets `build = "tests/harness/build.rs"`. Every downstream `cargo build` would run it and panic without the `reference/` C source.
2. Nine `[[bin]]` targets point at dev-only paths (`tools/`, `tests/harness/`). They'd ship to every consumer.
3. Manifest missing `description`, `license`, `repository`, `keywords`, `categories`, `authors`, `readme`, `rust-version`.
4. No `LICENSE` file in the repo. **BSD-3-Clause requires preserving the upstream xiph copyright verbatim** (see License section) — our `LICENSE` must reproduce `reference/COPYING`.
5. No crate-root public API re-exports. Users would have to write `ropus::opus::encoder::OpusEncoder` instead of `ropus::OpusEncoder`.
6. Default feature `unchecked-indexing` enables `unsafe { get_unchecked }` in hot paths — wrong posture for a codec parsing untrusted bytes.
7. Stray `src/main.rs` (Hello world stub).

## Decisions (resolved open questions)

| Question | Decision |
|---|---|
| Crate name | `ropus` — verified available on crates.io (API returned 404) |
| License | **BSD-3-Clause** — matches upstream xiph/opus; inherits upstream's IETF IPR patent grants by reference |
| `unchecked-indexing` removal | Approved; macros become plain indexing. Reintroduce only with benchmark evidence. |
| Repository URL | `https://github.com/0x4D44/ropus` (assumed — set to match the crate name) |

## Proposed structure

A 2-crate Cargo workspace. The library crate is publishable and self-contained; everything else stays local.

```
/ (workspace root — on-disk dir name stays "mdopus")
├── Cargo.toml          # workspace manifest + shared profiles
├── LICENSE             # BSD-3-Clause with upstream xiph copyright preserved
├── README.md           # dev workflow (harness, fuzz, coordinator)
├── ropus/              # published library crate
│   ├── Cargo.toml      # no build.rs, no bins, no features; has include whitelist
│   ├── README.md       # crates.io landing page
│   ├── LICENSE         # copy of root (crates.io requires in-dir)
│   └── src/            # was /src, with lib.rs expanded for crate-root re-exports
├── harness/            # dev crate, publish = false
│   ├── Cargo.toml
│   ├── build.rs        # compiles xiph/opus C reference via cc
│   └── src/
│       ├── main.rs     # ropus-compare (was mdopus-compare)
│       ├── bindings.rs
│       └── bin/        # repro_*, replay_*, diff_*, interframe, trace_silk_wb
├── tests/
│   ├── fuzz/           # unchanged (own nested workspace); path dep + imports updated
│   └── vectors/        # unchanged; harness binaries load via CLI paths
├── reference/          # xiph C source, gitignored
└── wrk_docs/, wrk_journals/, notes/, tools/, assets/, logs/   # unchanged
```

Rationale: the library crate becomes pure Rust with zero external build requirements. All test/bench/repro tooling moves to `harness/` which stays in-repo but is never published.

## Concrete changes

### Moves

- `src/*` → `ropus/src/*`
- `tests/harness/{build.rs, bindings.rs, main.rs, interframe_compare.rs}` → `harness/` (build.rs at crate root; other files under `src/`)
- `tools/{replay_fuzz_decode.rs, diff_fuzz_decode.rs, repro_fuzz_roundtrip.rs, trace_silk_wb.rs, repro_bug_d2.rs, repro_bug_k.rs}` → `harness/src/bin/`

### Deletes

- `src/main.rs` (Hello world stub).
- `MDOPUS_SKIP_REFERENCE` env var handling in `build.rs`. After the split the library has no build script, so nothing to skip.
- Every `[[bin]]` entry from the published crate.
- The `[features]` section (see below).

### Renames

- Binary `mdopus-compare` → `ropus-compare` — update `harness/Cargo.toml`, plus every invocation in: `tools/bisect_fix.py`, `tools/coordinator.py` (help text), `tools/integrate.py`, `tools/pgo_build.sh`, `tools/torture_test.sh`, `tools/trace_fix.py`.
- `harness/build.rs`: fix path calculations — `CARGO_MANIFEST_DIR.join("reference")` → `.join("../reference")`; `.join("tests/harness")` → `.join(".")`. The manifest dir is now `harness/`, so relative paths change.

### Fuzz subcrate

`tests/fuzz/` stays as its own nested cargo-fuzz workspace. Required updates:

- `tests/fuzz/Cargo.toml`: change dependency to `ropus = { path = "../../ropus" }`; remove `default-features = false` and `features = ["simd"]` (neither exists after feature-flag removal).
- All 9 files in `tests/fuzz/fuzz_targets/`: `use mdopus::...` → `use ropus::...`.

### Public API

Two changes in `ropus/src/lib.rs`:

**1. Crate-root re-exports** for the types library users construct:

```rust
pub use opus::encoder::{
    OpusEncoder,
    OPUS_APPLICATION_VOIP, OPUS_APPLICATION_AUDIO, OPUS_APPLICATION_RESTRICTED_LOWDELAY,
    OPUS_SIGNAL_VOICE, OPUS_SIGNAL_MUSIC,
    OPUS_AUTO, OPUS_BITRATE_MAX,
};
pub use opus::decoder::{
    OpusDecoder,
    OPUS_OK, OPUS_BAD_ARG, OPUS_BUFFER_TOO_SMALL, OPUS_INTERNAL_ERROR,
    OPUS_INVALID_PACKET, OPUS_UNIMPLEMENTED,
    OPUS_BANDWIDTH_NARROWBAND, OPUS_BANDWIDTH_MEDIUMBAND, OPUS_BANDWIDTH_WIDEBAND,
    OPUS_BANDWIDTH_SUPERWIDEBAND, OPUS_BANDWIDTH_FULLBAND,
};
pub use opus::multistream::{OpusMSEncoder, OpusMSDecoder};
pub use opus::repacketizer::OpusRepacketizer;
```

**2. Stability contract** documented in the crate-level `//!`:

> Stable public API consists of the items re-exported at the crate root. Items reachable via module paths (`ropus::opus::*`, `ropus::celt::*`, `ropus::silk::*`, `ropus::dnn::*`) are accessible but not subject to semver stability guarantees pre-1.0. A full `pub → pub(crate)` audit is deferred to post-1.0.

This defines the semver surface explicitly without requiring us to audit every one of the ~314 `pub` symbols before publishing.

### Keeps

- All codec source, unit tests, property tests, coverage tests, asset docs, fuzz targets, journals.
- `tests/vectors/` at the workspace root; harness binaries take paths as CLI args.

## ropus/Cargo.toml

```toml
[package]
name = "ropus"
version = "0.3.0"
edition = "2024"
rust-version = "1.85"
description = "Safe Rust port of the xiph Opus audio codec (fixed-point), bit-exact against the reference."
license = "BSD-3-Clause"
repository = "https://github.com/0x4D44/ropus"
readme = "README.md"
keywords = ["opus", "audio", "codec", "dsp"]
categories = ["multimedia::audio", "multimedia::encoding"]
authors = ["Martin Davidson <martingdavidson@gmail.com>"]
include = ["src/**/*", "Cargo.toml", "README.md", "LICENSE"]

[dependencies]
wide = "0.7"

[dev-dependencies]
proptest = "1"
```

No `[features]`. No `[build]`. No `[[bin]]`. No `autobins` / `autotests` / `autobenches` toggles. Profiles live in the workspace root, not here. The `include` whitelist is belt-and-braces against accidental packaging of stray local files.

## Workspace Cargo.toml

```toml
[workspace]
resolver = "3"
members = ["ropus", "harness"]
exclude = ["tests/fuzz", "reference"]

[profile.dev]
overflow-checks = false   # match C wrapping semantics (required for bit-exactness)

[profile.release]
lto = "fat"
codegen-units = 1
```

`tests/fuzz/` is already its own nested workspace, so the `exclude` is belt-and-braces.

## Feature flags — all removed

| Flag | Today | After | Rationale |
|------|-------|-------|-----------|
| `simd` | gates the `wide` dep | removed — `wide` becomes unconditional | `wide` is pure Rust, builds on every target; gating it was noise |
| `unchecked-indexing` | wraps `get_unchecked` in `uc!` / `uc_mut!` / `uc_set!` macros | removed — macros become plain indexing | Default-unsafe is the wrong posture for code parsing untrusted bytes. If a real benchmark later shows a material regression, revisit with measurements in hand. |
| `dnn` | declared, never used | removed | Dead flag — DNN module already compiles unconditionally and isn't wired into the pipeline. Zero behavioural change. |

Net: `[features]` section disappears. No env vars. `cargo add ropus` gives the intended build.

## Documentation — minimum bar for v0.3.0

- Crate-level `//!` in `ropus/src/lib.rs`: ~20 lines covering (a) what it is, (b) stability contract (crate-root items are stable; module-path items are not), (c) one runnable code example (encoder → decoder round-trip using the crate-root re-exports).
- 1–3 line rustdoc on each type exported at crate root: `OpusEncoder`, `OpusDecoder`, `OpusMSEncoder`, `OpusMSDecoder`, `OpusRepacketizer`.
- `cargo doc -p ropus --no-deps` builds with no warnings.

Deferred: `#![warn(missing_docs)]`, rustdoc on every `pub fn`, doctest validation. These are post-v0.3.0.

## README strategy

- `/README.md` (existing, revised): dev-workflow focus — harness, fuzz, coordinator, comparison testing. Add a one-liner pointing library users at `ropus/README.md` or the crates.io page.
- `/ropus/README.md` (new, ~50 lines): "What it is" / "Install" / "Quickstart" (using crate-root imports) / "Status (works today / deferred)" / "License" / "Acknowledgements".

## Verification

Gates before `cargo publish`:

```
cargo +1.85.0 check -p ropus                          # MSRV sanity (wide, proptest)
cargo build                                           # whole workspace compiles
cargo test -p ropus                                   # unit + property tests green
cargo test -p ropus-harness                           # integration parity vs C reference green
cargo check --manifest-path tests/fuzz/Cargo.toml     # fuzz targets still compile
cargo doc -p ropus --no-deps                          # zero doc warnings
cargo package -p ropus --list                         # inspect packaged file list
cargo publish -p ropus --dry-run                      # simulated publish succeeds
```

Pass criteria:
- All existing tests green. Feature-flag removal and crate-root re-export additions must not regress behaviour.
- Packaged tarball contains only `ropus/{Cargo.toml, README.md, LICENSE, src/**}`.
- `cargo publish --dry-run` exits zero.
- Crate-root imports work: `use ropus::{OpusEncoder, OpusDecoder};` compiles in an external smoke test.

## Risks

1. **`unchecked-indexing` removal** — we don't have a published perf baseline, so the delta is unknown. Acceptable: safe-default is correct; reintroduce only with measurement support, and via a proper API rather than a feature flag.
2. **Crate-root re-exports collide with module names.** `ropus::encoder` vs `ropus::OpusEncoder` — no collision expected, but verify after the edit.
3. **Rust 2024 + `resolver = "3"`** — MSRV pinned at 1.85; older toolchains won't compile. Acceptable since we already require 2024.
4. **Semver surface is wider than ideal.** We ship every module-level `pub` symbol as reachable-but-unstable. A future 1.0 release will want to tighten this. Documented; no blocker for 0.3.0.

## Explicitly out of scope

- CI / automated publish.
- `#![warn(missing_docs)]` enforcement, doctests.
- Full `pub → pub(crate)` audit.
- Benchmarks, platform-SIMD expansion, DNN wiring.
- Stable-API / semver commitments (pre-1.0).
