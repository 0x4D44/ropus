## 2. Function signatures

```rust
const DRED_BITS_TABLE: [f32; 16] = [
    73.2, 68.1, 62.5, 57.0, 51.5, 45.7, 39.9, 32.4,
    26.4, 20.4, 16.3, 13.0, 9.3, 8.2, 7.2, 6.4,
];

/// Mirrors C `estimate_dred_bitrate`. Returns `(estimated_bits, target_chunks)`.
/// We return a tuple instead of taking a nullable out-pointer because the only
/// call site always wants both — idiomatic Rust departure, behaviour-equivalent.
fn estimate_dred_bitrate(q0: i32, d_q: i32, qmax: i32, duration: i32, target_bits: i32) -> (i32, i32);

/// Free function taking `&mut OpusEncoder`. Style matches existing C-port siblings
/// `user_bitrate_to_bitrate`, `compute_redundancy_bytes`. Writes back four
/// `dred_*` fields per call (mirrors C 725-728).
fn compute_dred_bitrate(enc: &mut OpusEncoder, bitrate_bps: i32, frame_size: i32) -> i32;
```

**Reasoning.** Free functions match how the existing C-port helpers are arranged in this file. `estimate_dred_bitrate` is pure; `compute_dred_bitrate` writes four fields back to encoder state, hence `&mut OpusEncoder`.

## 3. Threading of `dred_bitrate_bps`

Match C exactly — pass as parameter through `encode_frame_native` (mirrors C 1174, 1212, 1806, 1842, 1859):

1. Compute once in `encode_native_with_analysis` after CBR adjustment finalises `self.bitrate_bps` (~line 1692).
2. Apply F53.
3. Pass as `i32` parameter into both `encode_multiframe` and `encode_frame_native`.
4. Forward from `encode_multiframe` into per-sub-frame `encode_frame_native` calls.

**Why parameter, not stored on `self`:** matches C model verbatim (audit just verified line-for-line); avoids manufacturing extra state fields; one F53 mutation point is enough.

## 4. The four divergences — exact resolutions

### F33 — DRED CBR override

Location: `encoder.rs` ~line 3126, immediately after VBR/HYBRID block (3112-3126), **before** prefill block at 3132. Inside existing `if let Some(ref mut celt) = self.celt_enc` borrow scope and inside `if self.mode != MODE_SILK_ONLY`.

```rust
if self.use_vbr == 0 && self.dred_duration > 0 {
    let mut celt_bitrate = self.bitrate_bps;
    celt.ctl(CeltEncoderCtl::SetVbr(1));
    celt.ctl(CeltEncoderCtl::SetVbrConstraint(0));
    if self.mode == MODE_HYBRID {
        celt_bitrate -= self.silk_mode.bit_rate;
    }
    celt.ctl(CeltEncoderCtl::SetBitrate(celt_bitrate));
}
```

### F53 — `bitrate_bps -= dred_bitrate_bps`

Location: `encoder.rs` ~line 1692, immediately after CBR block sets `self.bitrate_bps`. Sequence:

1. CBR adjustment writes `self.bitrate_bps`.
2. (NEW) `let dred_bitrate_bps = compute_dred_bitrate(self, self.bitrate_bps, frame_size);`
3. (NEW) `self.bitrate_bps -= dred_bitrate_bps; let bitrate_bps = self.bitrate_bps;` (rebind local).
4. `let max_rate = …` (existing) now sees the post-DRED bitrate.

## 6. Risk register

| Risk | Severity | Mitigation |
|---|---|---|
| **f32 determinism across compilers** — five sequential f32 ops in `compute_dred_bitrate`, table lookups, final `floor(.5+bits) → i32` cast. | Medium | (a) Plain f32, no `mul_add`, mirror C order. (b) `f32` not `f64` for intermediates where C is `float`. (c) `(0.5_f32 + bits).floor() as i32`, NOT `(bits as f64).round() as i32`. (d) If divergence: print all intermediates and add a comment naming the suspect op. Harness already runs with `-ffp-contract=off` C-side. |
| **Parameter-passing churn** — adding parameter to `encode_frame_native` touches ~7 call sites. | Low | Test sites pass `0` (no-DRED behaviour exact: F33/curr_max-tweak guarded by `dred_duration > 0`, F53 happens before the call). |
| **CBR rebinding in `encode_native_with_analysis`** — F53 mutates `self.bitrate_bps` after the local shadow captured post-CBR value. | Low | Re-bind: `let bitrate_bps = self.bitrate_bps;` after subtraction. C does the same via `st->`. |
| **F33 interacts with budget steal at C 2399-2411** — order matters. | Medium | Strictly mirror C: budget steal first (~3043 = C 2386-2413), then F33 setters (~3126 = C 2470-2477). |
