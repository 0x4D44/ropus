# Measurement: Platform/Sanitizer Smoke

Date: 2026.05.06

Scope: proof notes for the platform/sanitizer breadth implementation.

## Generic Profile Proof

Goal: verify that the planned generic x86_64 command actually escapes the
repository default `.cargo/config.toml` `x86-64-v3` codegen profile.

Commands:

```text
timeout 120 env CARGO_TARGET_DIR=target/platform-breadth/probe-default cargo rustc -p ropus --lib -- --print cfg
timeout 120 env CARGO_TARGET_DIR=target/platform-breadth/probe-generic RUSTFLAGS='-C target-cpu=x86-64' cargo rustc -p ropus --lib -- --print cfg
```

Observed result:

- Default profile emitted v3 feature cfgs including `avx`, `avx2`, `bmi1`,
  `bmi2`, `f16c`, `fma`, `lzcnt`, `movbe`, `popcnt`, `sse3`, `sse4.1`,
  `sse4.2`, `ssse3`, and `xsave`.
- Generic profile emitted only `fxsr`, `sse`, and `sse2` CPU feature cfgs.

Conclusion: on this host, `RUSTFLAGS='-C target-cpu=x86-64'` represents the
intended generic x86_64 profile for the scoped cargo command.
