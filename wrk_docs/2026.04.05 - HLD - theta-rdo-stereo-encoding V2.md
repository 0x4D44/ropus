# HLD: theta_rdo Stereo Encoding — V2

## Changes from V1
- **NEW Change 0**: Fix zeroed Y buffer — the actual root cause of stereo failures, independent of theta_rdo
- **Revised Change 3**: Buffer byte save/restore now uses direct byte copy matching C, not `save_snapshot()`
- **Revised Change 5**: No EcCoder trait extension — use `snapshot()`/`restore()` for scalars + manual `buffer()` access
- **Added**: BandCtx cannot be cloned (holds `&mut EC`); save/restore only scalar fields
- **Added**: `special_hybrid_folding` re-call between passes when `i == start+1`
- **Added**: fold accumulation loop must be do-while (at least 1 iteration when fold_start < fold_end)

---

## Problem

All 30 stereo non-silence encode tests fail. Two independent bugs:

1. **Zeroed Y buffer**: The Rust encoder passes a freshly-allocated zero buffer as the second channel's normalized spectral data to `quant_all_bands`, instead of the actual second channel from `x_norm[n..]`. This means the second channel is always encoded as silence. This bug is independent of theta_rdo and affects ALL stereo encoding at ALL complexity levels.

2. **theta_rdo disabled**: For complexity >= 8, the C encoder tries two theta quantization roundings per stereo band and picks the lower-distortion one. The Rust hardcodes `theta_rdo = false`.

## Design

### Change 0: Fix zeroed Y buffer (prerequisite)

In `celt_encode_core` (`src/celt/encoder.rs` ~line 2927):

```rust
// Before (BUG: zeroed buffer):
let mut y_norm_buf = vec![0i32; n as usize];
let y_norm_opt = if c == 2 {
    Some(&mut y_norm_buf[..])
} else {
    None
};
quant_all_bands(true, mode, start, end, &mut x_norm, y_norm_opt, ...);

// After (correct: pass second channel from x_norm):
let (x_part, y_part) = if c == 2 {
    let (a, b) = x_norm.split_at_mut(n as usize);
    (a, Some(b))
} else {
    (&mut x_norm[..], None)
};
quant_all_bands(true, mode, start, end, x_part, y_part, ...);
```

This matches C: `quant_all_bands(1, mode, start, end, X, C==2 ? X+N : NULL, ...)`.

**Test after this change alone** — many stereo tests may flip to PASS even without theta_rdo, since theta_rdo only activates at complexity >= 8 and makes small quantization adjustments.

let resynth_alloc = if theta_rdo {

### Change 3: The two-pass block

In the stereo non-dual-stereo band loop, for bands below the intensity boundary:

```rust
if theta_rdo && i_band < intensity {
    let w = compute_channel_weights(band_e[iu], band_e[iu + nbe]);
    let cm = x_cm | y_cm;

    // --- Save state before pass 1 ---
    let ec_snap1 = ec.snapshot();           // scalar fields only (offs, rng, val, etc.)
    let nstart = ec.range_bytes();          // front cursor position
    let nend = ec.storage();                // total buffer capacity
    // Save BandCtx scalar fields (BandCtx holds &mut EC, cannot be cloned)
    let (seed_save, remaining_bits_save) = (ctx.seed, ctx.remaining_bits);
    x_save[..nu].copy_from_slice(&x[x_off..x_off + nu]);
    y_save[..nu].copy_from_slice(&y[y_off..y_off + nu]);

    // --- Pass 1: round down ---
    ctx.theta_round = -1;
    let (cm_out_1, _) = quant_band_stereo(/* ... */);
    let dist0 = mult16_32_q15(w[0], celt_inner_prod_norm_shift(&x_save[..nu], &x[x_off..x_off+nu], nu))
              + mult16_32_q15(w[1], celt_inner_prod_norm_shift(&y_save[..nu], &y[y_off..y_off+nu], nu));

    // --- Save pass-1 result ---
    let ec_snap2 = ec.snapshot();
    let (seed_save2, remaining_bits_save2) = (ctx.seed, ctx.remaining_bits);
    x_save2[..nu].copy_from_slice(&x[x_off..x_off + nu]);
    y_save2[..nu].copy_from_slice(&y[y_off..y_off + nu]);
    if !last { norm_save2[..nu].copy_from_slice(&norm[norm_off..norm_off + nu]); }
    // Save buffer bytes (C: bytes from pre-trial offset to end)
    let save_bytes = nend - nstart;
    bytes_save[..save_bytes].copy_from_slice(&ec.buffer()[nstart..nend]);

    // --- Restore pre-pass-1 state ---
    ec.restore(&ec_snap1);
    ctx.seed = seed_save;
    ctx.remaining_bits = remaining_bits_save;
    x[x_off..x_off + nu].copy_from_slice(&x_save[..nu]);
    y[y_off..y_off + nu].copy_from_slice(&y_save[..nu]);
    // Re-call special_hybrid_folding if i_band == start + 1
    if i_band == start + 1 {
        special_hybrid_folding(m, &mut norm, &mut norm2, start as usize, big_m, dual_stereo);
    }

    // --- Pass 2: round up ---
    ctx.theta_round = 1;
    let (cm_out_2, _) = quant_band_stereo(/* ... */);
    let dist1 = mult16_32_q15(w[0], celt_inner_prod_norm_shift(&x_save[..nu], &x[x_off..x_off+nu], nu))
              + mult16_32_q15(w[1], celt_inner_prod_norm_shift(&y_save[..nu], &y[y_off..y_off+nu], nu));

    // --- Pick winner ---
    if dist0 >= dist1 {
        // Pass 1 (round-down) was better — restore its state
        ec.restore(&ec_snap2);
        ctx.seed = seed_save2;
        ctx.remaining_bits = remaining_bits_save2;
        x[x_off..x_off + nu].copy_from_slice(&x_save2[..nu]);
        y[y_off..y_off + nu].copy_from_slice(&y_save2[..nu]);
        if !last { norm[norm_off..norm_off + nu].copy_from_slice(&norm_save2[..nu]); }
        ec.buffer_mut()[nstart..nend].copy_from_slice(&bytes_save[..save_bytes]);
        (x_cm, y_cm) = (cm, cm);  // C uses cm from before the split
    }
    // else: pass 2 state already in place
} else {
    ctx.theta_round = 0;
    // existing single-pass quant_band_stereo call
}
```

**Key design decisions for buffer save/restore:**
- `ec.snapshot()` / `ec.restore()` handle the scalar encoder fields (offs, rng, val, end_offs, etc.) WITHOUT touching buffer bytes
- Buffer byte save/restore is done manually via `ec.buffer()` / `ec.buffer_mut()`, saving the region `[nstart..nend]` where nstart was the write cursor before any trial encoding
- This matches the C approach exactly: `ec_save = *ec` copies scalars, `OPUS_COPY(bytes_save, bytes_buf, save_bytes)` saves the dirty buffer region

### Change 4: Fix fold_start off-by-one

```rust
// Before (tests then decrements — off by one):
while big_m * m.ebands[fold_start - 1] as i32 > effective_lowband + norm_offset as i32 {
    fold_start -= 1;
    if fold_start == 0 { break; }
}

// After (decrements then tests — matches C pre-decrement):
loop {
    fold_start -= 1;
    if !(big_m * m.ebands[fold_start] as i32 > effective_lowband + norm_offset as i32) { break; }
}
```

Also fix the fold accumulation loop to be do-while (at least 1 iteration):

```rust
// Before (for loop, may be zero iterations):
for fold_i in fold_start..fold_end { ... }

// After (do-while, at least 1 iteration when fold_start < fold_end):
let mut fold_i = fold_start;
loop {
    // accumulate x_cm/y_cm from norm data
    fold_i += 1;
    if fold_i >= fold_end { break; }
}
```

### Change 5: EcCoder method access

The `quant_all_bands` function is generic over `EC: EcCoder`. The theta_rdo block needs `snapshot()`, `restore()`, `buffer()`, `buffer_mut()`, `range_bytes()`, `storage()` — these exist on `RangeEncoder` but not the trait.

**Approach**: Add these to the `EcCoder` trait with default implementations that panic. This is safe because theta_rdo is guarded by `encode == true`, so only `RangeEncoder` will execute these paths.

```rust
// In ec_ctx.rs, add to EcCoder trait:
fn snapshot(&self) -> RangeSnapshot { unimplemented!("encoder-only") }
fn restore(&mut self, _snap: &RangeSnapshot) { unimplemented!("encoder-only") }
fn range_bytes(&self) -> usize { unimplemented!("encoder-only") }
fn storage_size(&self) -> usize { unimplemented!("encoder-only") }
fn buffer(&self) -> &[u8] { unimplemented!("encoder-only") }
fn buffer_mut(&mut self) -> &mut [u8] { unimplemented!("encoder-only") }
```

The `RangeSnapshot` type is a simple struct holding the scalar fields — it already exists in `range_coder.rs` as the return type of the existing `snapshot()` method.

## Risks

1. **Buffer byte save/restore correctness**: The manual `buffer()[nstart..nend]` approach directly mirrors C. The risk is if `nstart`/`nend` are computed differently than C's `ec_save.offs`/`ec_save.storage`. Verify `range_bytes()` returns `offs` and `storage_size()` returns `storage`.

2. **Norm buffer save/restore**: Must save `norm[norm_off..norm_off+nu]` after pass 1 and restore if pass 1 wins. Omitting this corrupts inter-band folding for subsequent bands.

3. **`special_hybrid_folding` re-call**: Only needed when `i_band == start + 1`. Forgetting this changes the folding basis for the second trial.
