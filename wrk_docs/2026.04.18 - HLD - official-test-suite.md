# mdopus ‚Äî HLD: run the official xiph/opus test suite

**Date:** 2026-04-18
**Status:** Proposed (v2 ‚Äî scope widened to full test set after review)
**Author:** Claude (with Arthur)

## Goal

Run every test program in `reference/tests/` that we can compile against a fixed-point build ‚Äî `test_opus_api.c`, `test_opus_encode.c`, `test_opus_decode.c`, `test_opus_padding.c`, `test_opus_extensions.c` ‚Äî against **our** Rust codec. Pass/fail is binary: the same source that validates xiph/opus must validate mdopus.

Our FFI differential harness only proves "Rust matches C on inputs we picked." This closes the contract and spec-level gap.

## Oracles

This work adds a new class of testing oracle to the project, distinct from what we already have.

| Oracle | What it encodes | Blind spot |
|---|---|---|
| Differential FFI harness + fuzz | *Rust output ‚â° C reference output*, byte-for-byte | Bugs where Rust and C agree but both are wrong |
| Unit tests in `src/` | Hand-specified expected values for leaf functions | Integration-level and API-contract behaviour |
| **Official test suite (this work)** | **API contracts and regression assertions written by the codec authors** | **Not a full spec oracle ‚Äî only what xiph chose to assert** |

The critical gap the official suite closes: **shared-with-C bugs.** Our Rust was ported from C, so if we faithfully copied a C bug, differential testing masks it forever ‚Äî both sides agree. The assertion-based tests catch this because they're written against "what correct looks like," not against the other implementation.

Worth flagging honestly: passing `test_opus_*.c` means "no known-bug regressions + common API contracts hold," not "conforms to RFC 6716 in full." The IETF conformance vectors via `run_vectors.sh` are the canonical spec oracle; that remains a follow-up.

## Non-goals

- Exposing a stable, published C ABI. The `capi/` crate is a dev-only shim for the reference tests; not `libopus.so`-as-a-product.
- IETF RFC 6716 vectors via `run_vectors.sh`. That drives `opus_demo` (a full CLI we'd have to port). Follow-up.
- `test_opus_dred.c`, `test_opus_custom.c`, `test_opus_projection.c`. DRED decoder-side inference, custom-mode encoder, and ambisonics projection are whole modules we haven't ported. Separate scope.
- Porting or modifying any `reference/tests/test_opus_*.c` source. They compile verbatim; that is the conformance value.
- Performance parity.

## What came out of scope review

v1 of this doc treated DRED and `opus_private.h` as blockers for the encode/extensions tests. Fresh look:

- **DRED is `#ifdef ENABLE_DRED`** in every test. If we don't define the macro, every DRED call site compiles out. No stubs, no ports.
- **`opus_private.h` usage is small.** `test_opus_encode.c` references only `MODE_SILK_ONLY` (a `#define`) and `OPUS_SET_FORCE_MODE` (a CTL). `test_opus_extensions.c` uses `opus_extension_data` (a struct), `opus_packet_parse_impl`, `opus_repacketizer_out_range_impl`, and the five `opus_packet_extensions_*` functions. No test reads fields from `struct OpusRepacketizer`, `OpusMSEncoder`, etc., so those can stay opaque.
- **`opus_pcm_soft_clip`** is required by `test_opus_decode.c` (9 call sites). ~130 LOC of DSP. Straight port.

What's genuinely new work is therefore ~400‚Äì600 LOC of Rust codec additions plus the capi wrapper.

## Building and running the tests

**Not** "build.rs compiles executables" (that's off-road in cargo). Instead:

1. `build.rs` in `tests/conformance/` loops over the five `.c` files. For each: `cc::Build::new().define("main", "test_opus_api_main").file("...").include("include").compile("test_opus_api")` ‚Äî produces a static lib with `main` renamed uniquely.
2. `tests/run.rs` declares each renamed main as `extern "C" fn test_opus_api_main(argc: c_int, argv: *const *const c_char) -> c_int` and wraps each in `#[test] fn api() { assert_eq!(unsafe { test_opus_api_main(0, ptr::null()) }, 0); }`.
3. `cargo test -p conformance -- --test-threads=1` runs them. `--test-threads=1` is required because tests share `fast_rand()` global state.

This keeps everything cargo-native ‚Äî per-test pass/fail in the normal test output, no spawned processes.

## CTL varargs handling

The `opus_*_ctl` interface is C variadic. We avoid the unstable `c_variadic` feature with a C shim that does all `va_arg` work; Rust only sees typed scalars.

```c
// capi/src/ctl_shim.c
int opus_encoder_ctl(OpusEncoder *st, int request, ...) {
    va_list ap; va_start(ap, request);
    int ret;
    switch (request) {
        case OPUS_SET_BITRATE_REQUEST: case OPUS_SET_COMPLEXITY_REQUEST:
        case OPUS_SET_APPLICATION_REQUEST: /* ...int-setters... */
        case OPUS_SET_FORCE_MODE_REQUEST:
            ret = mdopus_encoder_ctl_set_int(st, request, va_arg(ap, int)); break;

        case OPUS_GET_BITRATE_REQUEST: case OPUS_GET_COMPLEXITY_REQUEST:
        case OPUS_GET_SAMPLE_RATE_REQUEST: /* ...int*-getters... */
            ret = mdopus_encoder_ctl_get_int(st, request, va_arg(ap, int*)); break;

        case OPUS_GET_FINAL_RANGE_REQUEST:
            ret = mdopus_encoder_ctl_get_uint32(st, request, va_arg(ap, opus_uint32*)); break;

        case OPUS_RESET_STATE:
            ret = mdopus_encoder_ctl_reset(st); break;

        default: ret = OPUS_UNIMPLEMENTED;
    }
    va_end(ap); return ret;
}
```

Three entry points per state type (`set_int`, `get_int`, `get_uint32`, `reset`) ‚Äî four for decoders to handle `OPUS_GET_PITCH`, `OPUS_GET_GAIN`, `OPUS_GET_LAST_PACKET_DURATION`. Enumerated from `opus_defines.h` once; drift is caught by the `default` arm returning `OPUS_UNIMPLEMENTED`.

## Panic safety

`src/opus/` has ~757 panic sites (`unwrap`, `expect`, bounds-checked indexing). Unwinding across an `extern "C"` boundary that the compiler doesn't mark `extern "C-unwind"` is UB. Two guards:

1. **`panic = "unwind"`** in the capi crate's test profile (default on most targets but explicit).
2. **`catch_unwind` at every FFI entry point.** A helper macro `ffi_guard!(OPUS_INTERNAL_ERROR, { ... })` wraps the body and returns the error code on panic. Boilerplate, but mechanical.

We accept that a panic becomes `OPUS_INTERNAL_ERROR` rather than a test abort ‚Äî which is what the C codec does on assertion failure anyway.

## `OpusRepacketizer<'a>` lifetime

Our Rust struct has `'a` because it holds `&'a [u8]` slices pointing into the caller's input. C has no lifetimes; the capi layer erases by either:

- Introducing `struct OpusRepacketizerOwned` that holds `*const u8` + `i32 len` pairs internally, no lifetime, `#[repr(C)]`. Same logic as the current struct, typed for raw pointers. Hidden behind the capi boundary; `mdopus`'s lifetimed API is untouched.

Preferred: the second. ~50 LOC of duplication, but keeps the public Rust API clean.

## State lifecycle

C tests use two patterns. Both must work:
1. `*_create()` + `*_destroy()` ‚Äî heap allocation.
2. `size = *_get_size(); p = malloc(size); *_init(p, ‚Ä¶)` ‚Äî caller-allocated storage.

Plan:
- `*_create`: `Box::into_raw(Box::new(state))`. `*_destroy`: `Box::from_raw` + drop.
- `*_get_size`: `mem::size_of::<State>()`. Must be large enough that stack allocations in tests (via the sized-opaque struct in vendored `opus_private.h`) fit.
- `*_init`: `ptr::write(ptr as *mut State, State::default())` then call real init routine.

Our state structs (`OpusEncoder`, `OpusDecoder`, `OpusMSEncoder`, `OpusMSDecoder`, `OpusRepacketizerOwned`) hold `Option<Vec<_>>` sub-encoder state. `get_size` returns the Rust struct size (plenty of room for heap-pointer-sized fields); tests allocate storage of at least that size. Vecs allocate lazily in sub-encoder init ‚Äî the storage handed to `*_init` doesn't have to back them.

No test reads fields of these structs directly, so layout divergence from the C side is invisible.

## Phasing

Reordered after discovering `test_opus_api.c` (1916 lines) is actually the broadest-surface test ‚Äî exercises multistream, repacketizer, and packet pad/unpad in full. We start with the narrowest test and grow the capi surface incrementally.

1. **Skeleton + `test_opus_padding.c`.** Minimal `capi/` exporting encoder, decoder, and `opus_packet_pad`/`unpad`. Establishes build plumbing, CTL shim, panic guards, state lifecycle. ~2 days.
2. **Decode + soft-clip.** Port `opus_pcm_soft_clip`. Wire decoder CTLs. `test_opus_decode.c` passes. ~2 days.
3. **Multistream + repacketizer ‚Üí `test_opus_api.c`.** Full repacketizer wrappers + multistream encoder/decoder wrappers. `test_opus_api.c` passes (it's the one that transitively needs both). ~3 days.
4. **Encode.** `OPUS_SET_FORCE_MODE` CTL + whatever test_opus_encode.c needs that phases 1‚Äì3 didn't already cover. `test_opus_encode.c` passes (with `ENABLE_DRED` undefined). ~1 day.
5. **Extensions.** Port `OpusExtensionIterator` + five `opus_packet_extensions_*` functions. Export `opus_packet_parse_impl`, `opus_repacketizer_out_range_impl`. `test_opus_extensions.c` passes. ~3 days.

Total: **~11 working days** (~2.5 calendar weeks with slack).

## Success criteria

`cargo test -p conformance -- --test-threads=1` runs all five test binaries to exit-code-0 on Linux and Windows, with zero modifications to the `.c` sources.

## Known risks

- **Surround (>2 channels).** `test_opus_encode.c` does exercise multichannel/multistream. Our multistream port handles stereo; surround is unvalidated (per the publish-as-crate HLD). If the test enables surround paths we haven't exercised, phase 4 could slip. Mitigation: phase 4 starts by reading the test's actual channel configurations before coding; if surround is hit, we scope it in explicitly.
- **CTL drift.** New CTLs land upstream faster than we transcribe. Mitigation: `default: OPUS_UNIMPLEMENTED` ‚Äî the test fails loudly with a clear error code; one-line patch to add the mapping.
- **`opus_packet_extensions_generate` round-trip.** Encoding + decoding extensions is bit-exact sensitive. Mitigation: this is the first place to apply the existing FFI differential harness ‚Äî feed both impls the same inputs and compare byte-for-byte before running the conformance test.
- **Panic on internal invariant.** If our codec has a latent panic on some edge case the conformance suite probes, we surface it as `OPUS_INTERNAL_ERROR`. That reads as a test failure (good ‚Äî it's a real bug) but the bug will need fixing in `src/`, not in `capi/`.
