# Quick Smoke Conformance Skip HLD

Date: 2026.05.06

Scope: design only. No source implementation in this round.

## Problem Statement

The intended smoke command:

```text
cargo run -p full-test -- --release-preflight --quick --skip-coverage --skip-benchmarks --skip-ambisonics
```

does not behave like a quick smoke check because Stage 2 still runs the full
workspace test suite. That includes `/home/md/language/ropus/tests/conformance/tests/extensions.rs`,
whose single `extensions` test calls the upstream
`test_opus_extensions_main()` and reaches `test_random_extensions_parse()` in
`/home/md/language/ropus/reference/tests/test_opus_extensions.c`, looping
100,000,000 times.

This is expected long execution, not a hang. The runner needs a small,
explicit policy for the quick release-preflight smoke path so it preserves
useful test signal without paying for the very slow conformance extension
stress test.

## Stakeholder Value

Arthur and other agents need a bounded command for release-preflight smoke
validation. The value is operational confidence: the command should quickly
catch build, harness, preflight asset, and ordinary test regressions without
appearing stuck on an upstream stress loop.

The full conformance signal still matters, but it belongs in the non-quick
validation path, not in this smoke path.

## Considered Options

1. Keep current behavior and document the delay.
   - Reject. A command named `--quick --release-preflight` should not spend
     its time in a 100,000,000-iteration conformance stress loop.

2. Add a dedicated CLI flag such as `--skip-extensions`.
   - Reject for now. The user-facing surface is broader than the immediate
     need, and it invites ad hoc test selection policy before we need it.

3. For `--quick --release-preflight`, append `--skip extensions` to Stage 2's
   libtest arguments.
   - Recommended. It is the smallest implementation, keeps Stage 2 as one
     cargo invocation, and matches the known binary/test name.

4. Split Stage 2 into multiple invocations so the conformance `extensions`
   binary can be excluded precisely.
   - Defer. This gives more exact selection but adds command orchestration,
     result aggregation, coverage-shape complexity, and more parsing surface.
     It is not justified for this smoke-path fix.

## Recommended Design

Add an internal Stage 2 policy boolean, not a new public CLI flag:

```rust
let skip_slow_conformance = options.quick && options.release_preflight;
```

Pass it from `/home/md/language/ropus/full-test/src/main.rs` into
`tests::run(...)` in `/home/md/language/ropus/full-test/src/tests.rs`.

Extend `build_cargo_args(...)` in
`/home/md/language/ropus/full-test/src/tests.rs` to append this libtest filter
when `skip_slow_conformance` is true:

```text
--skip extensions
```

The resulting smoke command shape with coverage skipped should be:

```text
cargo test --workspace --lib --tests --no-fail-fast -- --test-threads=1 --skip extensions
```

If IETF vectors are unavailable, preserve the existing `--skip testvector`
filter as well. Multiple libtest `--skip` filters are acceptable; the tail
should become:

```text
--test-threads=1 --skip testvector --skip extensions
```

Update the Stage 2 printed header so the rendered command itself shows the
smoke policy, for example by including this tail when active:

```text
--test-threads=1 --skip extensions
```

Do not change `--quick` globally. `cargo run -p full-test -- --quick
--skip-coverage` without `--release-preflight` should keep the current Stage 2
behavior.

Reason: plain `--quick` currently means "skip quality and benchmarks, keep the
test stage intact." Expanding every quick run to filter conformance tests would
silently weaken existing local validation. The bounded exception belongs to
`--quick --release-preflight` because that command is explicitly a smoke path
for release asset/preflight confidence, not a full conformance pass.

Update `/home/md/language/ropus/full-test/src/cli.rs` help text as part of the
implementation. This is mandatory, not optional. The current line:

```text
--quick             Skip stages 1 (quality) and 4 (benchmarks).
```

should be revised to disclose the exception, for example:

```text
--quick             Skip stages 1 and 4; with --release-preflight, also filters the slow extensions stress test.
```

Keep the parser unchanged; only the help text needs to move with the behavior.

### Visibility And Skip Reasons

Keep visibility small and command-shaped. No broad report model refactor.

Minimum required visibility:

- The Stage 2 printed header must include the actual cargo/libtest command with
  `--skip extensions` when the policy is active.
- Unit tests for `build_cargo_args(...)` must assert that `--skip extensions`
  appears only for the quick release-preflight smoke policy.
- The HTML/report can rely on the existing captured Stage 2 command/header if
  that is already what the runner surfaces; do not add a new structured report
  field just for this.

Optional small note: it is acceptable to append a plain-text note to the
existing `tests.skip_reason` string, but only without hiding the IETF-vector
synthetic failure behavior. If both reasons apply, combine them with a stable
separator:

```text
IETF vectors unavailable after provisioning; filtered testvectorNN_mono/stereo probes to preserve non-IETF Stage 2 signal. <reason>; quick release-preflight smoke filtered slow conformance test: extensions
```

If only the slow conformance filter applies, the skip reason may be:

```text
quick release-preflight smoke filtered slow conformance test: extensions
```

Do not change `append_ietf_provisioning_failure(...)`: unavailable IETF vectors
must still add the synthetic failed test so the gate cannot pass green without
the spec vectors.

## `libtest --skip extensions` Risk

`libtest --skip` is substring-based and global to every test binary that cargo
runs. `--skip extensions` can skip any test whose full name contains
`extensions`, not only `/home/md/language/ropus/tests/conformance/tests/extensions.rs`.

This is real risk. It can hide future fast tests that happen to contain the
same substring.

Decision: the minimal global skip is acceptable for this specific smoke path.
The reason is that `--quick --release-preflight` is not the full conformance
gate; it is a bounded smoke check. The skip should be explicit in code comments,
unit tests, terminal/report text, and validation notes. The non-quick path must
continue to run the extension conformance test.

Do not split Stage 2 yet. Split only if this policy grows beyond one known slow
test, if we need exact package/binary-level accounting, or if skipped-test
visibility becomes insufficient for release decisions.

## Affected Files

- `/home/md/language/ropus/full-test/src/main.rs`
  - Compute `skip_slow_conformance` from `options.quick && options.release_preflight`.
  - Pass it to Stage 2.

- `/home/md/language/ropus/full-test/src/tests.rs`
  - Add a `skip_slow_conformance` parameter to `run(...)` and
    `build_cargo_args(...)`.
  - Append `--skip extensions` after `--test-threads=1` when enabled.
  - Keep the Stage 2 header's rendered command accurate.
  - Optionally combine a plain-text `tests.skip_reason` note with the existing
    IETF-vector note using `; `, without altering the synthetic IETF failure.

- `/home/md/language/ropus/full-test/src/cli.rs`
  - No parser change.
  - Mandatory help text update so `--quick` discloses the
    `--quick --release-preflight` slow conformance filter exception.

## Validation Plan

Run the smallest checks first:

```text

timeout 120 cargo test -p full-test cli::tests
```

Then validate the command shape indirectly through the real smoke command:

```text
timeout 300 cargo run -p full-test -- --release-preflight --quick --skip-coverage --skip-benchmarks --skip-ambisonics
```

Expected result: Stage 2 starts with a cargo test command containing
`--skip extensions`, completes without entering the long extension stress loop,

```text

```

The full non-quick workspace/conformance command does not need to be run during

known to be long by design.

## Risks And Tradeoffs

- Global substring skip can hide other tests containing `extensions`.
  - Accepted only for `--quick --release-preflight`; rejected for the default
    and non-quick paths.

- The smoke command loses one conformance stress test.
  - Accepted because the goal is bounded smoke feedback, not complete
    conformance.

- Adding another boolean to `tests::run(...)` is less elegant than a richer
  Stage 2 profile type.
  - Accepted because there are only two Stage 2 skip policies today:
    unavailable IETF vectors and quick release-preflight slow conformance.
    A profile type becomes worthwhile if a third policy appears.

- Report parsing may count skipped tests differently depending on cargo/libtest
  output.
  - Keep this as reporting detail. The functional requirement is that the
    smoke command avoids executing the slow test and does not report a false
    build failure.

## Rollback Plan

Revert the later implementation commit that changes
`/home/md/language/ropus/full-test/src/main.rs` and
`/home/md/language/ropus/full-test/src/tests.rs`.

Because the design adds no migration, data format, or public CLI flag, rollback
returns Stage 2 to the current single workspace test invocation. The smoke
command would again run the slow conformance extension test.
