# 2026-05-07 — PLN — Parameterized round-trip grid

Design doc for the new differential test grid that closes the four real
coverage gaps surfaced by 2026-05-07 reconnaissance. Drafted by the
planning agent; saved by the supervisor (the agent ran read-only).

## 1. Verification of recon claims

### Claim 1 — Bandwidth restriction (NB/MB/WB/SWB/FB) not parameterized

**Verdict: VERIFIED.**

- `tests/fuzz/fuzz_targets/fuzz_roundtrip.rs` parses `data[0..7]` into
  sr / ch / app / bitrate / cx / vbr / fec / dtx / loss / frame_duration
  only. No bandwidth byte. `CEncodeConfig`
  (`tests/fuzz/fuzz_targets/c_reference.rs:286-296`) has no `max_bandwidth`
  field. `apply_c_encoder_config` (`c_reference.rs:298-309`) issues 7 CTLs —
  none is `OPUS_SET_MAX_BANDWIDTH` / `OPUS_SET_BANDWIDTH`.
- `harness/tests/c_ref_differential.rs` uses `set_bandwidth` only inside
  redundancy / Hybrid-specific reproducers (lines 687, 887-888, 900-901,
  1044-1045, 1060-1061, 1217-1218, 1231-1232, 1283-1284, 1413). One single-
  shot `set_max_bandwidth(FULLBAND)` at line 1413. There is no parameterized
  cross-product test that walks all five bandwidth values × representative
  configs.
- `fuzz_ctl_sequence.rs:191-192,295` walks bandwidth in CTL sequences, but
  only to fuzz CTL ordering, not to lock down deterministic differential
  equivalence.

### Claim 2 — Forced modes (AUTO / SILK_ONLY / HYBRID / CELT_ONLY) lack a regression suite

**Verdict: VERIFIED.**

- `c_ref_differential.rs` calls `set_force_mode` 8 times (lines 626, 887,
  900, 1044, 1060, 1217, 1231, 1283), all inside specific bug reproducers
  (CELT↔SILK redundancy XOR, CVBR Hybrid floor, MB→WB switch). None is a
  parameterized AUTO / SILK_ONLY / HYBRID / CELT_ONLY × {sr, ch, br, cx}
  sweep.
- `OPUS_SET_FORCE_MODE_REQUEST=11002` is defined in
  `harness/src/bindings.rs:86`, not in
  `tests/fuzz/fuzz_targets/c_reference.rs`. Force mode is not plumbed into
  any fuzz target at all.
- `tests/conformance/tests/encode.rs` only references
  `OPUS_SET_FORCE_MODE` in a comment (line 5, 102) describing the C suite —
  no Rust regression.

### Claim 3 — Multistream > 2 streams not covered

**Verdict: VERIFIED (with one nuance).**

- `c_ref_differential.rs:1909-1976` is the only multistream differential.
  It uses `streams=1, coupled_streams=1` (one coupled stream → 2 channels)
  and tests decode only.
- `fuzz_multistream.rs:96-417` does cover `channels=1..=8` and
  `mapping_family ∈ {0, 1, 255, arbitrary}`, capping
  `streams + coupled_streams ≤ 16`. It is not a regression test — it's
  surround-only fuzz (`new_surround` constructor; line 281, 533) and lacks
  deterministic, named cells. There is no deterministic >2-stream
  round-trip differential anywhere in `harness/tests/` or
  `tests/conformance/`.
- `ropus/src/opus/multistream.rs` unit tests
  (`test_surround_encoder_familyN`, `test_validate_ambisonics`) are
  pure-Rust assertions, not C-vs-Rust differentials.

### Claim 4 — Complexity capped at 9 for `analysis.c` divergence

**Verdict: VERIFIED, asymmetric (Rust side missing analysis, not C side).**

- Cap is real: `fuzz_roundtrip.rs:130`
  (`complexity = (data[5] as i32) % 10`) with the comment at lines 126-129
  citing Campaign 9 (2026-04-19) and `analysis.c`.
- `wrk_journals/2026.05.01 - JRN - 24h-fuzz-campaign.md:80-90` confirms the
  cause: the C reference builds with `DISABLE_FLOAT_API=off` (since the
  Stage-7a removal in `harness/config.h`) and runs `analysis.c` for
  `complexity ≥ 10 ∧ sr ≥ 16000 ∧ app != RESTRICTED_LOWDELAY`. ropus does
  not run analysis on this path. Documented one-sided divergence, not a
  both-sides issue.
- The interaction with `RESTRICTED_LOWDELAY` is already an escape hatch
  in the analysis trigger condition (`app != RESTRICTED_LOWDELAY`),
  meaning RESTRICTED_LOWDELAY × cx 10 should not run analysis on either
  side and is therefore safe to test bit-exactly. The recon claim that
  RESTRICTED_LOWDELAY × cx 10 is a real coverage hole is correct.

## 2. Scope decision

| Claim | Scope built | Why |
|-------|-------------|-----|
| 1 Bandwidth | YES — full grid {NB, MB, WB, SWB, FB} via `set_max_bandwidth` | Plumbing missing, low effort to add. |
| 2 Forced mode | YES — {AUTO, SILK_ONLY, HYBRID, CELT_ONLY} | Plumb `OPUS_SET_FORCE_MODE_REQUEST=11002` into fuzz `c_reference.rs` and harness FFI. |
| 3 Multistream | YES — deterministic 3- and 4-stream cells, plus 5.1 (family 1, 6 ch) and FOA (family 2, 4 ch ambisonics) | Use `OpusMSEncoder::new_surround` symmetric on Rust + C, mirroring fuzz_multistream wiring but as `#[test]`s. |
| 4 cx10 × RESTRICTED_LOWDELAY | YES — bit-exact differential cell (analysis disabled on both sides for this app) | Targeted; closes the coverage hole the cap incidentally created. |

## 3. Grid design

### Test 1 — `roundtrip_bandwidth_grid_matches_c_reference`

- max_bandwidth ∈ {NB, MB, WB, SWB, FB} — 5 values
- (sr, ch) pinned per-bandwidth: NB→(8k, 1), MB→(12k, 1), WB→(16k, 2),
  SWB→(24k, 2), FB→(48k, 2)
- application: VOIP for NB/MB, AUDIO for WB/SWB/FB
- bitrate: 24 kbps mono / 48 kbps stereo
- complexity: 5; frame: 20 ms; vbr: true; vbr_constraint: false; fec: 0;
  dtx: 0
- fixture: `speech_48k_mono.wav` (resample on read for sub-48k); cells use
  10 frames of audio after a 60 ms silence prefix.

Oracle: bit-exact compressed bytes; PCM exact for CELT-only paths,
`SilkHybridCodedComparable` (SNR ≥ 50 dB) elsewhere — classify per cell
using `oracle::classify_decode_packet`.

Cells: 5. Runtime ~ 0.4 s/cell × 10 frames = ~2 s; total ~10 s.

### Test 2 — `roundtrip_force_mode_grid_matches_c_reference`

- force_mode ∈ {AUTO, SILK_ONLY, HYBRID, CELT_ONLY} — 4 values
- per-mode legal (sr, ch, frame): SILK_ONLY → (16k, 1, 20 ms);
  HYBRID → (24k, 2, 20 ms); CELT_ONLY → (48k, 2, 5 ms);
  AUTO → (48k, 2, 20 ms)
- bitrate: 32 kbps; complexity: 5; vbr: true; rest as in Test 1
- fixtures: SILK_ONLY/AUTO use `speech_48k_mono.wav`; HYBRID uses
  `music_48k_stereo.wav` (decimated for 24k); CELT_ONLY uses
  `music_48k_stereo.wav`. 10 frames each.

Oracle:
- AUTO + speech → SilkHybridCodedComparable (SNR ≥ 50 dB)
- SILK_ONLY → SilkHybridCodedComparable
- HYBRID → SilkHybridCodedComparable; compressed bytes still exact
- CELT_ONLY → CeltCodedComparable (full bit-exact)

Compressed bytes asserted exactly in all four cells.

Cells: 4. Runtime ~10 s total.

### Test 3 — `roundtrip_multistream_grid_matches_c_reference`

- (channels, mapping_family) cells: (3, 1) → 3.0 with 2+1 streams;
  (4, 1) → quad with 2+2; (6, 1) → 5.1 with 4+2 (4 streams, 2 coupled);
  (4, 2) → ambisonics FOA (W, X, Y, Z, 4 mono streams)
- sr: 48k; bitrate: 64 kbps × channels; complexity: 5; application: AUDIO;
  vbr: true; frame: 20 ms; 10 frames per cell
- fixture: synthetic per-channel patterned PCM (`patterned_pcm_i16` at
  `c_ref_differential.rs:99`) extended to N channels with channel-distinct
  seeds. Augment with a single ambisonic cell using
  `tests/vectors/ambisonic/ambisonic_order1_100ms.wav`.

Oracle:
- Compressed bytes exact (multistream is deterministic given matching
  configs).
- PCM: `oracle::classify_multistream_decode_packet` per packet. Expected
  `SilkHybridCodedComparable` for music-like input → SNR ≥ 50 dB.

Construction symmetry: both sides via `*_surround_encoder_create` to
ensure mapping/streams/coupled are identical (matches
`fuzz_multistream.rs:281,533`).

Cells: 4. Runtime ~3 s/cell. Total ~12 s.

### Test 4 — `roundtrip_cx10_restricted_lowdelay_matches_c_reference`

- complexity ∈ {9, 10}
- sr ∈ {16000, 48000} (covers analysis trigger boundary on the C side;
  both should disable analysis under RESTRICTED_LOWDELAY)
- ch: 2; app: RESTRICTED_LOWDELAY; bitrate: 64 kbps; frame: 5 ms; vbr: true
- fixtures: `48k_sweep.wav` and `speech_48k_mono.wav` (decimated to 16k)

Oracle: `CeltCodedComparable` (full bit-exact) — RESTRICTED_LOWDELAY
forces CELT_ONLY, no analysis path on either side.

Cells: 4. Runtime <2 s total.

Total runtime estimate: 4 sub-tests, ~17 cells, ~35–45 s wall clock.

## 4. File layout

- New file: `harness/tests/c_ref_differential_param_grid.rs` (sibling of
  `c_ref_differential.rs`).
- Reuses existing FFI symbols via re-declared minimal `extern "C"` block,
  or via shared `harness/tests/common/mod.rs` if implementer prefers
  (recommended later, not blocking).
- Adds `OPUS_SET_FORCE_MODE_REQUEST=11002` and
  `OPUS_SET_MAX_BANDWIDTH_REQUEST=4004` constants locally.
- Imports `ropus::opus::decoder::{MODE_*, OPUS_BANDWIDTH_*}` and
  `ropus::opus::multistream::{OpusMSEncoder, OpusMSDecoder}`.
- For the SNR oracle, copy the reusable fragments from
  `tests/fuzz/fuzz_targets/oracle.rs` via `#[path]` include — the same way
  `fuzz_roundtrip.rs:17-18` does — to keep one source of truth for
  `SILK_DECODE_MIN_SNR_DB=50`.
- Test functions: `test_bandwidth_grid_*`, `test_force_mode_grid_*`,
  `test_multistream_grid_*`, `test_restricted_lowdelay_cx10_grid_*` — one
  per cell so failures pinpoint immediately, plus a parameterized helper
  `run_cell(...)` to share setup / assertion code.
- Fixtures: speech_48k_mono.wav, music_48k_stereo.wav, 48k_sweep.wav,
  ambisonic/ambisonic_order1_100ms.wav. Read once via WAV parser identical
  to `c_ref_differential.rs:2787-2820`.

## 5. Failure-mode prediction

1. Multistream 5.1 / FOA — most fragile. Analysis-plumbing path through
   surround encoder is known partially-wired; expect either compressed-
   bytes drift on the (6, family 1) cell with high-band content, or silent
   SNR pass with a hidden Hybrid high-band drift. If failures appear here,
   look first at downmix scaling per stream and per-stream CTL fan-out
   (`OpusMSEncoder::set_bitrate`).
2. Hybrid force_mode @ 24k stereo — historical fragility per
   `c_ref_differential.rs:1283-1284` reproducer. Predict tier-2 SNR pass
   but exact-bytes asserts could expose a residual rng / redundancy XOR on
   first frame.
3. decode24 PLC drift — not directly exercised here, but if any cell hits
   a malformed packet via the SNR_PRECHECK_MIN_REF_ENERGY=1e7 path, the
   existing `snr_oracle_applicable_for_packet` gate already covers us.
4. cx10 × RESTRICTED_LOWDELAY — most likely PASS. If it fails, the
   failure is a high-value find (the cap is shown to be over-broad).
5. Bandwidth NB/MB at sr=48k — possible mismatch in band-truncation
   heuristics. The grid pins matching (sr, max_bw) pairs to avoid this;
   if drift appears even on matched pairs, that's a real bug.

## 6. Out of scope

- Full cross product of bandwidth × app × cx × br × frame —
  ~600 cells, runtime > 10 min. Replaced with orthogonal pinning.
- Stateful sequences (multi-frame state accumulation) — covered by
  `corpus_diff_webm_defer.rs` and `fuzz_encode_multiframe`.
- > 10 channels / mapping family 255 — already covered by
  `fuzz_multistream.rs`.
- decode_fec=1 paths — `fuzz_decode_plc_seq.rs` and the existing PLC
  differential cover them.
- Analysis cap removal on the Rust side — multi-week task (port
  `analysis.c`); we explicitly carve the one cell
  (RESTRICTED_LOWDELAY × cx10) where the cap is artificial.
- 24-bit (`decode24`) round-trip — covered by
  `c_ref_differential.rs:1505-1976`.
