# HLD: DTX and Activity Detection Fixes (Bugs 13-16)

## Problem

Opus-level DTX (Discontinuous Transmission) never activates for CELT-only frames, and has incorrect behavior for all modes. Four related bugs prevent the encoder from matching the C reference's silence/inactivity handling:

1. CELT-only frames never compute energy-based activity -- they pass `VAD_NO_DECISION` to the DTX decision function, which treats it as "active" (nonzero), so DTX never fires.
2. The `peak_signal_energy` field is missing entirely, so even if the activity computation were present, it would have no signal history to compare against.
3. The opus-level DTX guard condition (`use_dtx && !silk_mode.use_dtx`) is never true because Bug 9 (SILK DTX ignores `is_silence`) makes `silk_mode.use_dtx == use_dtx` unconditionally. Bug 9 must be fixed first.
4. When the DTX guard condition is false (the `else` branch), the C reference resets `nb_no_activity_ms_Q1` to zero. The Rust port has no `else` branch, so the inactivity counter accumulates stale values across mode switches.

These bugs interact: even after fixing Bug 9 so that `silk_mode.use_dtx` is correctly gated by `is_silence`, the activity value fed to `decide_dtx_mode` is wrong (Bug 13), because it depends on state that doesn't exist (Bug 14), and the counter is never reset when SILK handles DTX itself (Bug 16).

## Root Causes

### Bug 13: Missing CELT-only activity computation

**C** (opus_encoder.c:1926-1929): When the float analysis path is disabled and the mode is CELT-only, computes `noise_energy` via `compute_frame_energy()` and compares against `peak_signal_energy` with a boosted SNR threshold:

```c
} else if (st->mode == MODE_CELT_ONLY) {
    opus_val32 noise_energy = compute_frame_energy(pcm, frame_size, st->channels, st->arch);
    activity = st->peak_signal_energy < (QCONST16(PSEUDO_SNR_THRESHOLD, 0) * (opus_val64)HALF32(noise_energy));
}
```

This sets `activity = 1` (active) only when peak energy is *less than* threshold x noise energy -- i.e., when the signal has never been much louder than the current frame, it's considered noise/inactive.

**Rust** (encoder.rs:1685): Unconditionally sets `activity = VAD_NO_DECISION` for all non-silent frames, regardless of mode. The CELT-only branch does not exist.

### Bug 14: Missing peak_signal_energy field

**C** (opus_encoder.c:133): The `OpusEncoder` struct has `opus_val32 peak_signal_energy`.

**C** (opus_encoder.c:1310-1320): Updated every frame where the float analysis says the frame is active (or when float analysis is disabled):

```c
if (!is_silence) {
    st->peak_signal_energy = MAX32(
        MULT16_32_Q15(QCONST16(0.999f, 15), st->peak_signal_energy),
        compute_frame_energy(pcm, frame_size, st->channels, st->arch));
}
```

This is a leaky-max tracker: it decays by 0.1% per frame but ratchets up to any new peak. It lives in `opus_encode_native`, *before* the call to `encode_frame_native`, in the section that computes stereo width and bitrate.

**Rust**: The field does not exist in the `OpusEncoder` struct. No tracking occurs.

### Bug 15: Opus-level DTX guard never true

**C** (opus_encoder.c:1461-1463): `silk_mode.useDTX = st->use_dtx && !is_silence` (fixed-point path). When the frame is silent, SILK DTX is disabled so that the *opus-level* DTX can handle it instead.

**Rust** (encoder.rs:1277): `self.silk_mode.use_dtx = if self.use_dtx != 0 { 1 } else { 0 }` -- unconditionally copies `use_dtx`, ignoring `is_silence`.

This means `silk_mode.use_dtx` always equals `use_dtx`, so the guard `use_dtx && !silk_mode.use_dtx` at line 2223 is always false for SILK/hybrid modes. The opus-level DTX path is dead code. This is Bug 9, which belongs to the VOIP fixes group but is a prerequisite for this group.

### Bug 16: Missing nb_no_activity_ms_q1 reset

**C** (opus_encoder.c:2574-2575):

```c
} else {
    st->nb_no_activity_ms_Q1 = 0;
}
```

When SILK is handling DTX (the `else` of the DTX guard), C resets the opus-level inactivity counter. This prevents stale accumulation from a previous period of opus-level DTX.

**Rust** (encoder.rs:2222-2230): No `else` branch. The counter retains whatever value it had, which can cause incorrect DTX activation if the mode later switches to CELT-only (where opus-level DTX takes over).

## Fix Plan

### Fix 14: Add peak_signal_energy field

**Struct change**: Add `peak_signal_energy: i32` to `OpusEncoder` (after `nb_no_activity_ms_q1`). Initialize to 0 in `new()` and in `reset_state()`.

**Tracking**: In `opus_encode_native`, after computing `is_silence` and before computing stereo width (~line 1152-1160 area), add the peak energy tracking. Since mdopus does not have float analysis (`analysis_info` is not implemented), the guard simplifies to "always track when not silent":

- Call `compute_frame_energy(pcm, frame_size, channels)` to get the current frame energy.
- Update: `peak_signal_energy = max(mult16_32_q15(QCONST16_0_999, peak_signal_energy), frame_energy)`.

The fixed-point constant `QCONST16(0.999f, 15)` is `0.999 * 32768 = 32735`. `MULT16_32_Q15(a, b)` is `(a as i64 * b as i64) >> 15` truncated to i32.

### Fix 13: Port compute_frame_energy and CELT-only activity

**New function**: `compute_frame_energy(pcm: &[i16], frame_size: i32, channels: i32) -> i32`. Port the fixed-point version from C (opus_encoder.c:1080-1105):

- Find `sample_max` via `celt_maxabs` (or equivalent scan).
- Compute shift to prevent overflow: `shift = max(0, (celt_ilog2(1 + sample_max) << 1) + celt_ilog2(len) - 28)`.
- Accumulate `energy += (sample >> shift) * (sample >> shift)`.
- Normalize: `energy = (energy / len) << shift`.

**Activity branch**: In `encode_frame_native`, replace the flat `VAD_NO_DECISION` assignment with the full C logic:

- If `is_silence`: `activity = 0` (already correct).
- Else if `mode == MODE_CELT_ONLY`: compute `noise_energy`, compare against `peak_signal_energy` with `PSEUDO_SNR_THRESHOLD` (316 in fixed-point, since `QCONST16(316.23, 0)` truncates to 316). Use `HALF32(noise_energy)` (right-shift by 1) with the boosted comparison.
- Else: `activity = VAD_NO_DECISION` (SILK will override later via signal_type check at line 1929).

The constant `PSEUDO_SNR_THRESHOLD` corresponds to 10^(25/10) = 316.23, which in `QCONST16(..., 0)` is simply 316 as an integer.

### Fix 15: Gate silk_mode.use_dtx on is_silence (Bug 9 -- prerequisite)

This is Bug 9 from the VOIP group. Change encoder.rs line 1277 from:

`self.silk_mode.use_dtx = if self.use_dtx != 0 { 1 } else { 0 };`

to:

`self.silk_mode.use_dtx = if self.use_dtx != 0 && !is_silence { 1 } else { 0 };`

This requires `is_silence` to be available at that point in `opus_encode_native`. It already is -- `is_silence` is computed at line 1152, and the SILK DTX assignment is at line 1277.

After this fix, when `is_silence` is true, `silk_mode.use_dtx` becomes 0, making the guard `use_dtx && !silk_mode.use_dtx` true, and the opus-level DTX path activates.

### Fix 16: Add else branch for nb_no_activity_ms_q1 reset

At encoder.rs line 2230 (after the closing brace of the DTX `if` block), add the `else` branch:

```
} else {
    self.nb_no_activity_ms_q1 = 0;
}
```

## Execution Order

1. **Bug 9 (prerequisite)**: Fix `silk_mode.use_dtx` gating. This unblocks the opus-level DTX path for SILK/hybrid modes during silence. Without this, Bugs 15/16 are moot -- the code path is unreachable.

2. **Bug 14**: Add `peak_signal_energy` field and tracking. This must exist before Bug 13 can use it.

3. **Bug 13**: Port `compute_frame_energy` and the CELT-only activity branch. Depends on Bug 14 for the peak energy value.

4. **Bug 16**: Add the `else` reset. Independent of 13/14 but logically last since the DTX path must be reachable (Bug 9) for it to matter.

## Verification Plan

### What to check

After all four fixes, DTX behavior should match the C reference in these scenarios:

1. **SILK-only + silence**: When `is_silence` is true, `silk_mode.use_dtx` should be 0. The opus-level DTX path should activate. `decide_dtx_mode` should receive `activity = 0`, accumulate `nb_no_activity_ms_q1`, and return true after `NB_SPEECH_FRAMES_BEFORE_DTX` (10) frames of 20ms = 200ms of silence. The encoder should return 1-byte packets (TOC only, `range_final = 0`).

2. **CELT-only + quiet frames**: For CELT-only mode with DTX enabled, the activity computation should use `compute_frame_energy` vs `peak_signal_energy`. Frames whose energy is close to the noise floor (peak < threshold x noise) should be classified as inactive, feeding `activity = 0` into `decide_dtx_mode`.

3. **Mode transitions**: When switching from SILK (where SILK handles DTX) to CELT-only (where opus handles DTX), the `else` branch should have reset `nb_no_activity_ms_q1` to 0, preventing false DTX activation from stale counter values.

4. **Counter saturation**: After `MAX_CONSECUTIVE_DTX` (20) frames of DTX, the counter should wrap back to the threshold, forcing a non-DTX frame. This is already implemented in `decide_dtx_mode` but can only be exercised once the path is reachable.

### Test signals

- **Encode a file with extended silence**: Expect DTX packets (1 byte) during silence, normal packets during speech. Compare packet sizes byte-for-byte with C reference output.
- **CELT-only at low bitrate with quiet passages**: Verify DTX activates when energy drops, matching C reference packet boundaries.
- **Mode-switching signal** (e.g., speech that triggers SILK, then tone that triggers CELT): Verify no spurious DTX at the transition point.

### Observable output

With DTX working correctly, the test harness should show:
- Encoded byte counts matching between Rust and C for DTX-enabled test vectors.
- DTX packets (1 byte) appearing at the same frame positions in both outputs.
- `range_final == 0` for DTX frames in both implementations.
