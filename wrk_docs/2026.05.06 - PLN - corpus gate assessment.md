# Corpus Gate Assessment

Date: 2026.05.06

Scope: documentation-only assessment of the real-world corpus and
`corpus_diff` situation after commit `85c510a Add DRED DNN release preflight
gate`. No code or assets were changed by this assessment.

## Testing Principle

The next work should keep making "green" machine-checkable. A corpus lane is
only useful if the report distinguishes:

- no corpus was present,
- corpus files were present but all skipped,
- only reference-encoded baseline files were tested,
- non-reference/generated files were tested,
- exact decoder PCM parity actually ran.

The smallest useful step is therefore a corpus-gate contract, not a broad media
curation pass.

## 1. Current Infrastructure

`/home/md/language/ropus/harness/src/bin/corpus_diff.rs` is a wrapper binary for
the real implementation in
`/home/md/language/ropus/harness/src/bin_inner/corpus_diff.rs`. When the C
reference is unavailable it exits `2`; otherwise it:

- scans one directory non-recursively;
- admits files ending in `.opus`, `.ogg`, or `.webm`;
- parses Ogg Opus pages directly, without `libopusfile`;
- supports channel mapping family 0 only, so mono/stereo Ogg Opus is the active
  lane;
- decodes each audio packet through ropus and the C reference;
- asserts exact sample-for-sample PCM equality per packet;
- catches per-file panics and continues through the directory;
- exits `2` when the directory has no candidate extensions.

The corpus location exists at
`/home/md/language/ropus/tests/vectors/real_world`, but the checked-in tree
currently contains only
`/home/md/language/ropus/tests/vectors/real_world/README.md`. The media
extensions are gitignored in `/home/md/language/ropus/.gitignore`, which avoids
accidental large or rights-ambiguous commits.

`/home/md/language/ropus/tools/fetch_corpus.sh` is a baseline-population
script. It intentionally documents that its listed sources are
reference/libopus-encoded sanity samples, not genuine non-reference encoder
diversity. Its current `SOURCES[]` entries are pinned as `TBD`, so the script
refuses to admit them until checksums are deliberately pinned.

`/home/md/language/ropus/full-test` does not currently run `corpus_diff`, does
not expose a real-world corpus stage in JSON/HTML, and does not include a
`real_world_corpus` preflight key. A passing full-test or non-quick
`--release-preflight` therefore does not claim real-world corpus coverage.

The fuzz README points to the real-world corpus as a possible source of packet
seeds, but that is manual seeding guidance. It is not a release gate.

## 2. False-Green And Empty-Coverage Risks

The main risk is not that `corpus_diff` is weak. For supported Ogg family-0
files, its oracle is strong exact PCM parity. The risk is that the lane is easy
to confuse with coverage it did not provide.

Remaining risks:

- `full-test` green says nothing about the real-world corpus because
  `corpus_diff` is not integrated.
- The checked-in corpus is empty apart from the README.
- A local developer may have private gitignored corpus files while CI or another
  agent has none, producing inconsistent confidence.
- `tools/fetch_corpus.sh` currently refuses every listed source because all
  checksums are `TBD`; this is safe, but means the baseline fetch path does not
  currently populate a usable corpus.
- The fetch-script baseline is reference/libopus encoded. It is useful, but it
  must not be marketed as non-reference real-world diversity.
- `.webm` files are admitted by extension but always skipped because Matroska
  parsing is out of scope.
- Unsupported channel mappings and malformed files are skipped. If a directory
  contains only skipped candidates, `corpus_diff` can still finish with no
  mismatches or panics.
- A malformed file where both decoders reject at the same point can be reported
  as matched up to that point. That is reasonable for exploratory corpus runs
  but too weak for a required fixture manifest.
- The Ogg parser only uses the `OpusHead` fields needed for decoder setup. It
  does not assert container semantics such as pre-skip trimming, output gain,
  tags, seeking, granule gaps, or WebM behavior.

## 3. Recommended Smallest Next Slice

Implement a narrow slice called:

```text
Corpus gate contract and generated non-reference smoke
```

Do not start by checking in third-party audio, messaging samples, YouTube
extracts, or WebRTC captures. That creates licensing and provenance decisions
before the gate semantics are good enough to trust.

The slice should do three things.

First, harden `corpus_diff` result semantics:

- return non-green when zero supported files actually produce decoded audio;
- distinguish candidate count, supported-run count, matched count, skipped
  count, mismatched count, panic count, and zero-audio/matched-error cases;
- keep empty-candidate exit `2`;
- make skipped-only and WebM-only directories non-green for corpus claims.

Second, expose the corpus lane in `full-test` as a visible claim:

- default and quick profiles should report corpus status but not claim corpus
  coverage;
- non-quick release-preflight should either require the corpus gate or state
  explicitly that real-world corpus coverage is not claimed;
- JSON/HTML should show the corpus status and the exact command/result, not
  hide it under the general tests stage.

Third, add one asset-safe non-reference smoke path without committing media:

- generate a temporary Ogg Opus file from an existing repo WAV fixture using
  FFmpeg's native `opus` encoder (`-c:a opus -strict -2`), when available;
- run `corpus_diff` on that temporary directory;
- treat missing FFmpeg/native-opus support as a clear missing-tool status, not
  a pass;
- keep curated external corpora as a later, explicit asset/provenance decision.

This is not a complete "real-world corpus" solution. It is a small correctness
signal that proves ropus can decode at least one non-reference-encoded Ogg Opus
stream exactly against the C reference, while making empty/skipped corpus lanes
impossible to overstate.

```bash

timeout 120 cargo test -p full-test corpus

timeout 120 cargo test -p full-test html
```

- empty `tests/vectors/real_world` still exits `2`;
- WebM-only or skipped-only directories are non-green for a corpus claim;
- at least one supported Ogg family-0 file must decode nonzero audio to satisfy
  the gate;

  was claimed;

  pass;

  `corpus_diff`.

If the implementation adds a shell helper, also run:

```bash

timeout 30 bash -n tools/fetch_corpus.sh
```

## Recommendation

Proceed with the corpus gate contract and generated non-reference smoke slice.
It improves correctness signal without importing legally ambiguous assets, and
it leaves the later curated real-world corpus decision clean: once Arthur is
ready to approve specific external fixtures, the gate will already have honest
pass/fail/report semantics.
