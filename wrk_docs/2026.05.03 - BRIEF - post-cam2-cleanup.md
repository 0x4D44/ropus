## Scope

### Task 1 (HIGH) — Fix `multiframe-vbr-fec-dtx-divergence`

This is a real codec bug currently masked by a fuzzer skip filter, so
every fuzz hour from now on pays a coverage tax until it lands.

**Repros (6 files):**
`tests/fuzz/known_failures/multiframe-vbr-fec-dtx-divergence/crash-*`.
README in that directory has the full repro recipe and a hypothesis.

**Representative case** (per README):
- `sr=8000, ch=2, application=VOIP, bitrate≈59713, complexity=0`
- `vbr=1, fec=2, dtx=1, loss_perc=7`
- 2-frame stream; frame 1 Rust=164B vs C=181B → bytes diverge.

**Suspect path** (per README):
> Rust's encoder rate-control state caches differently from C's when
> `set_inband_fec` and `set_dtx` are toggled between frames in VBR
> mode. Suspect derived field: `rangeFinal`, `bitsFinal`, or
> `silk_mode.bitRate` recomputation re-derived on a different schedule
> between Rust and C.

**Currently masked by:**
`tests/fuzz/fuzz_targets/fuzz_encode_multiframe.rs:79-81` —
`is_known_class` returns true for `sr=8000 && ch=2`, suppressing the
differential assertion. Removing this filter must be part of the
landing.

**Existing instrumentation to reuse:**
- `harness/src/bin_inner/interframe.rs` — frame-by-frame encoder state
  comparison driver.
- `harness/debug_helper.c` — `debug_get_silk_*` FFI accessors for
  C-side state extraction.
- The encoder-state-accumulation HLD
  (`wrk_docs/2026.05.02 - HLD - encoder-state-accumulation-fix.md`)
  documents the methodology used for the now-closed cross-frame class
  — same shape applies here.

**Definition of done:**
1. All 6 crash files in `known_failures/multiframe-vbr-fec-dtx-
   divergence/` produce byte-exact output between Rust and C across
   all frames.
2. `is_known_class` filter in `fuzz_encode_multiframe.rs:79-81`
   removed (or narrowed to genuinely-tolerated tuples with a comment
   explaining why).
3. 6 repros migrated to `tests/fuzz/seeds/fuzz_encode_multiframe/`
   as `regression-vbr-fec-dtx-{slug}.bin` per the pattern in
   `5d4bfd2`/`ecf2237`. `known_failures/multiframe-vbr-fec-dtx-
   divergence/` directory removed.
4. 1-hour smoke run: `cargo +nightly fuzz run --fuzz-dir tests/fuzz
   fuzz_encode_multiframe -- -jobs=4 -workers=4 -max_total_time=3600`
   produces **zero new artifacts**.
5. Existing test suites still pass: `cargo test --release -p ropus`,
   `-p ropus-harness`, `-p conformance`.
6. `cargo llvm-cov` line/branch coverage on
   `ropus/src/opus/encoder.rs`, `ropus/src/silk/encoder.rs`,
   `ropus/src/silk/common.rs` does not regress.

### Task 2 (MED) — Triage the 3 decode-side asymmetries

These are tolerated rather than fixed. Each needs one of three
verdicts: **fix**, **document as Rust-stricter-by-design** (with a
test that locks in the asymmetry), or **file a tracking issue and
defer** (with a rationale for why it's safe).

**Targets:**

| Dir | Tolerance code | Suggested verdict (validate) |
|---|---|---|
| `multistream-decode-recovery-divergence` | `fuzz_multistream.rs:455-465` (sample-count parity only) | Likely **defer** — the README in `silk_decode_recovery_divergence_loud/` indicates this is the same SILK recovery class gated on a separate fix. Confirm and link. |
| `multistream-decode-family-asymmetry` | `fuzz_multistream.rs:480-488` ((Err Rust, Ok C) tolerated for family ≥ 1) | Investigate: Rust over-rejecting at family=1 (ch=3, sr=48000) — usability bug, security-safe. Decide fix vs document. |
| `multistream-family255-error-asymmetry` | `fuzz_multistream.rs:475-478` ((Ok Rust, Err C) tolerated at family=255) | Likely **document** — family=255 has no normative constraints, so accepting more is defensible. Lock with a regression test. |

**Definition of done per dir:**
- Verdict recorded in the dir's README (or in `wrk_journals/`).
- If **fix**: repros migrated to regression seeds, tolerance carve-out
  removed.
- If **document**: regression test added that locks in the
  asymmetry; tolerance code annotated with the journal/commit
  reference; dir removed (the asymmetry is no longer a "failure").
- If **defer**: tracking note added to the dir's README pointing at
  the gating fix; tolerance code unchanged.
