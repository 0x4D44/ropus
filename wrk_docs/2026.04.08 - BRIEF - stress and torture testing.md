# Brief: Stress and Torture Testing

Date: 2026-04-08

## Context

The harness has `longsoak` and `test-all` commands that exercise encode/decode over extended periods. A 3-second local soak passed (PCM matched, final-range matched), and `test-all` runs a 5-second soak. The integration matrix covers 35 WAV files x 4 bitrates x 2 modes = 280 cases, all passing.

However, there is no evidence of sustained stress testing — multi-hour soaks, adversarial configurations, memory-growth monitoring, or systematic torture of the configuration space. The current testing is broad but shallow in duration and narrow in configuration variety.

## Objective

Build a torture testing program that exercises the codec under sustained, varied, and adversarial conditions to surface state-accumulation bugs, memory issues, and rare mode-transition failures.

## Scope

### 1. Extended soak testing

Expand the `longsoak` command or create a new `torture` command that:

- Runs for configurable duration (target: 1 hour default, overnight option)
- Randomly varies parameters every N frames:
  - Bitrate: sweep 6000-510000
  - Bandwidth: NB/MB/WB/SWB/FB
  - Frame size: 2.5/5/10/20/40/60ms
  - Application: VOIP/AUDIO/RESTRICTED_LOWDELAY
  - Complexity: 0-10
  - Channels: mono/stereo switching (via multistream)
- Compares C and Rust outputs after every frame
- Reports first divergence with full state dump (frame number, configuration, PCM delta, final range values)

The key insight: the current soak uses a single fixed configuration. Real bugs hide in transitions — switching from SILK to CELT mid-stream, changing bandwidth, switching CBR/VBR.

### 2. Mode transition torture

Create targeted tests for the riskiest transitions identified in the testing report:

- **SILK <-> CELT mode switches**: Encode in VOIP mode at low bitrate (forces SILK), then switch to AUDIO at high bitrate (forces CELT), and back. Verify PCM parity at every transition boundary.
- **Bandwidth transitions**: NB -> FB -> NB within a single stream. These exercise the resampling and band extension paths.
- **FEC/PLC sequences**: Encode normally, then decode with simulated packet loss (every Nth packet dropped), compare PLC output C vs Rust. Vary loss rates: 1%, 5%, 10%, 30%.
- **DTX transitions**: Enable DTX, feed silence/speech/silence patterns, verify DTX frames are identical and re-activation matches.
- **Redundancy paths**: Force hybrid mode with redundancy coding, verify SILK-to-CELT and CELT-to-SILK redundancy frames match.
- **VBR bitrate variation**: Encode with VBR across a signal that transitions from silence to loud complex audio and back. Verify packet sizes and decoded PCM match.

### 3. Bitrate sweep matrix

Extend the integration corpus (`tools/integrate.py`) to cover:

Current: 4 bitrates (16k, 32k, 64k, 128k)
Proposed additions:
- Extreme low: 6000, 8000, 10000
- Mid: 24000, 48000, 96000
- Extreme high: 256000, 320000, 510000

These exercise different code paths — very low bitrates stress the PLC and SILK paths, very high bitrates exercise CELT at full resolution. The integration script already handles the infrastructure; it just needs more bitrate entries.

### 4. Sample rate coverage

Current vectors cover 12/16/24/48 kHz. Add:
- 8000 Hz (narrowband, exercises maximum resampling)

This is important because 8 kHz is common in telephony and exercises the longest resampling chain.

### 5. Memory and resource monitoring

Add optional monitoring to the soak command:
- Track peak RSS over time (detect leaks)
- Verify no unbounded allocations (the codec should have bounded memory after initialization)
- Report allocation count per frame if feasible

This doesn't need to be sophisticated — even a simple "RSS at start vs RSS after 10 minutes" comparison would catch gross leaks.

### 6. Automation script

Create `tools/torture_test.sh` that:
- Runs the extended soak (default 30 minutes)
- Runs the mode transition tests
- Runs the full bitrate sweep matrix
- Reports pass/fail summary
- Can be invoked as `bash tools/torture_test.sh [--duration 60]`

## Acceptance criteria

- Extended soak runs for 1 hour without divergence on at least 3 different configurations
- Mode transition tests cover SILK<->CELT, bandwidth changes, FEC/PLC, DTX
- Bitrate sweep covers 6000-510000 range
- 8 kHz sample rate added to test vectors
- `tools/torture_test.sh` runs the full suite and reports results

## Out of scope

- Performance regression detection (covered by bench infrastructure)
- Formal memory profiling tools (basic RSS monitoring is sufficient)
- Concurrent/multi-threaded stress testing (codec is single-threaded by design)
