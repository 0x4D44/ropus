# HLD: Fuzz coverage expansion — closing the 10 gaps (V2)

**Date:** 2026-05-01
**Author:** Claude (with Arthur)
**Scope target:** `tests/fuzz/fuzz_targets/*.rs`, `tests/fuzz/Cargo.toml`,
`tests/fuzz/fuzz_targets/c_reference.rs`, `tests/fuzz/seeds/<target>/`,

**Supersedes:** V1 (`2026.04.30 - HLD - fuzz-coverage-expansion.md`).

## Changelog from V1

V1 self-review found 9 substantive issues. V2 incorporates all of them:

- **Bug fix — operator precedence**: V1's FEC byte-extraction snippet
  (`data[6] & 0b1100 >> 2`) parses as `data[6] & (0b1100 >> 2)` =
  `data[6] & 0b11`. Fixed with explicit parens throughout
  (`(data[6] & 0b1100) >> 2`).
- **Bug fix — `Arbitrary` derive feature**: V1 claimed "no new crate
  dependency required". `#[derive(Arbitrary)]` requires the derive
  feature, gated behind `libfuzzer-sys`'s `arbitrary-derive` feature.
  Cargo.toml gets `libfuzzer-sys = { version = "0.4", features =
  ["arbitrary-derive"] }`.
- **Bug fix — `MultiframeInput::pcm` size bound**: V1's
  "capped via take_rest pattern" was wrong (`take_rest` consumes all
  remaining bytes, leaving none for subsequent frames). V2 specifies
  `int_in_range` for both frame count and per-frame PCM length.
- **Sizing correction**: `c_reference.rs` extension is ~+250 lines
  (6 extern decls + 4 wrappers mirroring `c_encode`/`c_decode`),
  total ~600. V1 said ~+120, total ~470.
- **New surface — float path (gap 11)**: `OpusEncoder::encode_float`
  (encoder.rs:1540) and `OpusDecoder::decode_float` (decoder.rs:1380)
  are public surfaces missed in V1. Added as a one-bit input switch
  on the existing encode/decode targets.
- **New surface — decoder runtime setters (gap 12)**:
  `OpusDecoder::set_gain`, `set_complexity`, `reset()` are mutation
  surfaces with validation that V1 missed. Added to
  `fuzz_decode_safety` as a small pre-decode shuffle.
- **New surface — multistream runtime setters (gap 13)**:
  `fuzz_multistream` exercises `OpusMSEncoder::set_bitrate`,
  `set_complexity`, `set_vbr`, etc. between construction and first
  encode. V1 only specced encode/decode/roundtrip.
- **Confidence calibration**: V1's Open Question Q1 ("is byte-exact
  the right oracle for VBR?") asserted with confidence in V2.
  Codec is deterministic across instances; differential oracle holds.
- **Doc accuracy**: Line numbers corrected (parse-validation at
  `fuzz_decode.rs:154`, not 147; SNR-replacement target is the
  `if celt_only` block at `fuzz_decode.rs:125`); existing
  `[[bin]]` style is `doc = false` only (no `test = false`).

## Context

The 8-hour overnight fuzz campaign on 2026-04-29 ran ~24.4 B executions
across 9 targets and found 0 crashes / 0 panics
(`wrk_journals/2026.04.29 - JRN - overnight-fuzz-campaign.md`).
A follow-up audit catalogued 10 substantive coverage gaps; V2's
self-review added 3 more (float path, decoder setters, multistream
setters). 13 gaps total. This HLD designs the smallest set of changes
that closes all of them in a sandbox environment with no migration
constraints.

## Goal

Bring the fuzz suite into honest alignment with the codec's production
configuration space. Gaps:

1. VBR — currently every target forces `set_vbr(0)`.
2. Multistream encode/decode/roundtrip — zero coverage today.
3. FEC, DTX, packet-loss-perc — never enabled in any fuzz target.
4. Encoder runtime setters (new in 0.12.0) — validation logic unfuzzed.
5. Repacketizer beyond a single `cat` + `out`.
6. SILK / Hybrid decode oracle — currently panic-only (PCM compare skipped).
7. DNN weight-blob ingestion — `set_dnn_blob(&[u8])` is an unfuzzed parser.
8. Saturated targets — `fuzz_packet_parse` (cov 61) and
   `fuzz_encode_multiframe` (cov 36) are CPU sinks.
9. Mutator strategy — every target uses raw-byte modulo extraction;
   no seed corpus of pathological PCM patterns.
10. Hybrid-mode SILK↔CELT switchover — emerges naturally once gap 1
    (VBR) is closed; no separate target needed.
11. **Float-path API**: `encode_float` / `decode_float` are public,
    used, and entirely unfuzzed.
12. **Decoder runtime setters**: `set_gain`, `set_complexity`, `reset()`.
13. **Multistream runtime setters**: per-stream `set_bitrate`,
    `set_complexity`, `set_vbr`, etc.

## Non-goals

- No CI integration. Campaigns remain manual via
  `tools/overnight_fuzz_launch.sh`.
- No coverage-guided corpus-minimisation tooling.
- No custom libFuzzer mutators. We use `Arbitrary` derive where
  structured inputs help, raw bytes elsewhere.
- No fuzz target for analysis, soft-clip, or projection encoders.
- No cross-target panic-capture deduplication. The 35-line panic-hook
  block is duplicated in every target; refactoring is churn with no
  fuzz value.
- No new env vars. The existing `FUZZ_PANIC_CAPTURE_DIR` (set by
  `overnight_fuzz_launch.sh`) is the only one.
- No feature flags. New behaviour is unconditional; saturated targets
  are deleted, not hidden behind a cfg.

## Design

The work decomposes into four streams. Each stream is independently
mergeable and independently runnable in a fuzz campaign.

### Stream A — Extend existing encode/decode targets in place (gaps 1, 3, 4, 10, 11, 12)

Four encode-path targets share a common structured-input prologue.
V2 extends the prologue to consume **two** more config bytes and adds
a **one-bit** PCM-format switch:

```rust
// new total = 8 config bytes
sample_rate    = SAMPLE_RATES[data[0] % 5]
channels       = if data[1] & 1 == 0 { 1 } else { 2 }
application    = APPLICATIONS[data[2] % 3]
bitrate        = 6000 + u16::from_le_bytes([d[3], d[4]]) % 504_001
complexity     = data[5] % 11
// data[6] holds VBR + FEC config; explicit parens — `>>` binds tighter than `&`
vbr            = (data[6] & 0b0001) != 0
vbr_constraint = (data[6] & 0b0010) != 0
inband_fec     = ((data[6] & 0b1100) >> 2) as i32      // 0..=3 → wraps to 0=Off, 1=On, 2=Forced, 3=Off
dtx            = (data[7] & 0b0001) != 0
loss_perc      = ((data[7] & 0b1111_1110) >> 1) as i32 % 101  // 0..=100
use_float_pcm  = (data[1] & 0b0010) != 0               // formerly unused channels-bit-1
pcm_bytes      = &data[8..]
```

Note: `inband_fec == 3` clamps to `0` (Off) so we hit each of the three
documented FEC values with non-trivial probability rather than wasting
a value on noise. `set_inband_fec` rejects values outside `0..=2` with
`OPUS_BAD_ARG`, so feeding `3` raw would be tested by the safety
target's separate setter shuffle (below); the differential prologue
keeps inputs valid.

After construction, replace the unconditional `set_vbr(0)` with:

```rust
enc.set_vbr(if vbr { 1 } else { 0 });
enc.set_vbr_constraint(if vbr_constraint { 1 } else { 0 });
enc.set_inband_fec(inband_fec);
enc.set_dtx(if dtx { 1 } else { 0 });
enc.set_packet_loss_perc(loss_perc);
```

This closes gaps 1 (VBR), 3 (FEC/DTX/loss-perc), and 10 (the hybrid
SILK↔CELT switchover that VBR + bitrate randomization naturally
exercises).

#### Float-path branch (gap 11)

When `use_float_pcm` is set, the target converts the raw input bytes
to `f32` instead of `i16`, calls `encode_float()` instead of
`encode()`, and adds a parallel C-side wrapper `c_encode_float`:

```rust
if use_float_pcm {
    let pcm: Vec<f32> = pcm_bytes
        .chunks_exact(4)
        .take(samples_needed)
        .map(|c| f32::from_le_bytes([c[0], c[1], c[2], c[3]]))
        .collect();
    if pcm.len() < samples_needed { return; }
    let rust_ret = rust_enc.encode_float(&pcm, frame_size, &mut rust_out, 4000);
    let c_ret    = c_reference::c_encode_float(&pcm, frame_size, sample_rate, channels, ...);
    // existing differential / parseable / determinism oracles unchanged
} else {
    // existing i16 path
}
```

Decoder side: `data[1] & 0b0010` toggles between `decode()` (i16) and
`decode_float()` (f32) in `fuzz_decode.rs` /
`fuzz_decode_safety.rs` / `fuzz_roundtrip.rs` /
`fuzz_roundtrip_safety.rs`.

PCM payload size doubles in float mode (4 bytes/sample vs 2). The
length check at the prologue accounts for this.

#### CBR-only assertions become VBR-aware

The CBR-determinism block at `fuzz_encode.rs:163-182` only holds when
`vbr == false`. Wrap it in `if !vbr { ... }`. The differential oracle
(byte-exact compare against C reference) holds independently of VBR
mode — codec is deterministic across instances; the existing oracle
was never relying on CBR for correctness, only for the secondary
re-encode-and-compare check.

#### Random runtime-setter sequence (gap 4, 12)

After construction, before the first `encode()` / `decode()` call,
replay a short deterministic sequence of setter calls driven by the
input bytes. **Safety-target only** — the differential targets keep a
fixed configuration so C-vs-Rust comparison stays apples-to-apples.

For `fuzz_encode_safety.rs` and `fuzz_roundtrip_safety.rs`:

```rust
fn apply_encoder_setter_sequence(enc: &mut OpusEncoder, bytes: &[u8]) {
    for chunk in bytes.chunks_exact(2).take(16) {
        match chunk[0] % 11 {
            0  => { enc.set_complexity((chunk[1] % 11) as i32); }
            1  => { enc.set_signal(SIGNAL_VALUES[chunk[1] as usize % 3]); }
            2  => { enc.set_force_channels(FORCE_CH[chunk[1] as usize % 3]); }
            3  => { enc.set_max_bandwidth(BW[chunk[1] as usize % 5]); }
            4  => { enc.set_packet_loss_perc((chunk[1] % 101) as i32); }
            5  => { enc.set_inband_fec((chunk[1] % 3) as i32); }
            6  => { enc.set_dtx((chunk[1] & 1) as i32); }
            7  => { enc.set_lsb_depth(8 + (chunk[1] % 17) as i32); }
            8  => { enc.set_phase_inversion_disabled((chunk[1] & 1) as i32); }
            9  => { enc.set_prediction_disabled((chunk[1] & 1) as i32); }
            10 => { enc.set_bitrate(byte_to_bitrate(chunk[1], chunk[1])); }
            _  => unreachable!(),
        }
    }
}
```

**Setters 7-9 are on the V3 runtime-setters HLD's "deliberately
excluded from the high-level facade" list** (`set_lsb_depth`,
`set_phase_inversion_disabled`, `set_prediction_disabled`). They are
still legal on `OpusEncoder` low-level — which the fuzzer drives
directly — and shipping them through fuzz coverage is *intentional*
safety testing of the underlying CTL surface, not an endorsement of
mid-stream toggling for production callers. The runtime-setters HLD
exclusion stands.

For `fuzz_decode_safety.rs` (gap 12):

```rust
fn apply_decoder_setter_sequence(dec: &mut OpusDecoder, bytes: &[u8]) {
    for chunk in bytes.chunks_exact(2).take(8) {
        match chunk[0] % 4 {
            0 => { let _ = dec.set_gain(i16::from_le_bytes([chunk[1], 0]) as i32); }
            1 => { let _ = dec.set_complexity((chunk[1] % 11) as i32); }
            2 => { let _ = dec.set_phase_inversion_disabled((chunk[1] & 1) as i32); }
            3 => { dec.reset(); }
            _ => unreachable!(),
        }
    }
}
```

Capped at 8 setter calls (16 bytes). `reset()` mid-shuffle is the most
interesting case — it tests that subsequent `decode()` calls survive a
state reset.

#### Rebuild `fuzz_encode_multiframe` around `Arbitrary`

Current target ran 5.7 B execs and added 2 corpus entries — its
mutator can't escape its own input shape. Replace with `Arbitrary`
with explicit bounds:

```rust
use libfuzzer_sys::arbitrary::{self, Arbitrary, Unstructured};

#[derive(Debug)]
struct MultiframeInput {
    sample_rate_idx: u8,
    channels_minus_one: u8,
    application_idx: u8,
    frames: Vec<FrameConfig>, // bounded 2..=16 in custom Arbitrary impl
}

#[derive(Debug)]
struct FrameConfig {
    bitrate_raw: u16,
    complexity: u8,
    vbr: bool,
    fec: u8,
    dtx: bool,
    loss_perc: u8,
    pcm_bytes: Vec<u8>, // bounded to frame_size * channels * 2 in custom Arbitrary impl
}

impl<'a> Arbitrary<'a> for MultiframeInput {
    fn arbitrary(u: &mut Unstructured<'a>) -> arbitrary::Result<Self> {
        let sample_rate_idx = u.int_in_range(0..=4)?;
        let channels_minus_one = u.int_in_range(0..=1)?;
        let application_idx = u.int_in_range(0..=2)?;
        let n_frames = u.int_in_range(2..=16)?;
        let mut frames = Vec::with_capacity(n_frames);
        for _ in 0..n_frames {
            frames.push(FrameConfig::arbitrary_with_sr(u, sample_rate_idx, channels_minus_one)?);
        }
        Ok(Self { sample_rate_idx, channels_minus_one, application_idx, frames })
    }
}

impl FrameConfig {
    fn arbitrary_with_sr<'a>(
        u: &mut Unstructured<'a>,
        sr_idx: u8,
        ch_minus_one: u8,
    ) -> arbitrary::Result<Self> {
        let sample_rate = SAMPLE_RATES[sr_idx as usize];
        let channels = ch_minus_one as i32 + 1;
        let frame_size = (sample_rate / 50) as usize;
        let pcm_bytes_needed = frame_size * channels as usize * 2;
        Ok(Self {
            bitrate_raw: u.arbitrary()?,
            complexity:  u.int_in_range(0..=10)?,
            vbr:         u.arbitrary()?,
            fec:         u.int_in_range(0..=2)?,
            dtx:         u.arbitrary()?,
            loss_perc:   u.int_in_range(0..=100)?,
            pcm_bytes:   u.bytes(pcm_bytes_needed)?.to_vec(),
        })
    }
}
```

Each `FrameConfig` is applied via setter calls before its `encode()`,
exercising intra-stream config drift (adaptive bitrate, FEC toggling
on push-to-talk) — the realistic shape modulo-byte structuring can't
reach. `frames.len()` clamped to 16 keeps each iteration under ~200 ms.

### Stream C — Three new targets (gaps 2, 5, 7, 13)

#### `fuzz_multistream` (gaps 2, 13)

Combines encode + decode + roundtrip in one target, switched on a
config byte. Restricted to `OpusMSEncoder::new_surround(channels,
mapping_family)` — arbitrary stream/coupled/mapping triples saturate
the validator, not the codec. **Includes a per-stream setter shuffle**
(gap 13) between construction and first `encode`, mirroring Stream A's
shuffle but targeting `OpusMSEncoder`'s own setters
(`set_bitrate`, `set_complexity`, `set_vbr`, `set_inband_fec`,
`set_dtx`, `set_packet_loss_perc`, `set_lfe`).

```rust
#[derive(Arbitrary)]
struct MSInput {
    op: u8,                    // 0=encode, 1=decode, 2=roundtrip
    sample_rate_idx: u8,       // index into [8000,12000,16000,24000,48000]
    channels: u8,              // 1..=8 (clamped)
    mapping_family: u8,        // {0, 1, 2, 255} (clamped via match)
    application_idx: u8,
    bitrate_raw: u16,
    complexity: u8,
    vbr: bool,
    setter_bytes: [u8; 16],    // fixed-size shuffle input
    payload: Vec<u8>,          // PCM (encode/roundtrip) or packet (decode)
}
```

Differential against C via 6 new FFI symbols added to
`tests/fuzz/fuzz_targets/c_reference.rs`:

```rust
extern "C" {
    fn opus_multistream_surround_encoder_create(...) -> *mut OpusMSEncoder;
    fn opus_multistream_decoder_create(...) -> *mut OpusMSDecoder;
    fn opus_multistream_encode(...) -> i32;
    fn opus_multistream_decode(...) -> i32;
    fn opus_multistream_encoder_destroy(...);
    fn opus_multistream_decoder_destroy(...);
}
```

Plus 4 wrapper functions (`c_ms_encode_surround`, `c_ms_decode`,
plus create/destroy helpers) following the `c_encode` /
`c_decode` pattern at `c_reference.rs:173` and `:136`.

**Sizing**: ~+250 lines added to `c_reference.rs` (now 353 lines →
~600). Wrapper functions are ~50 lines each.

Multistream uses i16 PCM only — `OpusMSEncoder` has no
`encode_float` (only mono `OpusEncoder` does), so the
`use_float_pcm` switch from Stream A doesn't apply here.

Oracle: byte-exact compare of compressed output for encode/roundtrip;
PCM compare for decode (with the SILK-skip carve-out replaced in
Stream D).

#### `fuzz_repacketizer_seq` (gap 5)

Replaces and extends `fuzz_repacketizer.rs`. Reads N packets from the
input bytes, calls `cat()` for each, then exercises both `out()` and
`out_range(begin, end)` on the accumulated state.

```rust
#[derive(Arbitrary)]
struct RepacketizerSeq {
    packets: Vec<Vec<u8>>,    // bounded 1..=8 in custom Arbitrary
    out_range_begin: u8,      // dynamically clamped to [0, nb_frames)
    out_range_end: u8,        // dynamically clamped to (begin, nb_frames]
}
```

`out_range` bounds are clamped at runtime against the actual frame
count returned by the repacketizer — `Arbitrary` can't know that
ahead of time without re-implementing the repacketizer.

Existing `fuzz_repacketizer.rs` is replaced, not augmented — its test
surface is a strict subset of the new target.

#### `fuzz_dnn_blob` (gap 7)

Pure safety target. `OpusDecoder::set_dnn_blob(&[u8])` (decoder.rs:652)
is a parser surface returning `Result<(), i32>` that is currently
unfuzzed.

```rust
fuzz_target!(|data: &[u8]| {
    let mut dec = OpusDecoder::new(48000, 2).expect("valid sr/ch");
    let _ = dec.set_dnn_blob(data); // must not panic, OOB-read, or hang
    // Optional: try decoding a single small valid packet afterwards to
    // exercise the post-blob state.
});
```

No differential — the C reference does not expose `set_dnn_blob` with
the same semantics. Panic-free is the only oracle.

DRED encoder coverage (`set_dred_duration`) is **not** added in this
HLD: encoder lazily allocates DRED state and consumes the same
embedded weights, so once `fuzz_dnn_blob` is solid an encoder
counterpart is a 30-line follow-up.

### Stream D — Better oracles and seeds (gaps 6, 9)

#### SILK / Hybrid decode SNR oracle (gap 6)

`fuzz_decode.rs:125` and `fuzz_roundtrip.rs` currently skip PCM
compare when the encoded packet is SILK or Hybrid. Replace the skip
with a bounded-SNR check, matching the project's tier-2 envelope
(≥50 dB documented in `CLAUDE.md`).

The injection point is the `if celt_only { ... }` branch in
`fuzz_decode.rs:125`, gated on `is_celt_only_packet(packet)` (defined
at `fuzz_decode.rs:67`). New `oracle.rs` helper module included via
`#[path = "oracle.rs"] mod oracle;` (same pattern as `c_reference.rs`):

```rust
const SILK_DECODE_MIN_SNR_DB: f64 = 50.0;

pub fn snr_db(reference: &[i16], test: &[i16]) -> f64 {
    if reference.len() != test.len() { return f64::NEG_INFINITY; }
    let (mut sig, mut noise) = (0.0f64, 0.0f64);
    for (&r, &t) in reference.iter().zip(test) {
        sig   += (r as f64).powi(2);
        noise += ((r as f64) - (t as f64)).powi(2);
    }
    if noise == 0.0 { return f64::INFINITY; }       // exact match
    if sig   == 0.0 { return f64::NEG_INFINITY; }   // silence-in, noise-out = bug
    10.0 * (sig / noise).log10()
}
```

Replace the skip:

```rust
if is_celt_only_packet(packet) {
    assert_eq!(rust_slice, &c_pcm[..], "PCM mismatch ...");
} else {
    let snr = oracle::snr_db(&c_pcm, rust_slice);
    assert!(
        snr >= SILK_DECODE_MIN_SNR_DB,
        "SILK decode SNR {snr:.2} dB < {SILK_DECODE_MIN_SNR_DB:.0} dB \
         (sr={sample_rate}, ch={channels}, packet_len={})",
        packet.len()
    );
}
```

**Validation of the SNR helper itself**: `oracle.rs` ships with a
`#[cfg(test)] mod tests` block exercising `snr_db` on golden cases —
identical signals → INFINITY, off-by-one-LSB → ~96 dB, silence-vs-noise
→ NEG_INFINITY, length mismatch → NEG_INFINITY. Run via
`cargo +nightly test --fuzz-dir tests/fuzz` (cargo-fuzz supports `cargo
test` on the fuzz crate). This avoids the V1 "deliberately break the
decoder, watch the assert fire" manual ritual.

CELT-only packets remain byte-exact compared (no regression).

#### Pathological seed corpus (gap 9)

`tools/gen_fuzz_seeds.sh` (~50 lines bash) generates ~10 PCM seeds per
encode target covering known-pathological signals:

- DC offset (constant max +i16, min, zero)
- Max-amplitude sine at 100 Hz, 1 kHz, 8 kHz, 22 kHz (Nyquist)
- White noise at full scale
- Step click (silence → max → silence)
- Periodic impulse train (pitch-detector stress)
- Near-silence with single-LSB toggling (quantizer stress)

Seeds emitted into `tests/fuzz/seeds/<target>/` (committed) matching
the per-target structured-input layout (8 config bytes + PCM).
`overnight_fuzz_launch.sh` copies them into
`tests/fuzz/corpus/<target>/` at run start with `cp -n` (no clobber):

```bash
# new pre-launch step
for t in "${TARGETS[@]}"; do
    SEEDS_DIR="$ROOT/tests/fuzz/seeds/$t"
    CORPUS_DIR="$ROOT/tests/fuzz/corpus/$t"
    if [ -d "$SEEDS_DIR" ]; then
        mkdir -p "$CORPUS_DIR"
        cp -n "$SEEDS_DIR"/* "$CORPUS_DIR/" 2>/dev/null || true
    fi
done
```

Total seed budget: ~10 patterns × 4 encode targets × ~10 KB ≈
**400 KB committed**. Small enough to commit; treats them as fixtures,
not corpus, sidestepping the "no multi-megabyte corpora" README hint.

## Acceptance criteria

A subsequent overnight campaign run after all four streams land must:

1. Build cleanly (`cargo +nightly fuzz build --fuzz-dir tests/fuzz`).
2. Show all 10 targets running and producing libFuzzer pulse output.
3. Observe non-CBR encode paths in coverage — verify by checking the
   `cov:` line in `fuzz_encode/run.log` exceeds the 8020 ceiling from
   the 2026-04-29 run. Expected: ≥9000 with VBR enabled.
4. Float-path coverage non-zero — check `cov:` line on
   `fuzz_encode/run.log` and `fuzz_decode/run.log` shows growth past
   their 2026-04-29 baselines (5501 and 8020 respectively) by ≥10%.
5. SNR-helper unit tests pass (`cargo +nightly test --fuzz-dir
   tests/fuzz oracle::tests`). Replaces V1's manual deliberate-break
   ritual.
6. Run for 8 h with 0 crashes / 0 panics, OR if crashes are found,
   minimise them and produce regression entries — that is the
   intended outcome of this work.

## Deliberately excluded

Adding any of these later requires a separate HLD-level review.
YAGNI per CLAUDE.md.

- **Custom libFuzzer mutators.** Standard mutations + structured
  `Arbitrary` cover the surface.
- **Coverage-guided corpus minimisation.** `cargo fuzz cmin` exists;
  we don't run it.
- **Soft-clip / projection encoder targets.**
- **DRED encoder fuzz target.** Follow-up after `fuzz_dnn_blob`.
- **Tonality analysis target.** Reachable transitively.
- **Multistream float-path target.** `OpusMSEncoder` has no
  `encode_float` to fuzz; if/when one is added, extend
  `fuzz_multistream`.
- **OpusInfo / Ogg parsing.** No Ogg muxer in this crate.
- **Cross-target panic-capture deduplication.**
- **CI-gated fuzzing.** Out of scope per the original BRIEF.

## Implementation order

Each stream is a separate PR. Streams are independently mergeable.

1. **Stream A** (smallest delta, biggest coverage win): VBR + FEC +
   DTX + float-path + decoder-setter shuffle + encoder-setter
   shuffle. ~250 lines diff across 4 targets. **First**, because
   gaps 1 and 11 are the largest uncovered surfaces and the change
   is mechanical.
2. **Stream B**: retire `fuzz_packet_parse`, rebuild
   `fuzz_encode_multiframe` with `Arbitrary`. ~300-line net change
   (-124 + ~420). **Second**, because it unlocks CPU for Stream C
   targets and validates the `Arbitrary` derive feature gate.
3. **Stream C**: three new targets (multistream, repacketizer_seq,
   dnn_blob). ~700 lines total across 3 targets + ~250 lines
   `c_reference.rs`. Can ship as one PR or three. **Third**,
   because the multistream FFI extension needs care.
4. **Stream D**: SILK SNR oracle + pathological seeds + launcher
   seed-copy preamble. ~80 lines (oracle helper) + ~50 lines (seed
   generator) + ~10 lines (launcher) + seed binaries. **Fourth**,
   because the SNR oracle is most useful once gap 1 is closed.

Recommend overnight campaigns *between* streams to validate each
landing. ~32 h of fuzz time across 4 PRs is cheap insurance against
late-stage regressions.
