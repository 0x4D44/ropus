# Coverage Improvement Plan

Date: 2026-04-08

## Goal

Raise Rust library coverage from the current nightly baseline:

- Lines: 86.99%
- Functions: 90.75%
- Regions: 87.99%
- Branches: 69.35%

Primary target: drive line coverage toward 98% while materially improving functions, regions, and branches at the same time.

## Stage 0: Keep The Measurement Clean

Status: complete

- Use `cargo +nightly llvm-cov --branch --lib --features "simd dnn"`
- Export `MDOPUS_SKIP_REFERENCE=1` until the xiph/opus `reference/` tree is available
- Keep validation scoped to `--lib` so harness issues do not pollute Rust library coverage
- Treat any failing test as a blocker and fix it before measuring

## Stage 1: Highest-Yield Orchestrator Tests

Target files:

- `src/opus/encoder.rs`
- `src/celt/decoder.rs`
- `src/dnn/lpcnet.rs`
- `src/opus/multistream.rs`

Work:

- Add deterministic tests for invalid ctl combinations, edge frame sizes, tiny-budget branches, forced mode transitions, and fallback/error behavior
- Add PLC/FEC/DTX and decode-side branch coverage where state transitions can be asserted directly
- Add constructor/layout/packet parsing tests in multistream code to pick up the missing function count quickly

Expected result:

- fastest immediate lift in missed functions
- meaningful line and branch improvement without touching codec math internals

## Stage 2: Large Missed-Line Buckets

Target files:

- `src/silk/encoder.rs`
- `src/celt/bands.rs`
- `src/opus/decoder.rs`
- `src/opus/repacketizer.rs`

Work:

- Add tests around decision heuristics, edge-case state transitions, packet shapes, bandwidth/control branches, and spread/anti-collapse style logic
- Prefer direct unit tests over broad end-to-end vectors where branch intent can be asserted locally

Expected result:

- reduce the largest remaining pools of missed lines
- lift branch coverage in files currently stuck in the 59-70% band

## Stage 3: Function Closure Sweep

Target files:

- `src/celt/ec_ctx.rs`
- residual low-function-coverage helpers across `opus`, `celt`, and `dnn`

Work:

- inspect remaining uncovered functions from the llvm-cov JSON
- add narrow tests for untouched helpers, wrapper paths, and small state machines
- avoid duplicating coverage that Stage 1 or 2 already created incidentally

Expected result:

- push functions and regions upward with relatively cheap tests

## Stage 4: Branch-First Cleanup

Work:

- use branch deltas after each wave to choose the next file
- add tests specifically for opposite branches, not just more happy paths
- target modules with high line coverage but stubborn branch gaps:
  - `src/celt/bands.rs`
  - `src/celt/encoder.rs`
  - `src/opus/repacketizer.rs`
  - `src/silk/decoder.rs`

Expected result:

- convert “looks covered” files into genuinely decision-covered files

## Working Rules

- Commit after each meaningful coverage wave
- Keep worker ownership per file to avoid merge collisions
- Validate narrowly first, then rerun full `llvm-cov` after each wave
- Prefer tests that exercise existing internal APIs and state directly
- Avoid broad refactors unless coverage work exposes a real testability defect

## Exit Criteria

- `cargo +nightly llvm-cov --branch --lib --features "simd dnn"` passes cleanly
- test suite is green
- line coverage reaches or exceeds 98%
- functions, regions, and branches all improve materially from baseline

## If Progress Stalls

- Re-rank using actual missed-line deltas from the latest JSON instead of intuition
- Shift effort toward the file with the largest remaining missed-line pool
- Stop writing low-yield helper tests once a file is already above 95% lines unless it is blocking branch progress
