# Coverage Improvement Plan

## Stage 1: Finish Encoder Hot Paths

Goal: push `src/opus/encoder.rs` higher with deterministic tests only.

Targets:

- helper math: `frame_size_select`, `bits_to_bitrate`, `bitrate_to_bits`, `compute_frame_energy`
- buffer helpers: `update_delay_buffer`, `update_delay_buffer_from_pcm_buf`
- mode transitions: CELT -> SILK, SILK -> CELT, forced mode overrides
- branch edges: CBR padding, tiny budgets, FEC/redundancy decisions

Done so far:

- helper math coverage added
- delay-buffer branches added
- transition prefill path added

## Stage 2: Attack `dnn::lpcnet`

Goal: remove the biggest single branch sink in the current library run.

Targets:

- model-load error branches
- PLC state transitions
- frame-network helper paths
- low-cost math helpers with deterministic inputs

This file has the largest remaining branch gap, so it is likely the highest payoff after the encoder.

## Stage 3: Attack `silk::encoder` And `opus::multistream`

Goal: cover the large control-flow surfaces that drive encode decisions.

Targets:

- `silk::encoder`: control-input validation, bandwidth switching, DTX/FEC transitions, resampler edge cases
- `opus::multistream`: mapping validation, layout edge cases, surround/projection branches, packet assembly paths

These files are too broad for broad smoke tests. They need scenario-driven unit tests that pin one branch at a time.

## Stage 4: Clean Up `celt::decoder` And Remaining Gaps

Goal: mop up the remaining low-80s and mid-70s files.

Targets:

- `celt::decoder`: packet loss, prefilter/fold, transition, and PLC behavior
- `opus::decoder`: packet parsing and wrapper edge cases
- `celt::bands` / `celt::encoder`: the remaining spectral and control branches

## Stage 5: Re-run Nightly Branch Coverage

Goal: measure deltas after each stage and stop adding tests when the marginal gain collapses.

Rules:

- always run the narrowest file/module test set first
- rerun `cargo +nightly llvm-cov --branch --lib --features "simd dnn"` after each meaningful batch
- keep the reference harness skipped until the xiph tree exists in `reference/`

## Practical Success Criteria

- no failing tests
- no new warnings from the touched file
- repeatable branch gains on the large hotspot files
- stop when the remaining misses are concentrated in code that is realistically unreachable without changing product behavior
