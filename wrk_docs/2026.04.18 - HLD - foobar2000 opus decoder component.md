# HLD — foobar2000 input component backed by ropus

**Status:** draft, rev 2 (post-review: fb2k `file::ptr` integration, abort plumbing, fast-path info, seek pre-roll, ReplayGain mapping)
**Author:** Arthur + Claude
**Date:** 2026-04-18

## 1. Motivation

We already have a working Rust Opus decoder (`ropus`) and a libopus-shaped C ABI
(`capi`). foobar2000 is a widely-used Windows audio player with a stable C++
component SDK. Putting our decoder behind an `input_impl` service gives us:

- a realistic long-form listening test (hours of varied content, live seeking,
  A/B against foobar2000's built-in libopus decoder),
- a publishable artifact that exercises the C boundary in a third-party host,
- confidence that the decoder's threading and resource behaviour survives
  something more stressful than our harness.

foobar2000 already decodes Opus via its bundled libopus. The new component
replaces that path for `.opus` files with our code.

## 2. Scope

**In:**
- Read `.opus` files in an Ogg container (RFC 7845), channel-mapping family 0
  (mono / stereo), 48 kHz.
- Read through foobar2000's `file::ptr` abstraction, not raw disk paths, so
  files inside archives and on fb2k-supported remote filesystems work.
- Honour fb2k's `abort_callback` on every long-running operation
  (open, seek, decode loop).
- Decode to interleaved PCM and hand foobar2000 an `audio_chunk` with the
  correct `channel_config`.
- Report duration, bitrate, channel count, sample rate in file properties.
- Surface `OpusTags` (vorbis_comment) so fb2k can show artist / title /
  album / date.
- Map `R128_TRACK_GAIN` / `R128_ALBUM_GAIN` (and legacy `REPLAYGAIN_*`) into
  `file_info::replaygain_info` so playback loudness is correct.
- Seekable playback with ≥ 80 ms pre-roll per RFC 7845 §4.2 (no audible
  fizz after seeks).
- Fast path for fb2k library scans (`input_open_info_read`) that reads
  only what's needed for `file_info` — no full Ogg-page walk on 10 k-track
  rescans.
- Ship as a single 64-bit `.fb2k-component` (zip).

**Out (explicit non-goals):**
- Writing / editing tags (throws `exception_io_unsupported_format`).
- Multistream / surround (channel_mapping 1 or 255).
- Chained logical streams (multiple `OpusHead`s in one Ogg file). Real spec
  feature; rare in the wild; fallback is "plays the first logical stream".
- Embedded cover art (`METADATA_BLOCK_PICTURE` base64 in OpusTags). Skipped
  at parse time.
- Computing new ReplayGain values — we only read existing tags.
- Decoder options panel (fb2k preferences page).
- Non-Ogg containers (Matroska, MP4).
- Encoding (diskwriter). Playback only.
- 32-bit build. 64-bit only.
- Any attempt to replace foobar2000's built-in libopus at link time. Ours is a
  separate input that wins the association per §5.3.

## 3. Architecture

```
  .opus in an archive, on disk, or over HTTP
       │
       ▼   (opened by foobar2000 as a file::ptr)
┌──────────────────────────────────────────────────┐
│ fb2k-ropus (C++ DLL)                            │  ← foobar2000 SDK
│  input_impl_ropus_opus                          │
│    open(reason, file::ptr, abort_callback&)      │
│    decode_run → fills audio_chunk                │
│    decode_seek / decode_get_info                 │
│                                                  │
│  adapts file::ptr + abort_callback into a C      │
│  IO-callback struct (read / seek / size / abort) │
└─────────────────────────┬────────────────────────┘
                          │ C ABI (ropus_fb2k.h)
                          │   — no paths cross the boundary —
                          ▼
┌──────────────────────────────────────────────────┐
│ ropus-fb2k (Rust cdylib+rlib)                   │
│  OggOpusReader<Io>     — demux, tags, seek index │
│    └─> ropus::Decoder  — actual decode           │
└──────────────────────────────────────────────────┘
```

Two artefacts. The Rust cdylib owns the Ogg demux, seek index, tag parsing
and decoder. The C++ DLL is a thin adapter: it receives fb2k's `file::ptr`
and `abort_callback`, wraps them in a plain-C callback struct, and feeds the
result to Rust. Decoded samples come back into fb2k's `audio_chunk`.

This division keeps the C++ layer small and the interesting logic in Rust
where we can test it. The Rust side never touches the filesystem directly —
it only reads through the callback struct.

## 4. Rust crate — `ropus-fb2k`

New top-level workspace crate, `publish = false` — follows the `capi` /
`harness` pattern of narrow, in-repo tooling that co-evolves with `ropus`
but isn't meant for crates.io. `crate-type = ["cdylib", "rlib"]` — `cdylib`
for the DLL the C++ shim links, `rlib` so Rust integration tests in the
workspace can call the same entry points directly (no `libloading` dance).

```
ropus-fb2k/
├── Cargo.toml
├── include/
│   └── ropus_fb2k.h         # hand-written, shipped to the C++ side
├── src/
│   ├── lib.rs                # #[no_mangle] entry points + ffi_guard
│   ├── io.rs                 # CallbackReader: adapts the C IO struct to Read+Seek
│   ├── reader.rs             # OggOpusReader<R>: demux, header parse, seek index
│   ├── tags.rs               # OpusTags parser + ReplayGain extraction
│   └── error.rs              # thread-local last-error slot
└── tests/
    └── roundtrip.rs          # fixtures + compare against ropus-cli decode
```

### 4.1 Dependencies

- `ropus` (workspace path) — the decoder.
- `ogg = "0.9"` — same Ogg crate ropus-cli already depends on.

Nothing else. No symphonia, no async runtime, no logging framework.

### 4.2 C header

```c
// ropus_fb2k.h  (hand-written; stable across the C++ DLL and Rust cdylib)

#include <stdint.h>
#include <stddef.h>

typedef struct RopusFb2kReader RopusFb2kReader;

/* ----- IO callbacks supplied by the C++ side ------------------------------
 *
 * All callbacks receive the opaque `ctx` the caller registered in RopusFb2kIo.
 * The C++ shim wraps fb2k's file::ptr and abort_callback behind these; the
 * Rust side never touches the filesystem directly.
 *
 * - `read`:  read up to `n` bytes into `buf`. Return bytes read (>=0), or -1
 *            on I/O error. Return 0 only at EOF. Partial reads are fine —
 *            the Rust side loops until satisfied or EOF.
 * - `seek`:  seek to absolute byte offset. Return 0 on success, -1 on error.
 * - `size`:  write total stream length in bytes to *out_size. Return 0 on
 *            success, -1 if unknown (e.g. a live HTTP stream — we then
 *            disable seeking and report total_samples = 0).
 * - `abort`: return non-zero if the user has requested cancellation.
 *            Called frequently; must be cheap.
 */
typedef int64_t (*RopusFb2kReadFn) (void* ctx, uint8_t* buf, size_t n);
typedef int     (*RopusFb2kSeekFn) (void* ctx, uint64_t abs_offset);
typedef int     (*RopusFb2kSizeFn) (void* ctx, uint64_t* out_size);
typedef int     (*RopusFb2kAbortFn)(void* ctx);

typedef struct {
    void*              ctx;
    RopusFb2kReadFn   read;
    RopusFb2kSeekFn   seek;   /* may be NULL for unseekable streams */
    RopusFb2kSizeFn   size;   /* may be NULL if size is unknown */
    RopusFb2kAbortFn  abort;  /* may be NULL → never aborts */
} RopusFb2kIo;

/* ----- Open flags --------------------------------------------------------- */

/* Info-only: read just enough (first two pages + reverse-scan for last
 * granule) to populate RopusFb2kInfo + tags. Skips the full seek-index
 * build. This is the fast path for fb2k's `input_open_info_read` during
 * library scans. decode_next / seek will still work but will build the
 * seek index lazily on first seek. */
#define ROPUS_FB2K_OPEN_INFO_ONLY  (1u << 0)

typedef struct {
    uint32_t sample_rate;     /* always 48000 for our scope */
    uint8_t  channels;        /* 1 or 2 */
    uint16_t pre_skip;        /* 48 kHz samples; Rust trims these on first decode */
    uint64_t total_samples;   /* per-channel, post-pre_skip; 0 if unknown */
    int32_t  nominal_bitrate; /* bits/s; -1 if unknown */

    /* ReplayGain, extracted from OpusTags at open time. In dB relative to the
     * ReplayGain reference (i.e. already converted from R128's -23 LUFS target
     * by adding +5 dB). NaN means "not present". */
    float rg_track_gain;
    float rg_album_gain;
    float rg_track_peak;      /* linear, NaN if absent */
    float rg_album_peak;
} RopusFb2kInfo;

/* ----- API ---------------------------------------------------------------- *
 *
 * Status convention: 0 = OK, negative = error code.
 * Errors: -1 BAD_ARG, -2 IO, -3 ABORTED, -4 INVALID_STREAM, -5 UNSUPPORTED.
 * Error codes are stable; human messages via ropus_fb2k_last_error().
 */

RopusFb2kReader* ropus_fb2k_open(const RopusFb2kIo* io, uint32_t flags);
void              ropus_fb2k_close(RopusFb2kReader*);

int ropus_fb2k_get_info(RopusFb2kReader*, RopusFb2kInfo* out);

/* Tag enumeration. Keys are uppercased (per vorbis_comment convention);
 * values are UTF-8, null-terminated, valid only for the duration of the
 * callback. METADATA_BLOCK_PICTURE is filtered out (see scope). */
typedef void (*RopusFb2kTagCb)(void* ctx, const char* key, const char* value);
int ropus_fb2k_read_tags(RopusFb2kReader*, RopusFb2kTagCb cb, void* ctx);

/* Decode next packet into caller buffer (interleaved float, 48 kHz).
 * `max_samples_per_ch` must be >= 5760 (120 ms, the longest Opus frame).
 * Returns samples-per-channel decoded, 0 on EOF, negative on error. */
int ropus_fb2k_decode_next(RopusFb2kReader*,
                            float*  out_interleaved,
                            size_t  max_samples_per_ch);

/* Seek to per-channel sample position (48 kHz). Clamped to [0, total_samples].
 * Post-seek, the next `decode_next` silently discards ≥ 80 ms per RFC 7845
 * §4.2 before returning real audio; callers do not see the pre-roll. */
int ropus_fb2k_seek(RopusFb2kReader*, uint64_t sample_pos);

/* Thread-local last-error string for the calling thread. Never NULL. */
const char* ropus_fb2k_last_error(void);
```

Shape rationale:
- Paths never cross the boundary. fb2k's filesystem abstraction handles
  archives, HTTP, etc; we'd gain nothing by duplicating that in Rust and
  we'd lose everything non-local if we bypassed it.
- `abort` is on the IO struct, not a separate call, so every inner Rust
  loop (page scan, decode loop, seek binary-search) can poll without
  needing a side-channel.
- `flags` is a plain bitmask. No env vars; no per-call "mode" enum that
  fb2k only has two shapes of anyway.

### 4.3 Internals

`OggOpusReader<R: Read + Seek>` is the core. `R` is the Rust-side adapter
over the C IO callbacks (see `src/io.rs`), so the same type is used by both
the `cdylib` path and in-process tests (which pass a `Cursor<Vec<u8>>`).

**Open — single path** (regardless of `INFO_ONLY` flag):
1. Read first page → parse `OpusHead`. Reject `channel_mapping ≠ 0` with
   `-5 UNSUPPORTED`.
2. Read second page → parse `OpusTags`; extract ReplayGain fields
   (`R128_*` in Q7.8 dB + 5 dB offset, legacy `REPLAYGAIN_*` overrides
   per-field; peaks pass through linear; missing tags → `NaN`); drop
   `METADATA_BLOCK_PICTURE` at the reporting boundary.
3. Reverse-scan from EOF to find the last Ogg page belonging to our logical
   stream; read its `granule_position`. `total_samples = granule - pre_skip`
   in O(1) disk seeks. Unseekable / unknown-size → `total_samples = 0`,
   `nominal_bitrate = -1` (still opens successfully).
4. `nominal_bitrate = file_size × 8 × 48000 / total_samples` (u128 math).

**Page index** is built lazily on the first `seek()` call, not at open.
Rationale: library scans never seek (they open → info_read → close), so an
eager walk would be wasted work. Playback without seek never builds it. When
the user actually seeks we pay for the walk once, and that's cheap (~5–10 KiB
for a typical album).

The `INFO_ONLY` flag is accepted for forward-compatibility (a future change
might use it to skip additional work) but currently changes no observable
behaviour.

`decode_next`: drive `PacketReader`, hand each Opus packet to the ropus
decoder, write interleaved samples into the caller's buffer, trim leading
pre-skip on the first call. Return per-channel samples decoded. Poll
`io.abort` between packets; return `-3 ABORTED` if set.

`seek(sample_pos)`:
1. If the page index is not built yet, build it now.
2. Binary-search for the largest `granule ≤ (sample_pos − PRE_ROLL_SAMPLES)`
   — i.e. rewind an extra 80 ms worth of audio (3 840 samples at 48 kHz) so
   the decoder has time to converge before the caller's target position.
3. Seek the underlying reader to that page's byte offset.
4. Reset the ropus decoder state.
5. Decode forward, discarding samples until we reach `sample_pos`. The
   caller never sees the pre-roll.

This matches libopusfile's `op_pcm_seek` behaviour and eliminates the
stream-convergence fizz after a seek. Cost: one extra decode of ~80 ms
worth of packets — imperceptible.

**Ropus decoder reset.** Verified in M1: the high-level `ropus::Decoder`
does **not** expose a public `reset()`. The low-level
`ropus::opus::decoder::OpusDecoder::reset` IS public (at
`ropus/src/opus/decoder.rs:575`) and matches C `OPUS_RESET_STATE` semantics
(no reallocation, sub-decoders re-init). M3 uses the low-level type directly
for seek. No new `ropus` machinery required.

**Panic safety.** All `#[no_mangle] extern "C"` entry points wrap their
bodies in `std::panic::catch_unwind` using the same `ffi_guard!` idiom as
`capi`. A panic becomes a negative status + a thread-local error string.

### 4.4 Threading

foobar2000 calls `decode_run` from a single worker thread per input instance.
Our reader is `!Sync` but `Send`; each handle is owned by exactly one thread.
The Rust side does not spawn threads and does not hold global mutable state
apart from the thread-local last-error slot.

### 5.2 `input_impl` responsibilities

Single class `input_ropus_opus` implementing `input_impl`. The class holds a
`RopusFb2kReader*`, an `RopusFb2kIo` struct with `ctx` pointing to its own
adapter object, and a `file::ptr` it keeps alive for the reader's lifetime.

| `input_impl` method       | Implementation |
|---------------------------|----------------|
| `open(reason, file, abort)` | Stash `file` + `abort` in the adapter. Throw `exception_io_unsupported_format` on `input_open_info_write`. On `input_open_info_read` call `ropus_fb2k_open(io, INFO_ONLY)`; on `input_open_decode` call without flags. Throw `exception_io_data` on negative status (message from `ropus_fb2k_last_error`). |
| `get_info`                | Translate `RopusFb2kInfo` into `file_info`: sample_rate, channel_count, bitrate, length. Call `ropus_fb2k_read_tags` and map each vorbis_comment key to a `file_info` meta entry. Set `replaygain_info` from the `rg_*` fields (NaN → absent). |
| `get_file_stats`          | Forward to the `file::ptr` (`file::get_stats`). |
| `decode_initialize`       | No-op; the Rust reader lazily allocates the decoder on the first `decode_next` call. |
| `decode_run(audio_chunk, abort)` | Refresh adapter's stored `abort`. Call `ropus_fb2k_decode_next` into a scratch `float[2 * 5760]`. Set `audio_chunk.set_data` with `sample_rate=48000` and `channel_config = channels==1 ? audio_chunk::channel_config_mono : audio_chunk::channel_config_stereo`. Return `false` on EOF (status 0). Throw `exception_aborted` on status -3. |
| `decode_seek(seconds, abort)` | Refresh adapter's `abort`. Convert seconds × 48000 → sample_pos. Forward to `ropus_fb2k_seek`. |
| `decode_can_seek`         | `true` iff **both** the IO had a non-null `seek` callback AND a successful `size` lookup (i.e. `total_samples > 0`). Unseekable streams (`seek` NULL) or seekable-but-unknown-size streams both report `false` — a stream's seekability is the stricter of the two, mirroring the Rust-side invariant in `ropus_fb2k_seek` (rejects any target when `file_size.is_none()`). |
| `decode_get_dynamic_info` | Return false — we don't change bitrate mid-stream. |
| `decode_on_idle`          | No-op. |
| `retag / remove_tags`     | Throw `exception_io_unsupported_format`. |

**IO adapter.** A small C++ struct that holds `file::ptr` + `abort_callback*`
and exposes four static callbacks matching `RopusFb2kIo`:

- `read`  → `m_file->read(buf, n, *m_abort)` inside a try/catch. fb2k throws
  `exception_aborted` and `exception_io_*`; on either we return -1 (aborted
  abort-flag was set anyway; IO errors turn into -1 with the message routed
  via `ropus_fb2k_last_error` indirectly — we just log the SDK message).
- `seek` → `m_file->seek(off, *m_abort)`; return 0 / -1.
- `size` → `m_file->get_size(*m_abort)` → `*out_size = n`; -1 if `file_size_invalid`.
- `abort` → `m_abort->is_aborting() ? 1 : 0`.

The adapter's `abort_callback*` is *rebound* on every `decode_run` / `decode_seek`
call (fb2k hands us a fresh `abort_callback&` per call). This keeps cancellation
responsive without requiring the reader to own the lifetime of the callback.

Registered via `input_singletrack_factory_t<input_ropus_opus>` with content
type `audio/ogg` / `audio/opus` and extension `.opus`. Required statics on
the class:

| Static                                 | Value |
|----------------------------------------|-------|
| `g_get_guid()`                         | A fresh, hard-coded GUID generated once for this component. Distinct from any built-in. |
| `g_get_name()`                         | `"ropus (Rust Opus decoder)"` — shown in the prefs list. |
| `g_is_our_path(path, ext)`             | `stricmp_utf8(ext, "opus") == 0` |
| `g_is_our_content_type(ct)`            | `audio/ogg` or `audio/opus` |
| `g_is_low_merit()`                     | `false` |

### 5.3 Winning the `.opus` association

foobar2000 ≥ 1.4 exposes a user-editable decoder priority list at
*Preferences → Playback → Decoding*. `input_manager_v2::open_v2` walks the
registered `input_entry_v2` services in that list, catching
`exception_io_no_handler_for_path` / `exception_io_unsupported_format` and
falling through to the next one. First non-throwing opener wins.

Where a newly-installed decoder lands on first sighting is controlled by
`g_is_low_merit()`:

- `false` → inserted at the **top** of the list. This is what we want.
- `true`  → inserted at the **bottom**. vgmstream uses this deliberately
  because it does *not* want to beat built-ins; we do the opposite.

After first install, the user's saved order is authoritative — fb2k never
overwrites it. So:

1. **On a fresh install of our component**, the combination of a distinct
   GUID + `is_low_merit == false` puts us above `foo_input_std`'s Opus entry
   and we handle `.opus` without any user action.
2. **On a machine where fb2k has already indexed Opus files before our
   component existed**, we land at the top of new entries but the user's
   existing priority list is untouched. In practice the built-in is still
   on top. README must document the one-time fix: drag *ropus* above
   *Standard Input Array* on the Decoding prefs page (or Ctrl+Shift+Up).

Precedent: foo_input_monkey is described in the foobar2000 v2.26 preview
changelog as *"You can once again install Monkey's Audio decoder component
to replace built-in FFmpeg decoder"* — same pattern.

### 5.4 Error translation

Mapping from Rust status codes to fb2k exceptions:

| Status | fb2k exception                      | Notes |
|--------|-------------------------------------|-------|
| -1 BAD_ARG         | `exception_io_data`            | Programmer error; should never reach users. |
| -2 IO              | `exception_io_data`            | Message from `ropus_fb2k_last_error`. |
| -3 ABORTED         | `exception_aborted`            | User cancelled; fb2k handles gracefully. |
| -4 INVALID_STREAM  | `exception_io_data`            | Corrupt / truncated Ogg file. |
| -5 UNSUPPORTED     | `exception_io_unsupported_format` | Channel mapping ≠ 0, etc. Tells fb2k to fall through to next input. |

NULL handle from `ropus_fb2k_open` → `exception_io_data` with the last-error
message. fb2k catches all `exception_io_*` itself and skips the track.

### 5.5 ReplayGain mapping

OpusTags encodes gain using [RFC 7845 §5.2]. Two conventions appear in the wild:

- **`R128_TRACK_GAIN` / `R128_ALBUM_GAIN`** — string-encoded signed 16-bit
  Q7.8 dB relative to the R128 -23 LUFS reference. The canonical Opus form.
- **`REPLAYGAIN_TRACK_GAIN` / `REPLAYGAIN_ALBUM_GAIN`** — legacy string form,
  e.g. `"-6.75 dB"`, already in ReplayGain dB.

fb2k's `replaygain_info` expects ReplayGain dB. We parse R128 tags, divide
by 256, add +5 dB (the R128→RG offset — R128 targets -23 LUFS, RG targets
-18 LUFS), and expose via `RopusFb2kInfo::rg_*_gain`. Legacy `REPLAYGAIN_*`
tags override their R128 counterparts when both are present (they're
already in the right units; no conversion). Peak tags (`R128_*_PEAK` /
`REPLAYGAIN_*_PEAK`) are linear; pass through.

Without this mapping, Opus encoded with `opusenc --bitrate N` would play
~5 dB louder through our component than through fb2k's built-in, which
users would immediately notice.

## 6. Build and packaging

### 6.1 Rust side

Target: `x86_64-pc-windows-msvc` (64-bit only — see §2 non-goals).
`cargo build --release -p ropus-fb2k --target x86_64-pc-windows-msvc`
produces:

- `target/x86_64-pc-windows-msvc/release/ropus_fb2k.dll`
- `target/x86_64-pc-windows-msvc/release/ropus_fb2k.dll.lib` (import library)

These drop into the C++ project's `lib/` folder. No cbindgen — the header is
hand-written and reviewed. ~120 lines; generating it adds a build dependency
for no win.

Match fb2k's CRT linkage (`/MD`) by setting `RUSTFLAGS="-C target-feature=+crt-static=no"`
(cargo's default on MSVC is already dynamic CRT, so typically no action needed —
note in README that if a future cargo default changes this, it must be pinned).

### 6.2 C++ side

CMake project targeting Visual Studio 2022, Windows SDK 10, 64-bit. One
`add_library` producing a DLL renamed `foo_input_ropus.dll`. Links:

- `lib/ropus_fb2k.dll.lib`
- `sdk/foobar2000/shared/shared-Release.lib`
- `sdk/foobar2000/foobar2000_SDK/foobar2000_SDK.lib`
- `sdk/pfc/pfc.lib`

CRT: `/MD` to match foobar2000's DLLs.

### 6.3 `.fb2k-component` zip layout

fb2k 2.0+ expects 64-bit components under an `x64/` folder inside the zip;
32-bit components (not used here) would go at the root. We ship:

```
foo_input_ropus.fb2k-component     (zip archive)
├── x64/
│   ├── foo_input_ropus.dll         ← the C++ shim
│   └── ropus_fb2k.dll              ← the Rust cdylib
└── (no 32-bit bits)
```

Users install by dragging the `.fb2k-component` file onto foobar2000, or via
*Preferences → Components → Install…*. fb2k relocates both DLLs into the
profile's `user-components-x64\foo_input_ropus\` on first run.

### 6.4 No workspace integration of the C++ build

We don't wire CMake into `cargo build`. Two reasons: (1) it pulls
Windows-only toolchain detection into the workspace; (2) the Rust workspace
CI doesn't need the component built on every push. A standalone
`fb2k-ropus/build.bat` is enough — runs `cargo build -p ropus-fb2k --release
--target x86_64-pc-windows-msvc` then `cmake --build build\ --config Release`
then zips the output.

## 7. Testing

Tier 1 — Rust-only (runs in normal CI), via the `rlib` crate type so tests
call the public C API directly (no `libloading`):
- **Round-trip (wiring)**: feed a known `.opus` through the API using a
  `Cursor<Vec<u8>>`-backed `RopusFb2kIo`. Decoded PCM must match a
  reference path that feeds the same packets directly to
  `ropus::opus::decoder::OpusDecoder`, sample-for-sample. This test verifies
  the FFI wiring and pre-skip trimming, not decoder correctness — the
  underlying `ropus` decoder is covered directly by the top-level
  `harness/` workspace crate (`ropus-compare`), which links the C
  reference via FFI and compares outputs sample-for-sample. `ropus-fb2k`'s
  wiring is covered by this wiring test; we deliberately do not re-run the
  C oracle through the fb2k layer.
- **Open is bounded in reads**: `open` on any fixture uses a small constant
  number of `read` callback invocations (two Ogg pages + a single 128 KiB
  reverse-scan for the last granule), never a full-file walk. Verified by
  counting calls to the `read` callback — ceiling is a small constant, not
  O(file_size). The `INFO_ONLY` flag is accepted and exercised by this test
  but currently produces the same behaviour as a default open.
- **Seek with pre-roll**: open a ≥ 30 s file, seek to 10 s, decode the next
  200 ms, compare against a contiguous reference decode windowed at
  10.0–10.2 s. With the 80 ms pre-roll the tolerance is ≤ 1e-3 RMS.
- **Abort**: IO callback whose `abort` returns 1 after N calls. Every API
  entry must unwind with `-3 ABORTED`, not hang.
- **ReplayGain**: fixture files with known R128 tags; `RopusFb2kInfo::rg_*`
  matches expected values within ±0.01 dB.
- **Truncated / malformed Ogg**: open returns `-4 INVALID_STREAM`; no panics,
  no UB.
- **Panic safety**: inject a forced panic in a hook during decode (test-only
  feature gate is the one place we permit feature flags, scoped to `tests/`);
  confirm `catch_unwind` surfaces a negative status.

Tier 2 — manual (documented in the component README):
- Install in foobar2000, play tracks at 32 / 64 / 128 / 256 kbps, mono and
  stereo.
- Seek within each track; confirm audio resumes without a stall, click, or
  audible convergence fizz.
- Play a `.opus` file **inside a `.zip`** added to the fb2k library — confirms
  the `file::ptr` path is exercised.
- Play a live Opus HTTP radio stream (e.g. one of the Radio Paradise Opus
  feeds) — confirms the unseekable code path and exposes any abort issues.
- A/B against foobar2000's built-in Opus decoder on the same files; any
  audible difference is a bug.
- Confirm Preferences → Playback → Decoding lists *ropus* above *Standard
  Input Array* on a fresh install (§5.3).

Tier 3 — deferred:
- Automated playback regression across a corpus.
- Fuzz the Ogg demux / OpusTags parser via the existing `tests/fuzz/`
  harness (arbitrary bytes as input — must not panic, must not read past
  buffer bounds, must return a clean `-4 INVALID_STREAM`).

## 8. Resolved questions

All design questions raised during review are now closed:

- **SDK version** — pin **SDK 2025-03-07** (the current release as of this HLD).
  Input interfaces (`input_impl`, `input_entry_v2`) have been stable across
  the 2023–2025 changelog; no reason to pick an older snapshot. C++17 is
  sufficient (the 2025-03-07 release reverted the planned C++20 mandate).
  Virtual callbacks in our `input_impl` overrides use `noexcept` to match
  the SDK convention introduced in 2024-12-03. User fetches
  `SDK-2025-03-07.7z` from foobar2000.org (see
  `tools/fetch-fb2k-sdk.ps1`); `fb2k-ropus/sdk/` is git-ignored; README
  records the version.
- **Association priority** — distinct GUID + `g_is_low_merit() == false`, see §5.3.
- **File access model** — adapt fb2k `file::ptr` via C callbacks, see §4.2 / §5.2.
- **Cancellation** — `abort_callback` on the IO struct, polled in every inner loop, see §4.2 / §4.3.
- **Library-scan performance** — `open` never triggers a full page walk; it reads two Ogg pages + a single bounded reverse-scan for the last granule. `ROPUS_FB2K_OPEN_INFO_ONLY` is accepted for forward-compatibility (see §4.3).
- **Seek convergence fizz** — 80 ms pre-roll per RFC 7845 §4.2, see §4.3.
- **ReplayGain** — R128/REPLAYGAIN tags mapped in `RopusFb2kInfo`, see §5.5.
- **CRT linkage** — `/MD` to match foobar2000's DLLs, see §6.1 / §6.2.
- **32-bit support** — not in scope, see §2.

## 9. Milestones

Each is committable on its own and leaves the tree green.

1. **M1 — Rust skeleton with IO callbacks.** `ropus-fb2k` crate compiles
   (`cdylib` + `rlib`). Defines `RopusFb2kIo`, `CallbackReader` (implements
   `Read + Seek`), `ropus_fb2k_open(INFO_ONLY)` parses header + tags from a
   `Cursor<Vec<u8>>` in a Rust test. Verifies the `ropus::Decoder` reset
   path is accessible (the one outstanding assumption in §4.3). ~1.5 days.

2. **M2 — decode + fast-path info.** `decode_next` wired to `ropus::Decoder`;
   `get_info` populated via reverse-scan on `INFO_ONLY`. Round-trip test
   vs `ropus-cli decode`. Abort test. ~1 day.

3. **M3 — seek with pre-roll, tags, ReplayGain.** Full page index, 80 ms
   pre-roll, `read_tags` callback, R128/RG mapping in `RopusFb2kInfo`.
   Seek test under tolerance. ~1.5 days.

4. **M4 — C++ component shell.** fb2k SDK vendored, CMake builds an empty
   `input_impl` that logs its methods and shows up in Components. Proves
   SDK wiring and the `/MD` CRT choice. ~half a day (once SDK is in place).

5. **M5 — wire decoding + file::ptr adapter.** Flesh out `input_ropus_opus`:
   `file::ptr`/`abort_callback` → `RopusFb2kIo` adapter, `audio_chunk`
   channel_config, ReplayGain mapping into `file_info`. First audible
   playback, including a `.opus` inside a `.zip`. ~1.5 days.

6. **M6 — packaging.** `build.bat` produces the `x64/`-layout
   `.fb2k-component` zip. README covers install, SDK fetch, first-install
   vs existing-index priority fix. ~half a day.

Total: roughly one calendar week of focused work, plus a half-day buffer.

## 10. Explicitly not included

- No feature flags in the release build. No environment variables. One build
  configuration, one release artifact. (`tests/` may use a test-only cargo
  feature to inject panic hooks; that never ships in the cdylib.)
- No cross-platform story. foobar2000 is Windows; so is this component.
  `ropus-fb2k` itself is portable Rust and will happen to build on Linux,
  but we don't promise or test that.
- No 32-bit support.
- No chained-stream support (multiple `OpusHead`s in one Ogg file). We play
  the first logical stream and stop. If this turns out to matter, follow-up HLD.
- No embedded cover art extraction.
- No back-compat shim for a pre-existing `foo_input_ropus`. There isn't one.
- No tag-writing path or preferences page. Adding either is a separate HLD.
