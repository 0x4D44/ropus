# HLD: DNN Pipeline Integration

## 1. Summary

Wire the five standalone DNN modules (core inference engine, LPCNet PLC, FARGAN vocoder, PitchDNN pitch estimator, and Lossgen testing utility) into the Opus encoder/decoder pipeline behind a `dnn` cargo feature flag. This replaces the standard PLC in the decoder with neural PLC (LPCNet + FARGAN), adds neural pitch estimation to the LPCNet encoder's feature extraction, enables DRED (Deep REDundancy) FEC support in the decoder, and provides a `OPUS_SET_DNN_BLOB` CTL for loading model weights at runtime. The existing 280/280 bit-exact tests must pass unchanged when the feature is disabled.

## 2. Investigation Findings

### 2.1 DNN Module Architecture

**`src/dnn/core.rs` (1,989 lines)** -- The inference engine. All computation is f32 (IEEE 754 single-precision), not fixed-point. Provides:
- Layer types: `LinearLayer`, `Conv2dLayer`, `WeightArray`, `AdaConvState`, `AdaCombState`, `AdaShapeState`
- Compute functions: `compute_generic_dense`, `compute_generic_gru`, `compute_generic_conv1d`, `compute_conv2d`, `compute_glu`, `adaconv_process_frame`, `adacomb_process_frame`, `adashape_process_frame`
- Weight loading: `parse_weights(data: &[u8]) -> Result<Vec<WeightArray>, i32>` parses a binary weight blob into named arrays. `linear_init` and `conv2d_init` create layers from named arrays.

**`src/dnn/lpcnet.rs` (2,316 lines)** -- Three major subsystems:
1. **`LPCNetEncState`** (line 1321): Feature extractor. Calls `compute_frame_features()` to produce NB_FEATURES=20 spectral features per 10ms frame, including a PitchDNN call at line 1513-1515. Currently contains a **stub** `PitchDNNState` (lines 1150-1167) that always returns 0.0 -- the actual implementation is in `pitchdnn.rs`.
2. **`LPCNetState`** (line 1591): Neural speech synthesizer. `run_frame_network()` computes conditioning, `synthesize()` generates PCM.
3. **`LPCNetPLCState`** (line 1890): The key integration point. Combines the encoder, PLC prediction network, and FARGAN vocoder. API:
   - `load_model(&mut self, data: &[u8]) -> i32` -- loads weights for encoder, PLC net, and FARGAN
   - `update(&mut self, pcm: &[i16]) -> i32` -- feed received PCM (good frames)
   - `conceal(&mut self, pcm: &mut [i16]) -> i32` -- generate concealment PCM (lost frames)
   - `fec_add(&mut self, features: Option<&[f32]>)` -- add DRED FEC features
   - `fec_clear(&mut self)` -- clear FEC buffer
   - `reset(&mut self)` -- reset PLC state

   The FARGANState wrapper (lines 1169-1188) already delegates to `super::fargan::FarganState`. The PitchDNNState wrapper (lines 1150-1167) is a **no-op stub** that must be replaced with the real `pitchdnn::PitchDnnState`.

**`src/dnn/fargan.rs` (1,141 lines)** -- FARGAN vocoder. Synthesizes 160 samples (10ms at 16kHz) from NB_FEATURES feature vectors. Used inside `LPCNetPLCState::conceal()` via the `FARGANState` wrapper. Already fully wired to `LPCNetPLCState` -- no separate integration needed.

**`src/dnn/pitchdnn.rs` (677 lines)** -- Neural pitch estimator. `PitchDnnState::compute(if_features, xcorr_features) -> f32` returns a log-frequency pitch estimate. Called from `LPCNetEncState::compute_frame_features()` at line 1513. Currently disconnected because `lpcnet.rs` uses a stub. Must wire the real `pitchdnn::PitchDnnState` in place of the stub.

**`src/dnn/lossgen.rs` (808 lines)** -- Testing utility only. `LossGenState::sample_loss(percent_loss) -> i32` generates temporally-correlated loss patterns. Does NOT integrate into the codec pipeline -- it is used for evaluating PLC quality in test harnesses.

### 2.2 C Reference Integration Pattern

The C reference uses `#ifdef ENABLE_DEEP_PLC` and `#ifdef ENABLE_DRED` guards:

1. **OpusDecoder** contains `LPCNetPLCState lpcnet` (ref: `reference/src/opus_decoder.c:76`). Initialized in `opus_decoder_init` (line 180). Model weights loaded via `OPUS_SET_DNN_BLOB_REQUEST` CTL (line 1218).

2. **SILK PLC path**: `silk_Decode()` passes `&st->lpcnet` through to `silk_PLC()`. On good frames, `lpcnet_plc_update()` is called per subframe pair (every 10ms at 16kHz). On lost frames, if `lpcnet->loaded && fs_kHz == 16`, `lpcnet_plc_conceal()` replaces the classical PLC output. The condition `enable_deep_plc || fec_fill_pos != 0` gates whether neural PLC runs (ref: `reference/silk/PLC.c:400-401`).

3. **CELT PLC path**: `celt_decode_lost()` adds `FRAME_PLC_NEURAL` and `FRAME_DRED` frame types (ref: `reference/celt/celt_decoder.c:727-734`). Neural PLC requires `complexity >= 5`, `plc_duration < 80`, and loaded model. The neural PLC synthesizes at 16kHz and resamples to 48kHz via a sinc filter, with cross-fade during transition frames. `update_plc_state()` downsamples decode memory from 48kHz to 16kHz and feeds it to `lpcnet_plc_update()`.

4. **DRED FEC path**: In `opus_decode_native`, when a `dred` argument is provided, FEC features are fed to `lpcnet_plc_fec_add()` before decode (ref: `reference/src/opus_decoder.c:735-755`).

5. **Encoder side**: DNN integration in the encoder is minimal -- only `OPUS_SET_DNN_BLOB_REQUEST` to load DRED encoder weights (ref: `reference/src/opus_encoder.c:3325`). DRED encoding is a separate subsystem not yet ported. PitchDNN is used only in the `LPCNetEncState` feature extractor, which lives entirely within the DNN module.

### 2.3 Current Rust Decoder PLC Architecture

The Rust decoder at `src/opus/decoder.rs` currently:
- On PLC (data=None), calls `decode_frame(None, ...)` which sets `lost_flag=1`
- SILK path: `silk_decode()` calls `silk_decode_frame()` which calls `silk_plc_conceal()` -- classical PLC only
- CELT path: `decode_lost()` uses `FRAME_PLC_PERIODIC` or `FRAME_PLC_NOISE` -- no neural option
- There is no `LPCNetPLCState` in the `OpusDecoder` struct (line 424)
- The `SilkPlcState` has no `enable_deep_plc` field
- The `SilkDecoderState` has `// OSCE fields would go here if enabled` placeholder (line 188)

## 3. Design

### 3.1 Feature Flag

Add to `Cargo.toml`:
```toml
[features]
default = []
dnn = []
```

All DNN integration code uses `#[cfg(feature = "dnn")]`. The existing `src/dnn/` module compiles unconditionally (it already does), but the wiring into the codec pipeline is conditional.

### 3.2 Integration Point 1: PitchDNN in LPCNet Encoder

**File**: `src/dnn/lpcnet.rs`

**Current** (lines 1150-1167): Stub `PitchDNNState` with no-op `compute()`.

**Change**: Replace the stub with a conditional delegation:
```rust
#[cfg(feature = "dnn")]
pub struct PitchDNNState {
    inner: super::pitchdnn::PitchDnnState,
}

#[cfg(not(feature = "dnn"))]
pub struct PitchDNNState { _initialized: bool }
```

When `dnn` is enabled, `PitchDNNState::compute()` delegates to `self.inner.compute()`, returning the real neural pitch estimate. When disabled, it returns 0.0 as today.

`load_model()` parses the weight blob and initializes `PitchDnnState` via `PitchDnnState::load_model()`.

**No signature changes** -- `LPCNetEncState` already calls `self.pitchdnn.compute()` at line 1513. The only change is the backing implementation.

### 3.3 Integration Point 2: LPCNetPLCState in OpusDecoder

**File**: `src/opus/decoder.rs`

**Add field** to `OpusDecoder` struct (after line 448):
```rust
#[cfg(feature = "dnn")]
lpcnet: crate::dnn::lpcnet::LPCNetPLCState,
```

**Initialization**: In `OpusDecoder::new()` (line 461), add:
```rust
#[cfg(feature = "dnn")]
lpcnet: {
    let mut st = crate::dnn::lpcnet::LPCNetPLCState::new();
    st.init();
    st
},
```

**Model loading CTL**: Add method to `OpusDecoder`:
```rust
#[cfg(feature = "dnn")]
pub fn set_dnn_blob(&mut self, data: &[u8]) -> Result<(), i32> {
    let ret = self.lpcnet.load_model(data);
    if ret == 0 { Ok(()) } else { Err(ret) }
}
```

**Reset**: In `OpusDecoder::reset()`, add `self.lpcnet.reset()`.

### 3.4 Integration Point 3: SILK PLC Neural Path

**File**: `src/silk/decoder.rs`

**Step 3a -- Add `enable_deep_plc` to SilkPlcState** (after line 72):
```rust
#[cfg(feature = "dnn")]
pub enable_deep_plc: bool,
```

**Step 3b -- Modify `silk_plc` signature chain**

The key change: `silk_decode_frame` and the PLC functions need access to the `LPCNetPLCState`. The C reference threads it through as an extra parameter. In Rust, the cleanest approach is:

Add an `Option<&mut LPCNetPLCState>` parameter to:
1. `silk_decode` (line 2158) -- already `pub(crate)`, called from `opus/decoder.rs`
2. `silk_decode_frame` (line 2056) -- called from `silk_decode`
3. `silk_plc` (internal, called from `silk_decode_frame`)
4. `silk_plc_conceal` (line 1101)

When `feature = "dnn"` is disabled, these parameters don't exist. Use a helper type:

```rust
#[cfg(feature = "dnn")]
pub type DnnPlcArg<'a> = Option<&'a mut crate::dnn::lpcnet::LPCNetPLCState>;
#[cfg(not(feature = "dnn"))]
pub type DnnPlcArg<'a> = ();
```

**Step 3c -- Neural PLC logic in `silk_plc`**

In the existing `silk_plc` equivalent (called from `silk_decode_frame`):

On **good frames** (lost_flag == 0): After `silk_plc_update()`, if DNN is enabled and `fs_kHz == 16`:
```rust
#[cfg(feature = "dnn")]
if let Some(lpcnet) = lpcnet.as_deref_mut() {
    if dec.s_plc.fs_khz == 16 {
        for k in (0..dec.nb_subfr).step_by(2) {
            lpcnet.update(&frame[k * dec.subfr_length..(k+2) * dec.subfr_length]);
        }
    }
}
```

On **lost frames** (lost_flag != 0): After classical `silk_plc_conceal()`, if `lpcnet.loaded && fs_kHz == 16 && (enable_deep_plc || fec_fill_pos != 0)`:
```rust
#[cfg(feature = "dnn")]
if let Some(lpcnet) = lpcnet.as_deref_mut() {
    if lpcnet.loaded && dec.s_plc.fs_khz == 16 {
        let run_deep = dec.s_plc.enable_deep_plc || lpcnet.fec_fill_pos != 0;
        if run_deep {
            for k in (0..dec.nb_subfr).step_by(2) {
                lpcnet.conceal(&mut frame[k * dec.subfr_length..]);
            }
            // Backpatch sLPC_Q14 from neural output
        } else {
            for k in (0..dec.nb_subfr).step_by(2) {
                lpcnet.update(&frame[k * dec.subfr_length..]);
            }
        }
    }
}
```

**Step 3d -- Thread through from opus/decoder.rs**

In `OpusDecoder::decode_frame()`, the `silk_decode()` call at line 702 adds:
```rust
#[cfg(feature = "dnn")]
&mut self.lpcnet,
```

### 3.5 Integration Point 4: CELT PLC Neural Path

**File**: `src/celt/decoder.rs`

This is the most complex integration point. The C reference adds:
- `FRAME_PLC_NEURAL` and `FRAME_DRED` frame type constants
- 16kHz→48kHz sinc resampling for neural PLC output
- State for PLC PCM buffer and preemphasis memory
- `update_plc_state()` function for downsampling decode memory

**Step 4a -- Add state fields to CeltDecoder** (after line 135):
```rust
#[cfg(feature = "dnn")]
pub plc_pcm: Vec<i16>,        // 16kHz PLC buffer
#[cfg(feature = "dnn")]
pub plc_fill: usize,          // samples filled in plc_pcm
#[cfg(feature = "dnn")]
pub plc_preemphasis_mem: f32,  // preemphasis filter memory
```

**Step 4b -- Add frame type constants**:
```rust
#[cfg(feature = "dnn")]
pub const FRAME_PLC_NEURAL: i32 = 5;
#[cfg(feature = "dnn")]
pub const FRAME_DRED: i32 = 6;
```

**Step 4c -- Modify `decode_lost` and `decode_with_ec`**

Add `lpcnet: DnnPlcArg` parameter to:
- `decode_lost()` (line 750)
- `decode_with_ec()` (line 1134)

In `decode_lost()`, add neural PLC frame type selection (after line 766):
```rust
#[cfg(feature = "dnn")]
let curr_frame_type = if let Some(ref lpcnet) = lpcnet {
    if start == 0 && lpcnet.loaded && self.complexity >= 5
        && self.plc_duration < 80 && self.skip_plc == 0
    {
        if lpcnet.fec_fill_pos > lpcnet.fec_read_pos {
            FRAME_DRED
        } else {
            FRAME_PLC_NEURAL
        }
    } else {
        curr_frame_type  // fallback to periodic/noise
    }
} else { curr_frame_type };
```

For the neural synthesis path: downsample 48kHz decode memory to 16kHz, call `lpcnet.conceal()` to get 16kHz PCM, then upsample back to 48kHz using the sinc resampler, with cross-fade at transitions.

**Step 4d -- Add sinc resampler and update_plc_state**

Port the `update_plc_state()` and sinc filter from C reference (`reference/celt/celt_decoder.c:639-672`). This is a simple 17-tap polyphase filter with a preemphasis stage.

**Step 4e -- Thread through from opus/decoder.rs**

In `OpusDecoder::decode_frame()`, pass `&mut self.lpcnet` to both `silk_decode()` and the CELT `decode_with_ec()` call.

### 3.6 Integration Point 5: DRED FEC in decode_native

**File**: `src/opus/decoder.rs`

In `decode_native()` (line 1003), before the normal decode path, add DRED FEC handling. This requires a new `dred` parameter (or a separate `decode_with_dred` method):

```rust
#[cfg(feature = "dnn")]
pub fn decode_with_dred(
    &mut self,
    data: Option<&[u8]>,
    pcm: &mut [i16],
    frame_size: i32,
    dred: Option<&DredState>,
    dred_offset: i32,
) -> Result<i32, i32> { ... }
```

When DRED features are present and the packet is missing, feed FEC features to `self.lpcnet.fec_add()` before calling the PLC path. This matches C reference lines 735-755.

Note: Full DRED support requires the DRED decoder (rdovae), which is NOT yet ported. This integration point should be stubbed with the FEC API wired up, but actual DRED decoding deferred.

### 3.7 Integration Point 6: Lossgen (No pipeline integration)

`src/dnn/lossgen.rs` is a standalone testing utility. It does NOT integrate into the codec pipeline. It should be usable from the test harness when the `dnn` feature is enabled, but requires no changes to encoder/decoder code.

### 3.8 Data Flow Diagram

```
Encoder side (future DRED):
  PCM audio
    |
    v
  [SILK Encoder] --> bitstream
    |
    +-- [LPCNetEncState::compute_frame_features()]
            |
            +-- [PitchDNN::compute()] <-- neural pitch estimate (Integration Point 1)
            |
            +-- DRED features --> extension header (future, not in scope)

Decoder side:
  bitstream --> [OpusDecoder::decode_native()]
                    |
                    +-- DRED FEC features --> [lpcnet.fec_add()] (Integration Point 5)
                    |
                    v
                [decode_frame()]
                    |
                    +-- SILK path: [silk_decode()] --> [silk_plc()]
                    |       |
                    |       +-- good frame: [lpcnet.update(pcm)] (Integration Point 3)
                    |       +-- lost frame: [lpcnet.conceal(pcm)] replaces classical PLC
                    |
                    +-- CELT path: [decode_with_ec()] --> [decode_lost()]
                            |
                            +-- FRAME_PLC_NEURAL: 16kHz lpcnet.conceal() -->
                            |   sinc upsample 48kHz --> cross-fade (Integration Point 4)
                            +-- good frame: [update_plc_state()] feeds 48kHz→16kHz to
                                lpcnet.update() (Integration Point 4)

  Model weights loaded via:
    [OpusDecoder::set_dnn_blob(data)] --> lpcnet.load_model(data) (Integration Point 2)
```

## 5. Testing Strategy

### 5.1 Preserve Existing Tests

**Critical requirement**: All 280/280 bit-exact tests MUST pass with `dnn` feature disabled AND enabled (when no DNN blob is loaded). The neural PLC only activates when:
1. `feature = "dnn"` is enabled at compile time
2. Model weights have been loaded via `set_dnn_blob()`
3. The `loaded` flag is true in `LPCNetPLCState`

When weights are not loaded, all code paths fall through to classical PLC, so bit-exactness is preserved.

**Verification command**:
```bash
# Without DNN feature (must match current behavior exactly)
cargo test
cargo run --bin mdopus-compare -- roundtrip assets/test.wav

# With DNN feature but no weights loaded (must also match)
cargo test --features dnn
cargo run --features dnn --bin mdopus-compare -- roundtrip assets/test.wav
```

### 5.2 DNN-Specific Tests

**Unit tests** (in each modified module):
- `PitchDNNState` with loaded weights produces non-zero pitch estimates
- `LPCNetPLCState::update()` then `conceal()` produces non-silent output
- Sinc resampler 16kHz→48kHz roundtrip quality check
- `set_dnn_blob()` with invalid data returns error, does not panic

**Integration tests** (new test binary or extension to harness):
- Decode with artificial packet loss, compare neural PLC output vs C reference with same weights
- Verify FARGAN synthesize_int output matches C reference given same features and state

**Differential testing against C reference**:
- The comparison harness already links the C reference. Add a `--dnn-blob` flag to load weights into both Rust and C decoders, then compare PLC output sample-by-sample.
- This requires building the C reference with `ENABLE_DEEP_PLC` -- check `tests/harness/build.rs` for how to add the flag.

### 5.3 Feature Flag Matrix

| Config | Expected behavior |
|--------|-------------------|
| No `dnn` feature | Identical to current code. No DNN types in decoder state. |
| `dnn` feature, no blob loaded | `lpcnet.loaded == false`. All PLC falls through to classical. Bit-exact. |
| `dnn` feature, blob loaded, no loss | `lpcnet.update()` called on good frames. Decode output unchanged. |
| `dnn` feature, blob loaded, with loss | Neural PLC replaces classical PLC output. Not bit-exact vs classical (expected). |

## 6. Risks and Open Questions

### 6.1 Needs Arthur's Input

1. **Model weights distribution**: The C reference supports two paths:
   - Compiled-in weights (`#ifndef USE_WEIGHTS_FILE`) -- weights baked into the binary via generated C arrays
   - Runtime-loaded weights (`OPUS_SET_DNN_BLOB_REQUEST`) -- loaded from a binary blob at runtime

   For the Rust port, runtime loading is the obvious choice (matches `load_model()` APIs already present). But: **where do we get the weight blob for testing?** The xiph/opus repo has a `download_model.sh` script in `reference/dnn/`. Do we want to vendor a weight file, download it in CI, or skip DNN tests when weights are unavailable?

2. **OSCE (Opus Speech Coding Enhancement)**: The C reference has `ENABLE_OSCE` which runs LACE/NoLACE neural enhancement on SILK decoder output. This is separate from PLC and involves additional DNN modules (LACE, NoLACE) that are NOT ported yet. The `SilkDecoderState` has a placeholder comment at line 188. Should OSCE be part of this integration or deferred to a separate HLD?

3. **DRED encoder**: The C reference has `dred_encoder.c/h` for encoding redundancy features. This is NOT ported. The decoder-side DRED FEC path (feeding features to LPCNet PLC) can be wired up without the encoder, but testing requires either a C-encoded DRED stream or synthetic features. Should DRED support be deferred entirely?

4. **Stereo PLC**: The C reference does mono neural PLC only ("For now, we just do mono PLC") and copies to the second channel for stereo. This is a significant quality limitation. Confirm this is acceptable.

5. **Complexity gating**: The C reference gates neural PLC on `complexity >= 5` for CELT and on `enable_deep_plc` for SILK. Should we expose complexity as a decoder CTL (it exists in the C reference but is not in our Rust `OpusDecoder` yet)?

### 6.2 Technical Risks

1. **Signature threading is invasive**: Adding `lpcnet` parameters to `silk_decode`, `silk_decode_frame`, `silk_plc`, and the CELT functions changes core function signatures. Using `#[cfg(feature = "dnn")]` on individual parameters is awkward in Rust. The `DnnPlcArg` type alias approach (Section 3.4) is the cleanest, but still requires touching every call site. Alternative: use a trait object or a newtype wrapper to avoid signature changes -- but this adds indirection.

2. **Float contamination**: The DNN modules use f32 exclusively. The SILK/CELT core uses fixed-point i16/i32. The boundary is at `lpcnet.update(pcm: &[i16])` and `lpcnet.conceal(pcm: &mut [i16])`, which handle conversion internally. The sinc resampler in the CELT path operates in f32 (matching C). This mixing of float and fixed-point is intentional and matches the C reference, but need to verify no precision issues on the boundaries.

3. **State size**: `LPCNetPLCState` is large (~200KB including the FARGAN model, GRU states, PCM buffers, and FEC array). This is allocated once per decoder instance. For embedded/constrained environments this may be prohibitive, but behind the feature flag it's opt-in.

4. **Weight blob compatibility**: The binary weight format parsed by `parse_weights()` must match exactly between the C reference export tools and our parser. Any version mismatch will cause silent corruption or init failure. The `WEIGHT_BLOB_VERSION` constant (0) in `core.rs:31` must match.

### 6.3 Out of Scope

- DRED encoder (not ported)
- OSCE/LACE/NoLACE (not ported)
- QEXT (not ported)
- Bandwidth extension (BWE) features
- Model training or weight generation
