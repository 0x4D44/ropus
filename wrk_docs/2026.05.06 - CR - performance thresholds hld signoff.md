# Code Review - Performance Thresholds HLD Signoff

Date: 2026.05.06

Scope: supervisor review of `/home/md/language/ropus/wrk_docs/2026.05.06 - HLD - performance release thresholds.md`.

## Decision

Signed off for implementation with calibration constraints.

The next slice should make the performance lane's release claim explicit and
machine-checkable. It should not optimize codec code, expand vectors, or turn a
guessed global ratio into a flaky failure.

## Required Implementation Contract

Implementation is approved for:

- a benchmark policy/profile model derived from CLI options,
- explicit report/JSON/HTML/help wording for claimed vs unclaimed performance
  coverage,
- release-thresholded failure semantics for non-quick `--release-preflight`,
- unit-tested threshold classification using synthetic rows,
- oracle matrix updates,
- measured baseline notes in a doc or journal.

Required profile behavior:

- default full-test with benchmarks: `observed-warn-only`;
- default full-test with `--skip-benchmarks`: `not-claimed`;
- `--quick --release-preflight`: `not-claimed`;
- non-quick `--release-preflight`: `release-thresholded`;
- non-quick `--release-preflight --skip-benchmarks`: FAIL, because it disables
  a claimed release-performance gate.

Release-thresholded failures must include:

- benchmark build failure,
- missing `ropus-compare` binary after build,
- crashed benchmark rows,
- parser failure or missing encode/decode timings for required rows,
- missing required WAV fixtures,
- zero gateable rows,
- ratio values exceeding tracked per-vector/per-operation severe thresholds.

## Threshold Constraints

- Do not use the existing global `1.15x` WARN threshold as a release-fail
  threshold.
- Use same-run Rust/C ratios, not absolute milliseconds.
- Use per-vector/per-operation thresholds.
- Thresholds may live in a Rust const table or a tracked data file. Prefer the
  lower-complexity option for this slice; avoid adding dependencies unless the
  implementation really needs them.
- Threshold values must be tied to a measurement note. If the local machine
  cannot provide a full multi-sweep baseline, the implementation must say so
  and use conservative threshold data clearly labeled as initial calibration.
- The existing `1.15x` signal may remain WARN-only for default/local runs.

set. It does not prove broad performance behavior across platforms, power
profiles, WebM/container paths, DTX/FEC/PLC, projection, or real-world corpus
decode breadth.
