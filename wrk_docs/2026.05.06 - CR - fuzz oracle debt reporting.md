## Findings

1. Minor - `/home/md/language/ropus/tests/fuzz/fuzz_targets/oracle.rs:185`

   `DecodeOracleClassCounts::format_report` emits a comma-separated header and comma-separated rows, but the `asserts` and `does_not_assert` fields contain unescaped commas. The current ignored test and intended operator output are still deterministic and readable by humans, so this is not a blocker. The risk is that the report looks machine-parseable as CSV while producing variable-width rows for any consumer that splits on commas.

   Suggested fix before treating this as a durable machine contract: either rename the section to a human table/list format, use a delimiter not present in prose, or quote/escape fields as valid CSV.
