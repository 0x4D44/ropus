# ropus — HLD: deferred-work closeout

**Date:** 2026-04-19
**Status:** Draft. Acceptance criteria for Stages 6–8 superseded by
`2026.04.19 - HLD - stages-6-8-relaxed-oracles.md` (bit-exact oracle

**Author:** Claude (with Arthur)

## Goal

Close the deferred-work gaps identified during the 2026-04-18 audit, in priority order. Ship in small, independently landable steps; stop when the value/cost curve flattens.

## Priority order (fixed)

1. Ambisonic matrix data — P0, silent corruption
2. CVBR padding verification — P1, cheap check
3. Real-world bitstream corpus — P2, correctness gate
4. Analysis / tonality detection — P3, encoder quality
5. DNN wiring — P3, encoder quality + unlock for #6
6. OSCE / DRED — P4, advanced features
7. Platform SIMD (ARM NEON, x86 AVX) — P5, long-horizon optimisation
8. Fuzzer cadence — P6, operational hygiene
9. Independent CTL plumbing — P7, small standalone gaps
10. DNN/OSCE-adjacent CTL plumbing — P8, ships with #5 / #6

## Non-goals

- No migration, rollback, or compatibility shims. Dev environment — just change the code.
- No feature flags. Everything ships on by default or not at all.
- No new env vars. Existing env vars stay as-is; do not add more.
- No public-API refactor alongside any of these. Each item is the smallest change that closes its gap.
- No perf workstream here — separate agent owns it and we are at C-parity already.
- No attempt to ship all 10 at once. Items are independently landable; reassess after each.

## Cross-cutting constraints

Every item below must satisfy:
- No behavioural regression on `cargo test -p ropus` or `cargo test -p ropus-harness`.
- No new warnings on `cargo build` / `cargo doc`.
- Bit-exact output preserved on all existing test vectors.
- Touched crates get a patch-version bump in their `Cargo.toml`.

**Coordination with concurrent work.** A separate perf agent is actively editing `ropus/src/celt/simd.rs` and `ropus/src/celt/vq.rs` (see `wrk_journals/2026.04.18 - JRN - celt-encoder-unchecked-indexing.md`). `ropus/src/opus/encoder.rs` is also modified in the working tree. Items #4, #5, #6, and #9 all touch `opus/encoder.rs`; coordinate with the perf agent before landing. Do not pick up changes to their files or delete their uncommitted work. If a merge gets ugly, stop and ask.

## 1. Ambisonic / projection matrices (P0)

**Problem.** `ropus/src/opus/multistream_tables.rs` has FOA/SOA/TOA/4HOA/5HOA mixing tables zero-filled with a TODO. Framework passes validation but produces silent garbage for any `channel_mapping == 3` caller.

**Fix.** Port the data tables verbatim from `reference/src/mapping_matrix_data.h`. This is a data port, not a logic port — shape, ordering, and Q-format already match because the Rust struct layout was built from the same header. No API change.

**Files.**
- `ropus/src/opus/multistream_tables.rs` — replace the ten zero-filled arrays (five orders × mixing + demixing) with real coefficients. Verified the helpers at `ropus/src/opus/multistream.rs:1858-1879` consult all of them, including a non-zero gain parameter on `SOA_DEMIXING_DATA` — none are vestigial.

**Acceptance.**
- New harness target `ropus-compare projection-roundtrip <foa.wav>` encodes + decodes a 4-channel FOA input against the C reference. Byte-for-byte encode match; sample-for-sample decode match.
- Add one fixture under `tests/vectors/ambisonic/` (a short FOA sine sweep).

**Size.** S. Data transcription plus one new harness command.

**Fallback (if data port blocks).** Until the tables land, make `opus_projection_encoder_create` and `opus_projection_decoder_create` return `OPUS_UNIMPLEMENTED` when `channel_mapping == 3`. Silent garbage is the worst failure mode; loud failure is acceptable as a stopgap.

## 2. CVBR padding verification (P1)

**Problem.** `OpusEncoder::set_vbr_constraint` (`ropus/src/opus/encoder.rs:2777`) accepts and stores the flag, but the original audit didn't confirm the bitrate-padding path actually consumes it.

**Update from verification pass.** The flag *is* wired — `opus/encoder.rs:2452` passes it through to the CELT encoder (`celt.ctl(CeltEncoderCtl::SetVbrConstraint(self.vbr_constraint))`), and there's an existing CELT-only-path test at line 4537. So this item is most likely already done; the remaining task is confirmation, not implementation.

**Fix.**
1. Read the CELT encoder's handling of `SetVbrConstraint` and trace where it shapes output size.
2. Add one output-size-assertion test on the Opus encoder layer that confirms CVBR-vs-VBR produces the expected constrained-band behaviour at a given bitrate.
3. If any gap is confirmed, port the missing branch from `reference/src/opus_encoder.c`.

**Files.** Read-only pass first; one new test in `ropus/src/opus/encoder.rs` under existing `#[cfg(test)]`. Edits to the encoder body only if a gap is confirmed.

**Acceptance.** Green test asserting CVBR output-size behaviour.

**Size.** XS.

## 3. Real-world bitstream corpus (P2)

**Problem.** Existing conformance tests compare against the xiph C reference on synthetic vectors. They do not prove ropus decodes Opus files produced by other encoders in the wild (FFmpeg, WebRTC, Mozilla's `opus-tools`, Spotify, YouTube-extracted segments).

**Fix.** A new test binary under `harness/` that walks a directory of `.opus` / `.ogg` files, decodes each with both ropus and the C reference, and asserts sample-for-sample match.

**Files.**
- New: `harness/src/bin/corpus_diff.rs` — walk a directory, decode with both codecs, diff PCM.
- New: `tests/vectors/real_world/README.md` — documents provenance (which encoder produced each file) and how to fetch the set locally. Files themselves are gitignored — too big, and some have unclear redistribution rights.
- New: `tools/fetch_corpus.sh` — pulls from a curated list of public sources into `tests/vectors/real_world/`.

**Acceptance.**
- `cargo run --bin corpus_diff -- tests/vectors/real_world` returns zero exit when all files decode identically; prints a diff summary on mismatch.
- Corpus README lists at least: FFmpeg (libopus), FFmpeg (native), WebRTC endpoint capture, `opusenc` from `opus-tools`, one YouTube audio extraction, one WhatsApp voice note sample.

**Size.** S–M. The harness is a thin shell over existing decode paths; most work is corpus curation.

**Why no checked-in fixtures.** Real-world files are large and redistribution-encumbered; a fetch script keeps the repo clean and lets the corpus grow over time.

## 4. Analysis / tonality detection (P3)

**Problem.** Automatic speech/music classification is off. `OpusEncoder` relies on the caller's `OPUS_SIGNAL_*` hint; mode decisions for mixed content are materially worse than the C reference. The C source for this is `reference/src/analysis.c` (982 lines).

**Fix.** Port `reference/src/analysis.c` into `ropus/src/opus/analysis.rs` and wire it into the encoder mode-decision path. (Filed under `opus/` not `celt/` because the reference also lives at top-level `src/`, not in `celt/`.)

**Files.**
- New: `ropus/src/opus/analysis.rs` — tonality + transient detection, one struct mirroring `TonalityAnalysisState`.
- `ropus/src/opus/mod.rs` — `pub mod analysis;`.
- `ropus/src/opus/encoder.rs` — replace the stubbed-out analysis call sites with live ones. The crate-level marker at `opus/encoder.rs:5` (`"Analysis (tonality detection) is NOT included (matches DISABLE_FLOAT_API)"`) flags the scope; remove that comment when the wiring is live.

**Acceptance.**
- Bit-exact encode match against C on three new vectors: speech-over-music, speech-to-music transition, music-to-speech transition.
- Existing vectors remain bit-exact.
- No public API change; `OPUS_SIGNAL_AUTO` (the default) starts to actually do something.

**Size.** L. 982 lines of float-math-heavy C. Rust side is fixed-point but analysis runs in float in the reference — this must match.

**Risk.** Float determinism. The C reference runs analysis in float and we need bit-exact output. Compiler flags and rounding mode on the Rust side must match C's `-ffloat-store`-equivalent behaviour. Mitigate with the existing harness's byte-diff; any divergence shows up immediately.

## 5. DNN wiring (P3)

**Problem.** `ropus/src/dnn/` modules exist (core, LPCNet, FARGAN, lossgen, pitchdnn) and compile, but nothing calls them. They are inert. The encoder mode-decision path, decoder PLC path, and OSCE post-processing all have C-reference call sites that remain stubbed on the Rust side.

**Fix.** Three wiring points:

1. **Decoder PLC — neural path.** `ropus/src/celt/decoder.rs:~724` has the pitch/noise PLC. When a DNN weights blob is loaded, route to `dnn::lpcnet::plc` instead.
2. **Encoder analysis — neural classifier.** Depends on #4 landing first; the analysis struct grows a DNN-driven classifier field.
3. **DNN weights blob loading.** New API: `OpusDecoder::set_dnn_blob(&[u8])`. No env var — caller passes bytes. Default behaviour with no blob is today's fixed-point path. Weights file format matches the reference's `dnn/nnet_data.c` arena layout.

**Files.**
- `ropus/src/dnn/core.rs` — ensure the arena loader handles the full reference blob format.
- `ropus/src/celt/decoder.rs` — add the PLC dispatch branch.
- `ropus/src/opus/decoder.rs` — add `set_dnn_blob` method.
- `capi/src/ctl.rs` + `capi/src/ctl_shim.c` — wire `OPUS_SET_DNN_BLOB_REQUEST` through.

**Acceptance.**
- Decoder with no blob behaves identically to today (all existing tests green).
- Decoder with the reference weights blob matches the C reference's DNN-PLC output bit-exact on a packet-loss test vector.

**Size.** L. Blob loader is the longest lead item; everything else is plumbing.

**Why no feature flag.** The DNN modules compile unconditionally already; gating at runtime via "did the caller provide a blob" is the natural switch. Feature flags would just hide test coverage.

## 6. OSCE / DRED (P4)

**Problem.** Post-decode bandwidth extension (OSCE) and redundancy-payload PLC (DRED) are not wired. These are the "Opus 1.4+" quality features.

**Scope correction vs earlier belief.** "Modules exist but inert" is only true for FARGAN / LPCNet / lossgen / pitchdnn / core (~10k lines of real Rust already in `ropus/src/dnn/`). **OSCE and DRED have no Rust implementation at all** — they need fresh ports, not wiring:
- OSCE: `reference/dnn/{osce.c, osce_features.c, osce_config.h, osce_features.h, osce_structs.h, osce.h}` — six files.
- DRED: `reference/dnn/{dred_coding, dred_decoder, dred_encoder, dred_rdovae_dec, dred_rdovae_enc}.{c,h}` plus `dred_rdovae.h`, `dred_config.h` — ~twelve files.

**Fix.**
1. Port OSCE into `ropus/src/dnn/osce.rs` (+ helpers).
2. Port DRED into `ropus/src/dnn/dred_encoder.rs` + `ropus/src/dnn/dred_decoder.rs` (+ RDOVAE helpers).
3. Wire OSCE into `ropus/src/opus/decoder.rs` post-processing stage, gated on DNN blob presence.
4. Wire DRED into `ropus/src/opus/encoder.rs` (payload generation, `dred_duration` field) and `ropus/src/opus/decoder.rs` (payload parse + PLC consumption).
5. Wire the CTLs (see #10): `OPUS_SET_OSCE_BWE_REQUEST`, `OPUS_SET_DRED_DURATION_REQUEST`, plus getters.

**Acceptance.** Bit-exact match against C on OSCE and DRED test vectors from the reference's test suite.

**Size.** XL. Two fresh DNN ports plus wiring. Likely breaks into multiple landable steps (OSCE first as a pure decoder-side feature; DRED second because it spans encoder + decoder). Revisit the plan when we get here.

## 7. Platform SIMD (P5)

**Problem.** `wide` gives us portable SIMD at ~C parity. Platform-specific paths (ARM NEON, x86 AVX) would push above parity but aren't needed for correctness or competitive shipping.

**Scope decision.** Deferred further — not scheduled until a concrete workload justifies it. Document as "known limitation, parity acceptable" in `ropus/README.md`.

**If we do it later:**
- Per-module intrinsics files (`celt/simd_neon.rs`, `celt/simd_avx2.rs`) behind `#[cfg(target_arch)]`, not a feature flag. CPU detection is runtime via `std::is_x86_feature_detected!` / `std::arch::is_aarch64_feature_detected!`.
- Fallback to existing `wide` path if detection fails.
- No behavioural change — same output, faster.

**Size.** XL if pursued. Not scheduled.

## 9. Independent CTL plumbing (P7)

**Problem.** Five standalone CTLs accepted by the C reference aren't dispatched by `capi/`:

- `SET/GET_IGNORE_EXTENSIONS` (decoder)
- `GET_IN_DTX` (encoder)
- `SET/GET_VOICE_RATIO` (encoder)
- `SET_LFE` (encoder, multistream-internal)
- `SET_ENERGY_MASK` (encoder, multistream-internal)

**Fix.** Add each CTL to the right Rust state struct + dispatcher table + C shim switch case. Pattern is already established in `capi/src/ctl.rs`.

**Files.**
- `ropus/src/opus/encoder.rs` — `get_in_dtx`, `set_voice_ratio`/`get_voice_ratio`, `set_lfe`, `set_energy_mask` methods on `OpusEncoder`.
- `ropus/src/opus/decoder.rs` — `set_ignore_extensions`/`get_ignore_extensions` on `OpusDecoder`.
- `capi/src/ctl.rs` — add the new request-code constants and match arms in the existing `mdopus_*_ctl_*` functions.
- `capi/src/ctl_shim.c` — add the matching `case` labels to the right switch blocks.

**Acceptance.** Conformance suite exercises each new CTL without returning `OPUS_UNIMPLEMENTED`. Unit tests on the Rust side cover set-then-get round-trips.

**Size.** S. Five tiny wiring jobs, each following an existing template.

**Note.** `VOICE_RATIO` is considered semi-obsolete upstream; implement as a simple stored integer, no behavioural tie-in. Matches the reference.

## Risks

1. **Float determinism for #4** — analysis.c runs in float; bit-exact match with C depends on rounding. Existing byte-diff harness catches this immediately.
2. **DNN blob format drift** — the reference's weights blob format evolves across versions. Pin to a specific upstream revision at port time and test against that revision's weights.
3. **Real-world corpus redistribution** — some sources (Spotify/YouTube captures) have unclear redistribution rights. Fetch-script approach avoids checking them in; contributors fetch locally.
4. **CTL surface creep** — adding one CTL attracts the urge to add more. Stop at the list in #9 and #10; anything beyond is a separate decision.

## Explicitly out of scope

- Public API changes. Every fix preserves the existing `OpusEncoder` / `OpusDecoder` / multistream / repacketizer surface.
- Performance workstream — separate agent.
- Semver tightening, `#![warn(missing_docs)]`, rustdoc audit — all post-1.0.
- CI / automation — pre-1.0 stays human-driven.
- Any change motivated by "in case we need it later". If it isn't needed for the priority list above, it doesn't belong here.
