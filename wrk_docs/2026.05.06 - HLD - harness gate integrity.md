## Problem Statement

The testing improvement plan says the harness must make "green" precise and
automatable. Phase 0 specifically requires two gate fixes:

- Length mismatches must never be counted as exact byte/sample equality.
- Headline `ropus-compare encode`, `decode`, and `roundtrip` commands must exit
  non-zero when their primary comparison fails.

Today the comparison helpers treat a missing byte/sample as zero. That makes a
trailing zero on either side invisible to the stats. The headline commands also
print `FAIL` but do not convert that result into process failure, so scripts
cannot rely on exit status.

The smallest safe implementation is to keep the behavior local to
`/home/md/language/ropus/harness/src/cli.rs`, add focused helper tests, and
change only the headline command exit decision. Diagnostic commands should keep
printing rich output without being pulled into this gate change.

## Proposed API/Data Changes

Keep all implementation changes in `/home/md/language/ropus/harness/src/cli.rs`.

### CompareStats

Extend `CompareStats` so it is length-aware:

```rust
struct CompareStats {
    left_len: usize,
    right_len: usize,
    total: usize,
    matching: usize,
    first_diff_offset: Option<usize>,
    max_diff: i32,
}
```

Add small methods to centralize gate logic:

```rust
impl CompareStats {
    fn length_mismatch(&self) -> bool;
    fn is_match(&self) -> bool;
}
```

`is_match()` should be true only when both lengths are equal and no value
difference exists. `length_mismatch()` should be `left_len != right_len`.

The names `left_len` and `right_len` are deliberately neutral. The print
functions already receive labels and data slices, so they can still describe
the left side as C reference and the right side as Rust at output time.

### Comparison Algorithm

Change `compare_bytes` and `compare_samples` to:

1. Store both input lengths.
2. Compare only `a.len().min(b.len())` overlapping elements for value equality.
3. If all overlapping elements match but lengths differ, set
   `first_diff_offset = Some(overlap_len)`.
4. Keep `total = a.len().max(b.len())`.
5. Keep `matching` as the count of equal overlapping elements only.
6. Keep `max_diff` as the maximum numeric value difference observed in the
   overlap and, for non-zero extra trailing elements, versus absent zero. If the
   only mismatch is an extra trailing zero, `max_diff` may remain `0`; the
   explicit length mismatch is the failure reason.

This preserves the existing idea of "first differing offset" for print
diagnostics while making absence a first-class mismatch.

### Output

Update `print_result` and `print_sample_result` to use `stats.is_match()`
instead of treating `first_diff_offset == None` as sufficient.

When lengths differ, include both lengths in the headline failure:

```text
encode: FAIL at offset 42 (C len: 42, Rust len: 43, total: 43, matching: 42, max_diff: 0)
```

For equal lengths, the output can stay close to the current text to minimize
churn:

```text
decode: FAIL at sample 128 (total: 960, matching: 959, max_diff: 12)
```

### Headline Command Outcome

Add a tiny private outcome enum:

```rust
#[derive(Clone, Copy, Debug, Eq, PartialEq)]
enum CompareCommandOutcome {
    Pass,
    Fail,
    Skip,
}
```

Change only these functions to return it:

- `cmd_encode(...) -> CompareCommandOutcome`
- `cmd_decode(...) -> CompareCommandOutcome`
- `cmd_roundtrip(...) -> CompareCommandOutcome`

Outcome rules:

- Return `Skip` for the existing explicit unimplemented-path skips:
  `/home/md/language/ropus/harness/src/cli.rs:820`,
  `/home/md/language/ropus/harness/src/cli.rs:1050`, and
  `/home/md/language/ropus/harness/src/cli.rs:1088`.
- Return `Pass` when the primary comparison `stats.is_match()` is true.
- Return `Fail` when the primary comparison `stats.is_match()` is false.
- For `roundtrip`, the primary comparison is the final PCM roundtrip comparison
  at `/home/md/language/ropus/harness/src/cli.rs:1101`. The decode-only section
  starting at `/home/md/language/ropus/harness/src/cli.rs:1105` remains
  diagnostic output and should not decide the headline exit status in this
  slice.

Add one helper for process exit decisions:

```rust
fn exit_code_for_compare_outcome(outcome: CompareCommandOutcome) -> i32 {
    match outcome {
        CompareCommandOutcome::Pass | CompareCommandOutcome::Skip => 0,
        CompareCommandOutcome::Fail => 1,
    }
}
```

In the dispatch at `/home/md/language/ropus/harness/src/cli.rs:10314`, call the
headline command, then call `process::exit(1)` only when the helper returns
non-zero. This preserves explicit skip behavior as a successful, visible skip
instead of converting "not implemented" into a gate failure. The `SKIP` line is
already machine-readable enough for this slice because it is uppercase,
single-line, and command-prefixed; avoid inventing a broader skip protocol here.

## Test Plan

Add focused unit tests inside the existing
`#[cfg(test)] mod coverage_smoke_tests` block that starts at
`/home/md/language/ropus/harness/src/cli.rs:8908`, or rename/split the module if
that reads poorly after adding pure helper tests. Keeping tests in the same file
is the smallest change and avoids exposing test-only internals.

Required helper tests:

- Equal byte slices and equal sample slices:
  - `is_match() == true`
  - `matching == total`
  - `left_len == right_len`
- Same-length value mismatch:
  - `is_match() == false`
  - `first_diff_offset` points at the differing element
  - `matching < total`
- Left has trailing zero:
  - `compare_bytes(&[1, 0], &[1])` fails
  - `first_diff_offset == Some(1)`
  - `matching == 1`
  - `total == 2`
- Right has trailing zero:
  - `compare_samples(&[1], &[1, 0])` fails
  - same shape as above
- Left/right empty asymmetry:
  - `compare_bytes(&[], &[0])` fails
  - `compare_samples(&[0], &[])` fails
  - neither case can produce `matching == total`
- Exit decision helper:
  - `Pass -> 0`
  - `Skip -> 0`
  - `Fail -> 1`

Avoid subprocess command tests in this slice unless an existing command-test
pattern is introduced elsewhere first. The command paths require fixtures and
FFI setup, while the actual bug is pure helper and dispatch decision logic.

## Risks And Tradeoffs

- `max_diff == 0` for an extra trailing zero can look odd in a failure. This is
  acceptable if the output also shows both lengths and `is_match()` treats
  length mismatch as failure. Forcing `max_diff` to a sentinel value would mix
  numeric value difference with existence difference.
- Changing `cmd_encode`, `cmd_decode`, and `cmd_roundtrip` return types is
  small because they are only dispatched directly from `main` today. If another
  workstream starts calling them before implementation, refresh call sites first.
- This design intentionally leaves diagnostic commands alone. Some diagnostics
  currently use `compare_bytes` or `compare_samples`; they will get more honest
  length-aware printouts, but they should not automatically become process
  gates in this slice.
- Treating `Skip` as exit `0` preserves fresh-clone and unimplemented-path
  ergonomics. It does not solve release preflight strictness; that belongs to
  the separate preflight workstream.
- The current test module is named `coverage_smoke_tests`, which is not a great
  home for pure comparison helper tests. Renaming it would add noise in a large
  file. The pragmatic choice is to add the focused tests there or create one
  adjacent `compare_stats_tests` module under the same `#[cfg(test)]`.
