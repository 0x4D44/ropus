# CELT Prefilter (run_prefilter) Rewrite — V2

*V1 reviewed by: correctness reviewer, Rust implementation reviewer, devil's advocate.*
*V2 incorporates all review findings.*

## Problem

The Rust `run_prefilter` in `src/celt/encoder.rs:1424` does not match the C reference in `reference/celt/celt_encoder.c:1411`. This causes ALL non-silence CELT frames to produce different output. Silence is unaffected because zero input produces zero output regardless of the filter.

The prefilter is an analysis comb filter that removes pitch periodicity from the signal before CELT encoding. The decoder applies the inverse (synthesis) filter. For bit-exact output, the encoder's analysis filter must match exactly.

## What the C code does

The C `run_prefilter` has three phases:

### Phase 1: Pitch estimation
Build a work buffer `pre[]` from `prefilter_mem` (history) + current frame. Estimate pitch via one of three paths:

- **Tone path** (toneishness > 0.99): Compute pitch directly from tone frequency. Applies aliasing correction (mirrors frequencies above π). Sets gain = 0.75 (Q15 = 24576).
- **Standard path** (enabled && complexity >= 5): `pitch_downsample` → `pitch_search` → `remove_doubling`. Scale gain by 0.7. Attenuate for high loss_rate.
- **Disabled path** (else): gain = 0, pitch = COMBFILTER_MINPERIOD.

Apply adaptive threshold to decide if prefilter is on or off. Threshold adjusts based on pitch continuity, available bytes, and previous gain. Gain hysteresis snaps to previous gain if within 0.1. Quantize gain to 3-bit index (0-7), dequantize as `gain1 = 3072 * (qg + 1)` in Q15.

Note: QEXT_SCALE is a no-op in our build (ENABLE_QEXT not defined). The analysis->valid path is compiled out in FIXED_POINT builds. Neither needs handling.

### Phase 2: Comb filter application

This is the part that modifies the `in[]` buffer. It happens in four steps:

1. **Copy overlap from `in_mem`** — NOT from `prefilter_mem`. `in_mem` stores the tail of the *filtered* signal from the previous frame. Also clamp `prefilter_period >= COMBFILTER_MINPERIOD`.

2. **Compute before-energy**: Sum of absolute values (>>12) of `in[overlap..overlap+N]`.

3. **Apply comb filter** with NEGATIVE gains (analysis = inverse of synthesis), in two segments:
   - First `offset = shortMdctSize - overlap` samples: constant filter using old period/gain/tapset. Window=NULL, overlap=0. (For 48kHz, offset=0, so this segment is empty.)
   - Remaining `N - offset` samples: crossfade from old period/gain/tapset to new pitch_index/gain1/prefilter_tapset. Uses `mode.window` for the crossfade.
   
   The filter is ALWAYS applied (even when gain1=0) because the crossfade from old_gain to 0 must be smooth.

4. **Cancel check**: Compute after-energy same way as before.
   - Mono: if `after[0] > before[0]`, cancel.
   - Stereo: if either channel worsened beyond threshold (`0.25*gain1*before[c] + 0.01*before[1-c]`), OR neither channel improved enough, cancel.
   
   If cancelled: revert `in[]` from unfiltered `pre[]`, apply one more comb_filter segment crossfading old_gain → 0, then set gain1=0, pf_on=0, qg=0.

### Phase 3: Memory update
- `in_mem` ← last `overlap` samples of filtered `in[]`
- `prefilter_mem` ← last `max_period` samples of UNFILTERED `pre[]` (if N > max_period: direct copy; else: shift old memory left by N, append N new samples from `pre[]`)

The two memories serve different purposes: `in_mem` gives filtered overlap for the next frame's MDCT window, while `prefilter_mem` gives unfiltered history for the next frame's pitch search and comb filter reference.

## What the Rust code gets wrong

Six bugs, in order of impact:

| # | Bug | Location | Effect |
|---|-----|----------|--------|
| 1 | Comb filter skipped when `gain1 == 0` | encoder.rs:1552 | No crossfade from old_gain → 0; frame boundary discontinuity |
| 2 | Overlap copied from `prefilter_mem` instead of `in_mem` | encoder.rs:1456, 1828 | Wrong overlap signal; also affects disabled path |
| 3 | `prefilter_mem` updated from filtered `input[]` instead of unfiltered `pre[]` | encoder.rs:1610 | Corrupts pitch search history for next frame |
| 4 | Missing cancel_pitch check | (absent) | Filter can worsen signal without being reverted |
| 5 | Wrong gain dequantization formula | encoder.rs:1544 | `(qg*3+1)*32` ≠ C's `3072*(qg+1)`. Off by orders of magnitude. |
| 6 | Pitch estimation details differ | encoder.rs:1448-1546 | complexity >= 1 (should be >= 5); missing ×0.7 scaling, loss_rate attenuation, adaptive threshold, gain hysteresis, tone path |

Additionally: `in_mem` is never written anywhere in current code (always zeros).

## Design

Replace `run_prefilter` with a faithful port of the C function. No struct changes needed — `in_mem` and `prefilter_mem` already exist with correct sizes.

### Data flow

```
pre[] = prefilter_mem ++ input[overlap..]    (unfiltered work buffer)
                                              │
                              pitch_search ◄──┘
                                   │
                            pitch_index, gain1
                                   │
in[0..overlap] = in_mem            │  (filtered overlap from prev frame)
                    │              │
                    ▼              ▼
in[overlap..] = comb_filter(pre[max_period..], -old_gain → -gain1)
                    │
            cancel check: compute before/after energy
            if worse → revert from pre[], crossfade to gain=0
                    │
in_mem       ◄── in[N..N+overlap]      (save filtered tail)
prefilter_mem ◄── pre[N..N+max_period]  (save unfiltered tail)
```

### Function signature

Keep existing signature. Activate the four currently-ignored parameters (remove `_` prefix from `tf_estimate`, `nb_available_bytes`, `tone_freq`, `toneishness`).

### Implementation plan

#### Section A: Pitch estimation

Three paths, matching C lines 1447-1546:

**Path 1: Tone detection** (C lines 1447-1467)
```
if enabled && toneishness > QCONST32(0.99, 29):
    if tone_freq >= QCONST16(3.1416, 13):    // alias correction
        tone_freq = QCONST16(3.141593, 13) - tone_freq
    multiple = 1
    while tone_freq >= multiple * QCONST16(0.39, 13): multiple++
    if tone_freq > QCONST16(0.006148, 13):
        pitch_index = min((51472*multiple + tone_freq/2) / tone_freq, COMBFILTER_MAXPERIOD-2)
    else:
        pitch_index = COMBFILTER_MINPERIOD
    gain1 = QCONST16(0.75, 15)    // NOTE: 0.75, not 0.7
```

**Path 2: Standard pitch search** (C lines 1468-1494, complexity >= 5)
```
pitch_downsample(pre, pitch_buf, ...)
pitch_search(pitch_buf + max_period/2, pitch_buf, N, max_period - 3*min_period, &pitch_index)
pitch_index = max_period - pitch_index
gain1 = remove_doubling(pitch_buf, max_period, min_period, N, &pitch_index, ...)
if pitch_index > max_period - 2: pitch_index = max_period - 2
gain1 = MULT16_16_Q15(QCONST16(0.7, 15), gain1)    // scale by 0.7
if loss_rate > 2: gain1 /= 2
if loss_rate > 4: gain1 /= 2
if loss_rate > 8: gain1 = 0
```

**Path 3: Disabled** (else): gain1 = 0, pitch_index = COMBFILTER_MINPERIOD

**Threshold and quantization** (C lines 1506-1546):
```
pf_threshold = QCONST16(0.2, 15)
if abs(pitch_index - prefilter_period) * 10 > pitch_index:
    pf_threshold += QCONST16(0.2, 15)
    if tf_estimate > QCONST16(0.98, 14): gain1 = 0
if nb_available_bytes < 25: pf_threshold += QCONST16(0.1, 15)
if nb_available_bytes < 35: pf_threshold += QCONST16(0.1, 15)
if prefilter_gain > QCONST16(0.4, 15): pf_threshold -= QCONST16(0.1, 15)
if prefilter_gain > QCONST16(0.55, 15): pf_threshold -= QCONST16(0.1, 15)
pf_threshold = max(pf_threshold, QCONST16(0.2, 15))    // hard floor

if gain1 < pf_threshold:
    gain1 = 0; pf_on = 0; qg = 0
else:
    if abs(gain1 - prefilter_gain) < QCONST16(0.1, 15):    // hysteresis
        gain1 = prefilter_gain
    qg = ((gain1 + 1536) >> 10) / 3 - 1
    qg = clamp(qg, 0, 7)
    gain1 = QCONST16(0.09375, 15) * (qg + 1)    // = 3072 * (qg + 1) in Q15
    pf_on = 1
```

#### Section B: Comb filter application

Matches C lines 1549-1590. ALWAYS runs (not gated by gain1 > 0).

```rust
for c in 0..cc {
    let offset = mode.short_mdct_size - overlap;  // 0 for 48kHz
    st.prefilter_period = max(st.prefilter_period, COMBFILTER_MINPERIOD);
    
    // 1. Copy filtered overlap from in_mem
    input[c*(N+ov)..c*(N+ov)+ov].copy_from_slice(&st.in_mem[c*ov..(c+1)*ov]);
    
    // 2. Compute before-energy
    before[c] = sum of abs(input[c*(N+ov)+ov+i] >> 12) for i in 0..N
    
    // 3. Apply comb filter (two segments, negative gains)
    if offset > 0:
        comb_filter(output=in[ov..ov+offset], input=pre[max_period..],
            T0=old_period, T1=old_period, N=offset,
            g0=-old_gain, g1=-old_gain,
            tapset0=old_tapset, tapset1=old_tapset,
            window=NULL, overlap=0)
    
    comb_filter(output=in[ov+offset..ov+N], input=pre[max_period+offset..],
        T0=old_period, T1=pitch_index, N=N-offset,
        g0=-old_gain, g1=-gain1,
        tapset0=old_tapset, tapset1=new_tapset,
        window=mode.window, overlap=overlap)
    
    // 4. Compute after-energy
    after[c] = sum of abs(input[c*(N+ov)+ov+i] >> 12) for i in 0..N
}

// 5. Cancel check
cancel_pitch = false
if cc == 2:
    // stereo threshold with cross-channel coupling
    ...
else:
    if after[0] > before[0]: cancel_pitch = true

if cancel_pitch:
    for c in 0..cc:
        // Revert: copy unfiltered pre[] back
        input[c*(N+ov)+ov..c*(N+ov)+ov+N] = pre[c][max_period..max_period+N]
        // Apply one final crossfade: old_gain → 0
        comb_filter(in[ov+offset..], pre[max_period+offset..],
            old_period, pitch_index, overlap, -old_gain, 0,
            old_tapset, new_tapset, window, overlap)
    gain1 = 0; pf_on = 0; qg = 0
```

#### Section C: Memory update

```rust
for c in 0..cc {
    // Save filtered tail for next frame's overlap
    st.in_mem[c*ov..(c+1)*ov] = input[c*(N+ov)+N .. c*(N+ov)+N+ov]
    
    // Save unfiltered history for next frame's pitch search
    if N > max_period:
        prefilter_mem[c*max_period..(c+1)*max_period] = pre[c][N..N+max_period]
    else:
        // Shift old memory left by N, append N new samples
        prefilter_mem[c*mp..c*mp+mp-N] = prefilter_mem[c*mp+N..c*mp+mp]  // MOVE
        prefilter_mem[c*mp+mp-N..c*mp+mp] = pre[c][max_period..max_period+N]  // COPY
}
```

#### Section D: Disabled path

When `!enabled || complexity < 1` (C lines 1495-1498, gain1=0 path):

The disabled path still needs correct overlap and memory management. It follows the same Phase 2 + Phase 3 logic but with gain1=0, which means the comb filter crossfade from old_gain to 0 still runs. Do NOT early-return before the comb filter phase.

### comb_filter: no changes needed

Review confirmed the g10/g11 shadow bug does NOT exist — line 264 correctly defines `g11`. The comb_filter function is structurally correct. Callers must pass negative gains for the analysis filter.

### Borrow checker: no obstacles

Review confirmed all operations are safe:
- Reading `st.in_mem`/`st.prefilter_mem` while writing `input` — different allocations
- Local `pre` Vec as comb_filter input, `input` as output — no aliasing
- Per-channel processing in loops — natural ownership
- Memory updates via `&mut st` — straightforward

Use `Vec<i32>` for the `pre[]` work buffer (matches existing allocations in the function). For stereo, allocate per-channel in the loop.
