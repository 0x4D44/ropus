# Coverage Improvement Plan

Date: 2026-04-08
Repo: `C:\language\mdopus`
Target command: `cargo +nightly llvm-cov --branch --summary-only`

## Goal

Drive line, function, region, and branch coverage upward with the smallest test surface that materially reduces the uncovered area. The long-term aspirational target is 98%, but the baseline is 48.94% lines / 35.10% branches, so the plan is staged around real measurable gains.

## Stage 0: Baseline And Documentation

Status: done

Outputs:

- baseline coverage run
- coverage review report
- journal entry
- this plan

Exit criteria:

- nightly branch coverage runs cleanly
- current percentages and hotspots are documented

## Stage 1: Cheap Isolated Wins

Focus:

- `src\celt\ec_ctx.rs`
- `src\silk\common.rs`
- small missing branches in `src\opus\encoder.rs` and `src\opus\decoder.rs` setters/getters

Why:

- These files have clear local APIs.
- Tests here are cheap and deterministic.
- `ec_ctx` is severely undercovered and should move immediately.

Test ideas:

- trait dispatch through `EcCoder` for both `RangeEncoder` and `RangeDecoder`
- explicit `#[should_panic]` tests for wrong-direction default trait methods
- utility sweeps for `silk_rand`, `silk_sigm_q15`, `silk_lin2log`, `silk_log2lin`, bandwidth expanders, inverse/division helpers, sort/sum-squared helpers
- invalid-argument setter tests for Opus encoder/decoder CTLs

Success signal:

- remove the near-zero coverage outlier in `celt\ec_ctx.rs`
- raise `silk\common.rs` materially without touching codec behavior

## Stage 2: Single-Stream Opus Runtime Paths

Focus:

- `src\opus\encoder.rs`
- `src\opus\decoder.rs`
- indirectly `src\silk\encoder.rs`, `src\silk\decoder.rs`, `src\celt\encoder.rs`, `src\celt\decoder.rs`

Why:

- This is the highest uncovered-line concentration in the core product path.
- End-to-end tests exercise multiple modules at once.

Test matrix:

- mono and stereo
- silence vs non-silence PCM
- audio vs voip vs restricted-lowdelay applications
- forced `MODE_SILK_ONLY`, `MODE_CELT_ONLY`, `MODE_HYBRID`
- 2.5/5/10/20/40/60 ms frame durations where valid
- VBR and CBR
- FEC enabled + packet-loss percentage
- DTX enabled + repeated low-activity frames
- decode, decode24, decode_float
- PLC (`decode(None, ...)`) after valid packets

Success signal:

- `opus\encoder.rs` and `opus\decoder.rs` move out of the 30-50% band
- `silk\encoder.rs` and `silk\decoder.rs` gain large indirect coverage

## Stage 3: Direct SILK Path Tests

Focus:

- `src\silk\encoder.rs`
- `src\silk\decoder.rs`

Why:

- If Stage 2 leaves major gaps, the remaining dark surface will be low-level SILK control-flow not reliably triggered through top-level Opus tests.

Test matrix:

- direct `silk_encode()` with mono, stereo, mono->stereo, stereo->mono transitions
- payload sizes 10/20/40/60 ms
- resampler path coverage with API/internal sample-rate changes
- reduced-dependency path
- prefill path
- packet-loss and LBRR/LBRR-skip behavior
- direct `silk_decode()` with normal decode, packet loss, and LBRR decode flags

Success signal:

- large reductions in missed lines/branches inside `silk\encoder.rs` and `silk\decoder.rs`

## Stage 4: Multistream And Packet Topology Sweep

Focus:

- `src\opus\multistream.rs`
- `src\opus\repacketizer.rs`

Why:

- These files encode branchy channel-layout and packet-layout logic that single-stream tests do not cover.

Test matrix:

- family 0, 1, 2, 255 constructors
- non-trivial mappings
- invalid layout permutations
- ambisonics/projection helpers
- encode/decode with non-zero multi-channel PCM
- bitrate, complexity, reset, PLC, and gain control roundtrips

Success signal:

- branch coverage in `opus\multistream.rs` rises substantially above current 48.08%

## Stage 5: DNN Sweep

Focus:

- `src\dnn\core.rs`
- `src\dnn\lpcnet.rs`
- `src\dnn\fargan.rs`
- optionally `src\dnn\pitchdnn.rs`

Why:

- The DNN modules are a major remaining line deficit after codec-path tests.

Test matrix:

- state-machine initialization/reset branches
- bad-shape/bad-dimension validation
- small deterministic forward passes
- boundary and saturation paths
- helper branches that are currently compiled but never entered

Success signal:

- DNN modules stop dominating the missed-line count

## Stage 6: Hard Branch Closure

Focus:

- rerun full branch coverage after every meaningful batch
- identify the remaining low-coverage files only after Stages 1-5

Why:

- branch coverage will lag line coverage late in the campaign
- final work should be data-driven, not guessed

Tasks:

- inspect latest worst files
- write targeted tests for uncovered conditionals rather than broad new helpers
- keep commits atomic by stage

## Measurement Loop

For each implementation batch:

1. Run scoped tests for touched modules.
2. Run `cargo +nightly llvm-cov --branch --summary-only`.
3. Capture totals and note delta.
4. Commit if the batch is coherent and passing.

## Practical Risk Notes

- The 98% target is far from baseline and may prove unattainable without very broad test authoring across most modules.
- Branch coverage will likely be the limiting metric even if line coverage rises quickly.
- The external-agent requirement is currently blocked by sandbox networking/permissions, so execution falls back to local work unless the environment changes.

## Initial Implementation Order

1. `celt\ec_ctx.rs` trait coverage
2. high-yield end-to-end tests in `opus\encoder.rs`
3. complementary decode-path tests in `opus\decoder.rs`
4. direct helper sweep in `silk\common.rs`
5. deeper SILK runtime tests if overall coverage is still dominated by `silk\encoder.rs` / `silk\decoder.rs`
