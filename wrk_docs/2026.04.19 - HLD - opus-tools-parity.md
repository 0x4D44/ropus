# HLD: opus-tools feature parity for ropus{enc,dec,info,play}

Author: Arthur + Claude
Date: 2026-04-19
Status: DRAFT (v2 — review pass applied)

Follow-up to `2026.04.19 - HLD - ropus-cli-split.md`, which explicitly left
feature-parity flags out of scope. This HLD closes those gaps.

## Goal

Bring `ropusenc`, `ropusdec`, `ropusinfo`, and `ropusplay` to behavioural
parity with the `opus-tools` binaries they mirror. "Parity" = the CLI
flags an opus-tools user reaches for produce the same result, on the same
inputs, in a form compatible with downstream tooling.

## Non-goals

- Bit-exact argv-for-argv reproduction of opus-tools output text. Flag set
  and behaviour match; banner and log text remain the ropus house style.
- Backwards compatibility with any existing ropus-tools invocation.
  Development sandbox; no migration story needed.
- Feature flags / Cargo features. Everything lands unconditionally.
- Environment variables. The two `BUILD_*` rustc-envs each `build.rs`
  already emits are the ceiling.
- Interactive playback controls in `ropusplay` (seek, pause, volume
  during play). Opus-tools' `opusdec | aplay` flow has none of this; a
  real player is a separate product, not a CLI parity item.
- Chained multi-stream Ogg Opus on decode. Rare in practice; adds
  per-stream decoder reset and granule bookkeeping for little win.
  Explicitly deferred.
- Channel mapping families 1/2 (surround, ambisonics) — already out of
  scope repo-wide.

## Shared primitive: OpusTags read/write

The single biggest lift, and the one that unblocks three tools at once.
Everything else is small.

### Current state

`ropus-tools-core/src/container/ogg.rs:31` writes an empty-comment
`OpusTags` packet with only the vendor string. `read_opus_tags` at
line 72 verifies the magic and throws the payload away.

### Design

Replace both with a proper R/W model. One struct, one parser, one
serializer, shared by encode/decode/info.

```rust
// ropus-tools-core/src/container/ogg.rs

#[derive(Debug, Default, Clone)]
pub struct OpusTags {
    pub vendor: String,
    /// Each entry is one `KEY=value` string, per Vorbis comment spec.
    /// Keys are case-insensitive ASCII; values are UTF-8. We store the
    /// raw "KEY=value" form and let callers split/search — matches the
    /// on-disk layout and avoids re-encoding round-trips losing order.
    pub comments: Vec<String>,
}

impl OpusTags {
    pub fn parse(data: &[u8]) -> Result<Self> { ... }
    pub fn encode(&self) -> Vec<u8> { ... }

    /// Case-insensitive lookup; returns the value portion of the first
    /// matching `KEY=value` entry.
    pub fn get(&self, key: &str) -> Option<&str> { ... }
}
```

`parse` rejects: missing magic, length fields that overrun the buffer,
non-UTF-8 vendor, non-UTF-8 comments. Lax on anything else — real files
occasionally include duplicate keys and non-standard tags; we preserve
them verbatim.

`encode` writes little-endian length-prefixed strings exactly per
RFC 7845 §5.2. No framing bit, no padding.

### Call sites

- `commands::encode` replaces `build_opus_tags(&opts.vendor)` with
  `OpusTags { vendor, comments }.encode()`, where `comments` comes from
  the new `EncodeOptions.comments: Vec<String>` field.
- `commands::decode` and `commands::info` parse the tags packet and use
  it for `--tag-*` display.
- Unit tests live next to the struct: a round-trip test (encode → parse
  → compare) and a known-bytes fixture test (parse a hand-built payload
  that matches the RFC example).

## `ropusenc` gaps

### New flags

| Flag                 | Maps to                                        |
|----------------------|------------------------------------------------|
| `--cvbr`             | `EncoderBuilder::vbr_constraint(true)` (new)   |
| `--hard-cbr`         | `vbr(false)` + `vbr_constraint(false)`         |
| `--music` / `--speech` | `Signal::Music` / `Signal::Voice`            |
| `--framesize MS`     | `EncoderBuilder::frame_duration(…)`            |
| `--expect-loss PCT`  | `EncoderBuilder::packet_loss_perc(pct)` (new)  |
| `--comp N`           | clap alias: `#[arg(long = "complexity", alias = "comp")]` |
| `--downmix mono`     | PCM `(L + R) * 0.5` before resample (stereo→mono only) |
| `--serial N`         | override `OGG_STREAM_SERIAL` for this file     |
| `--artist`, `--title`, `--album`, `--genre`, `--date`, `--tracknumber`, `--comment K=V` (multi) | collected into `EncodeOptions.comments` |
| `--picture FILE`     | METADATA_BLOCK_PICTURE base64-in-comment       |
| `-` as input path    | stdin (read all to Vec, then wrap in Cursor)   |
| `-` as output path   | stdout                                         |

Dropped from the original gap list:

- `--save-range`: xiph-internal bitstream debugging; the `ropus-compare`
  harness does strict byte-diff, which is strictly better.
- `--max-delay MS`: libopus' real mapping is fiddly (forces
  `RESTRICTED_LOWDELAY` below 22.5 ms *and* caps frame duration). Users
  who want lowdelay already have `--application lowdelay`. Not worth the
  duplication.
- `--downmix stereo`: opusenc's stereo-downmix is for surround→stereo;
  we don't accept >2-channel input (channel mapping family 0 only), so
  the flag value is meaningless for us. `--downmix mono` (2→1) is the
  only case kept.

### `--framesize` buffering

The current encoder path hardcodes `FRAMES_PER_PACKET = 1` and a 20 ms
input chunking stride. Making framesize configurable is simpler than
looks: `ropus::FrameDuration` already exists and the encoder takes
full-frame PCM buffers of any supported size. The work:

1. Compute `samples_per_frame` from the selected `FrameDuration`
   (`FRAME_SAMPLES_PER_CH = 48 * ms`).
2. Replace the hardcoded `FRAME_SAMPLES_PER_CH * channels` stride with
   the computed one.
3. Size `packet_buf` at `MAX_OPUS_FRAME_BYTES` (1275, same for every
   frame duration — it's a per-frame max).
4. Tail handling stays identical: silence-pad to the chosen frame size.

The `debug_assert_eq!(MAX_PACKET_BYTES, MAX_OPUS_FRAME_BYTES * FRAMES_PER_PACKET)`
guard at `encode.rs:31` exists for this exact change. After the patch,
`FRAMES_PER_PACKET = 1` still holds (opus-tools' `--framesize` selects
the *frame* size, not multi-frame packets — those are a separate
repacketiser feature we don't expose), so the assert stays valid
unchanged.

### `--downmix`

Only `--downmix mono` is accepted (`(L + R) * 0.5` over the decoded
f32 PCM). `DecodedAudio.channels` is rewritten to 1, so resampler,
encoder build, and `OpusHead.channels` all see the post-mix count.
`OpusHead.input_sample_rate` keeps the *original* input SR (RFC 7845
§5.1: "informational; decoder does not resample based on it"), so
downmix does not touch it.

No-op when input is already mono; error on input with >2 channels
(we don't accept those anyway — this is a reinforced check).

### `--picture`

Base64-encoded METADATA_BLOCK_PICTURE structure in a
`METADATA_BLOCK_PICTURE=<base64>` comment. Same format FLAC uses.

`base64` is **not** currently a workspace dep. Hand-roll a ~30-line
base64 encoder next to the picture-packing code — matches the style
of `audio/wav.rs` (hand-rolled to avoid a dep for a tiny surface).
Decode isn't needed (we only write pictures, never parse them back).

Picture type defaults to 3 (Front Cover). Format auto-detected from
the first 12 bytes (JPEG `FFD8FFE0`, PNG `89 50 4E 47`). If detection
fails, error cleanly — don't guess.

### `--serial`

The current `OGG_STREAM_SERIAL = 0xC0DE_C0DE` constant moves from a
module-level `pub const` to the default value inside
`EncodeOptions::serial: Option<u32>`. Encode uses the option if set,
else the constant. Value is propagated to `PacketWriter::write_packet`.

### stdin / stdout

Convention: `-` means stdin/stdout. Implementation: library sentinel is
`PathBuf::from("-")`; the command function matches on it and branches:

- Input `-`: `io::stdin().read_to_end(&mut buf)?`, wrap `buf` in
  `Cursor`, hand that to symphonia's probe chain (which needs seek for
  WAV/FLAC container sniff). Memory-bounded by input size — fine for
  tools usage; a multi-GB pipe will use multi-GB RAM. Documented risk.
- Output `-`: return `io::stdout().lock()` instead of `File::create`.
  `PacketWriter` accepts any `Write`; no further change needed.

**Testability note:** sentinel-on-`PathBuf` keeps the options struct
clean but means stdin tests can't use in-process library calls — the
sentinel switches the branch inside the command. Stdin/stdout gets
shell-out integration tests only. Accepted.

### Encoder API extensions in `ropus::`

Two small additions to `ropus/src/api.rs` — both wrap internal state
that already exists:

```rust
// EncoderBuilder
pub fn vbr_constraint(mut self, on: bool) -> Self { self.vbr_constraint = Some(on); self }
pub fn packet_loss_perc(mut self, pct: u8) -> Self { self.packet_loss = Some(pct); self }
```

Range-check `packet_loss_perc` to `0..=100` at build time (libopus does
the same). Both setters flow into the existing opus encoder state in
`ropus/src/opus/encoder.rs`, which already has the fields; they're just
not exposed through the public builder today.

## `ropusdec` gaps

### New flags

| Flag                 | Behaviour                                      |
|----------------------|------------------------------------------------|
| `--float`            | Write 32-bit float WAV (IEEE 754) instead of i16 |
| `--rate HZ`          | Resample decoded PCM to `HZ` before output     |
| `--gain DB`          | Add user gain on top of header `output_gain` via `set_gain` |
| `--no-dither`        | Skip TPDF dither on i16 output                 |
| `--raw`              | Raw interleaved PCM (no WAV header)            |
| `--packet-loss PCT`  | Simulate random packet loss for PLC testing    |
| `-` as input         | stdin                                          |
| `-` as output        | stdout                                         |

Dropped `--force-wav`: opus-tools uses it to override a "stdout
defaults to raw on Windows consoles" behaviour. Our default is always
WAV regardless of sink; `--raw` is the only flip. Simpler argv, one
flag removed.

### Dither

Default on for i16 output, implicit-off for float (no bit-depth
reduction → nothing to dither). Triangular PDF, ±1 LSB, applied
per-sample pre-clamp. Same algorithm libopus' `opusdec` uses.

RNG: inline 8-line xorshift32 with a fixed seed. Dither doesn't need
crypto randomness, and a fixed seed gives reproducible output across
runs (convenient for diffing test fixtures). No new dep.

### WAV writers

The existing `write_wav_pcm16` is i16-only. `--float` needs a parallel
`write_wav_float32`:

- `fmt ` chunk size 18 (not 16) with a trailing 0-byte `cbSize`.
- Format code 3 (`WAVE_FORMAT_IEEE_FLOAT`), not 1.
- Required `fact` chunk containing sample-frame count (WAVEFORMATEX
  spec mandates it for non-PCM formats).
- `bits_per_sample = 32`, `bytes_per_sample = 4`.

Split the common header-building bits into a tiny helper and let the
two writers diverge where they must. Still no external dep — keeping
the "hand-rolled ~30 lines" pattern `audio/wav.rs` already follows.

### `--rate` pipeline position

Critical: pre-skip trim happens **before** `--rate` resample, not
after. The ropus decoder runs at 48 kHz internally and emits 312
(typical) leading samples of pre-skip. If we resample first, the
resampler smears those silence samples into real output. Correct
order:

```
packets → decode@48k → apply decoder gain (set_gain)
       → trim OpusHead.pre_skip samples@48k
       → resample 48k → --rate HZ (if set)
       → dither (i16 path only, unless --no-dither)
       → write WAV / raw
```

Also: `resample_to_48k` in `audio/resample.rs` is hardcoded-target.
Generalise to `resample(samples, from_sr, to_sr, channels)` with
identical body — rename and change the signature. Callers that want
48 kHz pass 48000 explicitly. Clearer than a hardcoded constant and
removes an argument-order footgun for encode (which needs 48 kHz) vs.
decode (which may need anything).

### `--gain`

Convert dB to Q8 fixed-point (`(db * 256.0).round() as i32`). Add to
the header's `output_gain` (also Q8 per RFC 7845 §5.1) and pass the
sum to `decoder.set_gain`. Range-check: libopus' `set_gain` accepts
`[-32768, 32767]` — i.e. ±128 dB. Error if the sum overflows that
range.

### `--packet-loss`

A simple uniform-random drop at packet-read time. Dropped packets are
replaced with a PLC call: `decoder.decode(&[], &mut out, DecodeMode::Fec)`
(ropus' decoder already supports empty-input PLC — verify in
`ropus/src/opus/decoder.rs`, otherwise extend to accept it). Seeded
xoshiro from a fixed constant so a given `--packet-loss 10` run is
reproducible.

### `--rate` resample

Post-decode resample via the existing `audio::resample` module.
Accept any sample rate `rubato` accepts. Default output stays 48 kHz.
Sample-rate choice affects the WAV header but nothing else in the
pipeline (ropus decode is always at 48 kHz internally).

### Output format matrix

```
(--raw, --float) → behaviour
(false, false)   → WAV i16 with dither (unless --no-dither)   [default]
(false, true)    → WAV float32 (dither irrelevant)
(true, false)    → raw interleaved i16 (with/without dither)
(true, true)     → raw interleaved float32
```

`--raw` + `--float` both allowed (raw 32-bit float is useful for
pipelines). `--no-dither` is silently a no-op in float modes.

### Tags on decode

Parse OpusTags after OpusHead; on successful parse print a short
`tags     vendor="…", N comments` line in the banner. No `--show-tags`
flag needed — if the user wants deep tag display, they reach for
`ropusinfo`.

## `ropusinfo` gaps

### New flags

| Flag                 | Behaviour                                      |
|----------------------|------------------------------------------------|
| (default)            | Multi-line human-readable output               |
| `-e`, `--extended`   | Adds per-packet TOC decode                     |
| `-q KEY`, `--query KEY` | Print one named value, no banner, no newline styling |

**Short-flag collision to resolve:** `ropusinfo` currently binds `-q`
to `quiet` at `ropusinfo/src/main.rs:22`. opus-tools' `opusinfo` binds
`-q` to query. We match opus-tools: the existing `quiet` flag becomes
`--quiet` only (long-form, no short). All other tools keep `-q` for
quiet — the collision is localised to `ropusinfo`. This is an argv
break for the `ropusinfo -q` muscle memory built over the past days;
acceptable at this sandbox stage.

### Default output

Drop the single-line format for a multi-line block that mirrors
`opusinfo`:

```
Input File: foo.opus
Channels: 2
Sample rate (input): 48000 Hz
Pre-skip: 312
Output gain: 0 dB
Channel mapping family: 0
Vendor: libopus 1.4
User comments:
  ARTIST=Example
  TITLE=Track
  ENCODER=ropusenc 0.1.0
Total data length: 42312 bytes
Playback length: 3m 12.05s
Average bitrate: 128.4 kb/s
```

Implementation: replace the current single-line formatter in
`commands::info`. Reuse existing granule + file-size + packet-count
code unchanged.

### `--extended`

For each data packet, decode the TOC byte (`packet[0]`):

- Config (0-31) → mode (SILK/Hybrid/CELT), bandwidth, frame size
- Stereo flag
- Frame count code (0..=3)

Emit one line per packet at the end of the default block. For long
files this is verbose; that matches opus-tools behaviour.

TOC decoding is ~20 lines of table lookup per RFC 6716 §3.1. No ropus
dependency needed — this is pure bit-twiddling on byte 0 of each
packet, which `ropusinfo` already has from `PacketReader`.

### `--query KEY`

Valid keys: `channels`, `samplerate`, `preskip`, `gain`, `duration`,
`bitrate`, `vendor`, `comment:KEY` (e.g. `comment:artist`).

Prints the raw value with no banner, no colour, no trailing context.
Unknown key → exit 2 with a one-line error to stderr. Missing comment
key → empty stdout, exit 0 (scriptable: `if ropusinfo -q
comment:artist x.opus | grep -q .; then …`).

`--query` implicitly disables banner and colour regardless of global
`-q/--quiet` / `--no-color`. This is a stricter mode.

### Granule-gap detection

Two unambiguous gap classes only:

1. **Backwards jump:** `page_absgp[i] < page_absgp[i-1]`. Never valid.
2. **Forward jump > 5760 samples:** 5760 @ 48 kHz is one 120 ms frame,
   the maximum any single packet can advance the granule. A page that
   jumps further has dropped at least one packet's worth of samples.

Per-page jitter between 0 and 5760 samples is normal (frame sizes vary
per-packet); flagging that would produce false positives on every
multi-framesize stream. Only emit a warning for the two cases above.

Add a `WARN: N granule gap(s) detected` line to the default output
when any gap is seen. Extended mode lists each gap's page index and
absgp before/after.

`ogg` crate's `PacketReader` doesn't expose per-page granules directly;
the reader is packet-oriented. Two options:

1. **Wrap it:** implement a thin `PageReader` that reads raw Ogg page
   headers and tracks granules, parallel to `PacketReader` for info's
   use only.
2. **Reverse-scan on demand:** we already have `read_last_granule` for
   a single page. Generalise to walk all pages at info time.

Pick (1). It's ~50 lines, lives next to `read_last_granule`, and gives
a clean iterator surface for the extended-mode per-packet output.

## `ropusplay` gaps

### New flags

| Flag                 | Behaviour                                      |
|----------------------|------------------------------------------------|
| `--list-devices`     | Print available output devices, exit 0         |
| `--device NAME`      | Select named device instead of default         |
| `--gain DB`          | Same as `ropusdec --gain`                      |

Everything else on the previous gap list (seek, pause, interactive
controls, stdin) is out of scope per the non-goals above.

### Device selection

We're on `rodio = "0.20"` (verified in
`ropus-tools-core/Cargo.toml`), which uses the newer
`OutputStreamBuilder` pattern rather than the 0.17-era
`OutputStream::try_from_device`. Concrete call shape needs a quick
code-first spike during Step 6 implementation — the rodio 0.20
builder API exposes device enumeration through `cpal` directly, and
the exact wiring differs enough from earlier rodio versions that
nailing it down at design time risks specifying the wrong function.

Behavioural intent (independent of API shape):

- `--list-devices` prints each cpal host output-device name on its
  own line and exits 0.
- `--device NAME` does an exact (case-sensitive) match against those
  names. Not found → list available to stderr, exit 1.
- Empty host device list → clean error, not a panic.

### `--gain`

Pipe through to the decoder's `set_gain` before the rodio sink takes
samples. Same semantics as `ropusdec --gain`.

## ropus API extensions

Two new builder methods on `ropus::EncoderBuilder`:

```rust
pub fn vbr_constraint(mut self, on: bool) -> Self;
pub fn packet_loss_perc(mut self, pct: u8) -> Self;  // range-checks 0..=100
```

Both are thin wrappers over existing internal state
(`ropus/src/opus/encoder.rs`). Tests: builder round-trip only; the
underlying encoder paths are already differential-tested against libopus.

Version bump: `ropus` → minor (new public API). Tool crates → patch
(no API change, behavioural only).

## Testing

### Unit tests (one per new piece of logic)

- `OpusTags::encode` / `::parse` round-trip, plus a fixture-byte
  parse matching the RFC 7845 example.
- Dither: given a deterministic seed, the output is stable. Given
  `--no-dither`, the output is bit-exact-equal to a manual i16 cast.
- TOC decoder: fixture bytes for each of the 32 config values; each
  returns the mode/bandwidth/framesize documented in RFC 6716 Table 2.
- Granule-gap detector: synthetic page stream with a deliberate skip,
  detector flags the gap.
- Base64 picture block: encode a tiny PNG fixture, parse it back, the
  `METADATA_BLOCK_PICTURE=` line round-trips.

### Integration tests

Extend `ropus-tools-core/tests/round_trip.rs` with:

1. Tag round-trip: encode with `--artist X --title Y`, decode (no-op)
   and `info --query comment:artist` returns `X`.
2. Framesize round-trip: encode at `--framesize 60`, decode, RMS-SNR
   check vs. input holds at the same threshold as 20 ms.
3. Float decode: encode → decode `--float`, parse the resulting
   WAV's format chunk, assert format code 3 and bit depth 32.
4. Stdin/stdout: `std::process::Command` shelling out to the built
   `ropusenc` binary (`CARGO_BIN_EXE_ropusenc`) with a WAV fixture
   piped to its stdin and `-` as argv. In-process Cursor path is
   blocked by the `PathBuf("-")` sentinel branching inside the
   command function; accepted.
5. `--packet-loss` round-trip: encode a sine, decode with
   `--packet-loss 10`, verify the output length matches the source
   (PLC fills dropped packets) and that no decode error occurs.
   Perceptual quality is not asserted — that's what differential
   tests against libopus are for, and PLC behaviour is already
   covered by `ropus::Decoder` tests.

### What we don't test

- Every flag combination — combinatorial blowup, minimal value.
- That libopus' opus-tools produces byte-identical output for the same
  flags. It doesn't today (different encoder paths, different pre-skip
  in edge cases) and isn't a project goal. The harness is for encoder
  bit-parity; the tools are UX parity only.

## Implementation order

Designed so each step lands as its own commit and leaves the tree
working.

1. **ropus API:** add `vbr_constraint` and `packet_loss_perc` builder
   methods. Minor-version bump. Tests.
2. **OpusTags R/W:** replace `build_opus_tags` + `read_opus_tags`
   with the `OpusTags` struct. Update the three call sites. All three
   tools still behave identically to today (empty comments vec).
3. **Encode flags:** `--cvbr`, `--hard-cbr`, `--music`, `--speech`,
   `--comp` alias, `--expect-loss`, `--framesize`, `--downmix`,
   `--serial`, plus the metadata flags (`--artist`, `--title`,
   `--album`, `--genre`, `--date`, `--tracknumber`, `--comment`,
   `--picture`). One commit.
4. **Decode flags:** `--float`, `--raw`, `--rate`, `--gain`,
   `--no-dither` (+ default dither), `--packet-loss`,
   `--force-wav`. One commit.
5. **Info rework:** multi-line default output, OpusTags display,
   `--extended` TOC decode, `--query`, granule-gap detection. One
   commit.
6. **Play flags:** `--list-devices`, `--device`, `--gain`. One commit.
7. **Stdin/stdout:** `-` plumbing in encode and decode. One commit,
   last because it touches all the I/O paths added above.

Each step: `cargo build --workspace`, `cargo test --workspace`, plus a
manual spot-check with one real invocation of the touched tool.

## Risks and open questions

- **Symphonia WAV-from-stdin seekability.** Symphonia' probe chain can
  require backward seeks for format sniffing. Mitigation: buffer stdin
  into a `Vec<u8>` then wrap in `Cursor`. Memory-bounded by input size;
  fine for tools usage, not fine for pipelines with multi-GB inputs.
  If that becomes real, revisit with a streaming probe.
- **`--packet-loss` reproducibility.** A fixed seed makes test output
  stable but surprises users who expect fresh randomness per run.
  Accept it — libopus' `opusdec --packet-loss` is also deterministic
  (seeds from the packet sequence).
- **Dither RNG crate choice.** `rand` is heavyweight for one xoshiro.
  Hand-write the 8-line xoshiro128** impl inline in `audio/wav.rs`.
  No new dep.
- **`opus-tools`' `--save-range` audit trail.** Power users of
  `opusenc` who diff range-coder state will find it missing. The
  harness (`ropus-compare`) is a strict superset of what `--save-range`
  provides. Document this in the tool's `--help` if anyone asks.
- **Extended info output verbosity.** A 3-minute Opus file has ~9000
  packets. Extended mode prints a line per packet. Not paginated.
  Acceptable — users pipe to `less` or `head`.

## Out of scope (explicit)

- Chained multi-stream Ogg Opus on decode/info.
- Interactive playback controls in `ropusplay`.
- `--save-range` on encode.
- Channel mapping families 1/2 (surround, ambisonics).
- Opus-in-WebM / Opus-in-MP4 containers.
- Any form of configuration file. All state is argv.
