# Multi-Stage Branch Coverage Plan — To 98%

**Date:** 2026-04-16
**Baseline:** 87.58% branch coverage (5,153 / 5,884). Gap: 731 missed branches.
**Target:** ≥98% (≤118 missed). Net delta: cover ~613 more branches.

## Principles

- **Keep bit-exact tests green.** New tests must not regress the existing 1,319 passing tests.
- **Prefer unit-style tests at the module boundary** over heavy round-trip tests for specific branch targets. Use the C-reference harness (`mdopus-compare`) only when bit-exact comparison is cheaper than crafting the branch manually.
- **Property / parameter sweeps** are the highest leverage: one parameterized test iterating across (bitrate × bandwidth × channels × complexity × FEC × DTX × VBR × CBR) can close dozens of branches.
- **Commit after each stage** — each stage has its own measurable coverage gain.
- **Iterate**: after each stage, re-run `cargo +nightly llvm-cov --branch --lib --summary-only` and list remaining hotspots.

## Stage 2 — Decoder / Repacketizer / Multistream

**Files:** `src/opus/decoder.rs` (+53), `src/opus/repacketizer.rs` (+57), `src/opus/multistream.rs` (+41).

**Tactics:**
1. Decoder:
   - PLC sequences with 1, 2, 3, 10, 50 consecutive lost packets; then resynchronize.
   - FEC-with-recovered-LBRR path (`opus_decode_native(…, decode_fec=1)`).
   - Every TOC config code (0..31) round-tripped through decode to validate config table.
   - Soft-clip on/off with clipping signals.
   - Gain query/set and float/int API boundary.
2. Repacketizer:
   - Pad to target length / strip padding / combine N packets for N in {1..48}.
   - Self-delimited framing parse (`opus_packet_parse_impl`) with truncated/padded buffers.
   - Mixed code-0/1/2/3 input packets; oversize reject; underflow reject.
3. Multistream:
   - Surround configs: stereo pairs + mono streams for 3/5/7/8 channels.
   - `opus_multistream_surround_encoder_create` with each `mapping_family` {0,1,2,255}.
   - Decode matrix mismatch (wrong channel count) — error-path branches.

**Acceptance:** each of the three files ≥ 95% branches. Total ≥ 93.5%.

## Execution Protocol

For each stage:
1. Spawn an agent with: (a) this plan as context, (b) the exact files and missed-branch line list for the stage, (c) instructions to add tests in the relevant `#[cfg(test)] mod tests` block.
2. Agent runs `cargo test --lib` to confirm all 1,319 + new tests pass.
3. Agent runs `cargo +nightly llvm-cov --branch --lib --summary-only` and reports new percentages.
4. Review diff, commit with message `Stage N: <file>: add branch-coverage tests (+M branches)`.
5. Update this plan's running total.

## Measurement Tooling

- `tools/cov_report.py` — per-file branch/line/region summary.
- `tools/cov_missed_branches.py` — specific uncovered branch locations (line + side).
- Generate JSON with: `cargo +nightly llvm-cov --branch --lib --json --output-path target/cov-stageN.json`.

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| New tests slow the suite | Keep sweep fanout bounded; gate expensive cases behind `#[ignore]` or `cargo test -- --test-threads` budgets. |
| Test flakiness from RNG | Use fixed seeds (`StdRng::seed_from_u64`) for any random input. |
| Branches that are genuinely unreachable in safe input | Document with a comment; do not hide with `#[cfg(not(test))]`. |
| Bit-exactness regressions | Run `mdopus-compare roundtrip` on a representative WAV after each stage. |
| Coverage instrumentation drift | Use only `cargo +nightly llvm-cov --branch --lib` for the headline metric. |
