## Strategy

Do not chase raw test count. Chase uncovered execution in files that can move the totals materially.

The plan is staged for yield and risk:

- hit deterministic helper/state surfaces first when they buy real coverage
- move next into packet/control code with bounded setup cost
- leave the largest codec state machines for focused branch-directed work
- rerun branch coverage after every meaningful batch and reprioritize from fresh data

## Progress Rules

After each stage:

1. run scoped tests for touched files
2. run `cargo test --lib --all-features`
3. run `cargo +nightly llvm-cov --branch --lib --all-features --summary-only`
4. update the journal
5. commit the finished logical unit

## Reality Check

This plan can improve coverage materially, but the current 98% target should be treated as a campaign target, not a near-term checkpoint.

From the current baseline, the remaining gap is too large to close with cosmetic test padding. The only path is sustained branch-directed testing in the largest codec state machines. That is real work, not bookkeeping.
