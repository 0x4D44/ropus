Author: Arthur + Claude
Date: 2026-04-19
Status: DRAFT (revision 2)

Revision history
- r1: two commits (OpusTags parser + player rewrite).
- r2: OpusTags parser landed independently (commit `799c818`); this HLD
  shrinks to one commit (player only). Folds in self-review gaps:

## Goal

Turn `ropusplay` from a one-shot file player into a useful CLI music player:

- Accept a directory as input and play every `.opus` file inside.
- `--loop {off,all,single}` repeat mode.
- Live status line showing track number, tags, progress bar, elapsed/total,
  average bitrate.
- Keyboard controls: `[space]` pause/resume, `[n]` next, `[p]` previous,
  `[q]` quit.

## Non-goals

- Streaming decode. Upfront-decode-to-`Vec<f32>` stays. A 10-minute 192 kbps
  Opus track fits in ~110 MB of f32 samples; fine for intended use. Hour-long
  tracks are feasible (~660 MB) but see "decode latency UX" below.
- Shuffle.
- Seeking within a track (`←/→` unused).
- Volume keyboard control. `--volume` CLI flag is enough.
- Metadata for non-Opus inputs. MP3/FLAC via symphonia show filename stem
  only; wiring symphonia's `MetadataRevision` is a separate piece of work.
- Gapless playback. Tracks separate by the time it takes to drop the old
  `Sink` and create a new one — on Windows/WASAPI this is 50–200 ms with a
  faint click. Documented tradeoff (see "Inter-track gap" below).
- A TUI framework (ratatui/cursive). Single repainted line matches existing
  `colored::*` CLI aesthetic.
- Feature flags, env vars, compile-time gates.
- Changes to `ropusenc`, `ropusdec`, `ropusinfo` CLI surface. (`ropusinfo`
  will gain tag display separately via the opus-tools-parity HLD step 5; not
  this HLD.)

## Current state

`ropusplay` (`ropusplay/src/main.rs`) takes a single file path, calls
`ropus_tools_core::commands::play`, which decodes the whole file into
`Vec<f32>`, hands it to `rodio::Sink`, and calls `sleep_until_end`. No
progress, no keys, no playlist.

`OpusTags` parsing landed at `container/ogg.rs` (commit `799c818`):

```rust
pub struct OpusTags { pub vendor: String, pub comments: Vec<String> }
impl OpusTags {
    pub fn parse(data: &[u8]) -> Result<Self>;
    pub fn get(&self, key: &str) -> Option<&str>;  // case-insensitive
}
```

`comments` holds raw `"KEY=value"` strings (preserves byte-identity on
round-trip). `get()` splits on the first `=` and does ASCII-case-insensitive
key match — exactly what the status line needs (`tags.get("TITLE")`, etc.).

## `PlayOptions`

```rust
// ropus-tools-core/src/options.rs
#[derive(Debug, Clone, Copy, Default, PartialEq, Eq)]
pub enum LoopMode { #[default] Off, All, Single }

#[derive(Debug)]
pub struct PlayOptions {
    pub input: PathBuf,          // file OR directory
    pub volume: Option<f32>,
    pub loop_mode: LoopMode,
}
```

`input` stays a single path; directory expansion happens inside
`commands::play`. A `Vec<PathBuf>` on the library API would push the
dir-scan concern onto every caller that doesn't care (fb2k plugin, tests).

## CLI surface

```
ropusplay INPUT [--volume V] [--loop off|all|single] [-q] [--no-color]
```

`INPUT` is a file or directory. Default `--loop off`. Clap maps via
`ValueEnum` derive on `LoopMode`.

## Playlist building

```rust
fn build_playlist(input: &Path) -> Result<Vec<PathBuf>> {
    if input.is_file() { return Ok(vec![input.to_path_buf()]); }
    let mut files: Vec<PathBuf> = fs::read_dir(input)?
        .filter_map(Result::ok)
        .map(|e| e.path())
        .filter(|p| {
            p.is_file() && p.extension()
                .and_then(|e| e.to_str())
                .map(|e| e.eq_ignore_ascii_case("opus"))
                .unwrap_or(false)
        })
        .collect();
    files.sort();
    if files.is_empty() { bail!("no .opus files in {}", input.display()); }
    Ok(files)
}
```

Plain lexicographic sort. Not recursive. `track1.opus`, `track10.opus`,
`track2.opus` ordering is the documented cost — fix is to zero-pad filenames
or pull a natural-sort crate later. Not worth a dep today.

## Status line

Single stdout line, repainted with `\r` + ANSI erase-to-end-of-line:

```
▶ 03/17 ⟳all  Franz Liszt — Sonata in B minor  [▓▓▓▓░░░░░░]  1:23 / 4:17  192 kbps
```

Composed left to right:
- Play/pause glyph: `▶` / `⏸`.
- `NN/MM` — 1-based track index / playlist length.
- Loop indicator: ` ⟳all` / ` ⟳one`, empty for `Off`.
- `ARTIST — TITLE` from `tags.get("ARTIST")` / `tags.get("TITLE")`,
  falling back to filename stem if either is missing.
- 10-cell progress bar from `pos / dur`, filled cells clamped at 10.
- Elapsed/total timestamp.
- Average bitrate from `file_size * 8 / duration_s / 1000`.

Repaint cadence: 100 ms, but only emit a write when the rendered string
actually differs from the previous frame (sub-second re-writes of the same
text flicker on Windows Terminal).

### Terminal width handling

```rust
let cols = crossterm::terminal::size().map(|(w, _)| w as usize).unwrap_or(80);
```

The status line has a fixed-width right side (progress bar + time + bitrate
≈ 32 columns) and a variable-width left side (`ARTIST — TITLE`). When
rendered length exceeds `cols`, truncate the title section with `…` so the
right side always fits. If `cols < ~50`, collapse to a minimal rendering:
`NN/MM  M:SS / M:SS` (drop bar and bitrate).

Width is re-read every repaint (cheap syscall) so a resize mid-playback
doesn't garble.

Duration format: `M:SS` when `dur < 3600s`, `H:MM:SS` otherwise.

### Long-title example at 60 cols

```
▶ 03/17 ⟳all  Franz Liszt…  [▓▓▓▓░░░░░░]  1:23 / 4:17  192 kbps
```

## On-screen layout

The main loop orchestrates this layout on startup:

```
ropusplay 0.2.0  (built 2026-04-19 abc1234)    ← print_banner
# play                                          ← heading("play")
playlist 17 tracks from /music/liszt            ← new: source summary
[space] pause  [n] next  [p] prev  [q] quit     ← help line, printed once
                                                ← one blank line
▶ 03/17 ⟳all  Franz Liszt — Sonata …  [▓▓▓…]    ← repainting status line
```

Help line scrolls up as tracks print warnings (see error handling). Status
line is anchored on the bottom, always the most recent line.

When a track fails, the error prints **above** the status line (a plain
`eprintln!` will scroll the status line up, but that's fine — user still
sees current state on the bottom row). Next repaint tick writes to the new
bottom row.

## Keyboard handling

Add `crossterm = "0.28"` to `ropus-tools-core/Cargo.toml`.

Enable raw mode only when stdout **and** stdin are ttys
(`crossterm::tty::IsTty` on both). If either fails the check, or `--quiet`
is set, fall back to the non-interactive path (see below).

Raw-mode RAII guard restores the terminal on panic, `?`-bubbling, or Ctrl-C:

```rust
struct RawModeGuard;
impl RawModeGuard {
    fn enable() -> Result<Self> {
        crossterm::terminal::enable_raw_mode()?;
        Ok(Self)
    }
}
impl Drop for RawModeGuard {
    fn drop(&mut self) { let _ = crossterm::terminal::disable_raw_mode(); }
}
```

### Explicit Ctrl-C mapping

**In raw mode, Windows delivers Ctrl-C as a `KeyEvent`, not a signal.** If
we don't match it, the user hangs. Explicit mapping:

```rust
match (event.code, event.modifiers) {
    (KeyCode::Char(' '),  _) => Action::TogglePause,
    (KeyCode::Char('n'), _) => Action::Next,
    (KeyCode::Char('p'), _) => Action::Prev,
    (KeyCode::Char('q'), _) => Action::Quit,
    (KeyCode::Char('c'), KeyModifiers::CONTROL) => Action::Quit,
    _ => Action::None,
}
```

### Prev semantics

Player convention: if current position > 2 s, `[p]` restarts the current
track; only a quick retap (position ≤ 2 s) goes to the actual previous
track. Saves users from accidental album rewinds.

```rust
Action::Prev => {
    if sink.get_pos() > Duration::from_secs(2) {
        // Restart current: re-decode is cheap vs. surprising the user.
        Action::Next_at_same_index
    } else {
        idx = if idx == 0 { playlist.len() - 1 } else { idx - 1 };
    }
}
```

## Control flow

One thread. The main loop is a small FSM over the playlist:

```rust
let playlist = build_playlist(&opts.input)?;
let (_stream, handle) = rodio::OutputStream::try_default()?;
let interactive = stdout.is_tty() && stdin.is_tty() && !quiet;
let _raw = interactive.then(RawModeGuard::enable).transpose()?;

let mut idx: usize = 0;
'outer: loop {
    let path = &playlist[idx];

    // Decode-latency UX: print a single ephemeral line before the (possibly
    // slow) decode, overwritten by the first status-line repaint.
    print!("\rdecoding {}/{}  {}…", idx + 1, playlist.len(),
           display_title(path));
    io::stdout().flush()?;

    let (decoded, tags) = match decode_track(path) {
        Ok(ok) => ok,
        Err(e) => {
            print!("\r\x1b[K");  // clear the "decoding..." line
            eprintln!("{} skipping {}: {e}", "warning:".yellow(),
                      path.display());
            idx = match advance_on_error(idx, &playlist, opts.loop_mode) {
                Some(next) => next,
                None => break 'outer,
            };
            continue 'outer;
        }
    };
    let duration = Duration::from_secs_f64(
        decoded.samples.len() as f64 / decoded.channels as f64
            / decoded.sample_rate as f64,
    );

    let sink = rodio::Sink::try_new(&handle)?;
    if let Some(v) = opts.volume { sink.set_volume(v.clamp(0.0, 1.0)); }
    sink.append(rodio::buffer::SamplesBuffer::new(
        decoded.channels as u16, decoded.sample_rate, decoded.samples,
    ));

    let action = if interactive {
        run_track_loop(&sink, &tags, path, idx, &playlist, duration,
                       opts.loop_mode)?
    } else {
        run_track_noninteractive(&sink, &tags, path, idx, &playlist,
                                 duration)
    };

    match action {
        Action::TrackFinished => {
            idx = match opts.loop_mode {
                LoopMode::Single => idx,
                LoopMode::All    => (idx + 1) % playlist.len(),
                LoopMode::Off    => {
                    if idx + 1 < playlist.len() { idx + 1 } else { break 'outer }
                }
            };
        }
        Action::Next => idx = (idx + 1) % playlist.len(),
        Action::Prev => {
            // Semantic: >2s means restart current, else go back.
            if sink.get_pos() <= Duration::from_secs(2) {
                idx = if idx == 0 { playlist.len() - 1 } else { idx - 1 };
            }
            // else idx unchanged — re-enter 'outer, decode same track again
        }
        Action::Quit => break 'outer,
    }
}
```

`run_track_loop` owns the 100 ms `event::poll` loop. Each tick:
1. Repaint status line if content changed (pause flipped, or elapsed second
   rolled over).
2. Handle pending key event (any that came in during the 100 ms window).
3. Return `TrackFinished` if `sink.empty()`.

Position comes from `sink.get_pos()` — rodio advances it against wall time
while playing and freezes while paused (confirmed by reading
`rodio-0.20.1/src/sink.rs:351`).

### Per-track error handling

A corrupt `.opus` in a directory must not kill the whole session. Decode
errors are caught per track, logged with a yellow `warning:` prefix, and
the loop advances as if the track had finished (respecting `loop_mode`).
If every track fails, `advance_on_error` returns `None` and we exit.

`decode_track(path)` returns `(DecodedAudio, OpusTags)` — wraps
`decode_to_f32` + reading the tags packet. Non-Opus inputs return an empty
`OpusTags::default()`.

### Non-TTY fallback

If `interactive == false`, `run_track_noninteractive` prints one line per
track and calls `sink.sleep_until_end()`:

```
playing 03/17  Franz Liszt — Sonata in B minor  (4:17)
```

No status line, no raw mode, no key polling. Piped invocations
(`ropusplay dir/ > /dev/null`), CI smoke tests, and `--quiet` users all land
here. `--loop single` still works (plays same track forever); Ctrl-C at the
OS level exits cleanly because raw mode is not engaged.

## Risks and open questions

- **Keypress latency during decode.** Main thread decodes synchronously;
  keys queue up in crossterm's buffer. Mashing `q` during a 5 s decode
  waits 5 s to register. Fix is decoding on a background thread and
  polling events during the wait — meaningful complexity for a short-window
  issue. Accepted for now; the `decoding …` line at least signals what's
  happening.
- **Inter-track gap (50–200 ms) with click.** Each track creates a new
  `Sink`, which reinits the cpal output stream on Windows/WASAPI. Fix
  would be reusing one sink and queue-ahead-by-one with `skip_one()` for
  `n`, but prev still requires a new sink and queue-ahead complicates the
  `sink.empty()`-based track-end signal. Accepted at the stated non-goal;
  revisit only if users complain about album playback.
- **Sub-second repaint flicker on slow terminals.** Mitigated by the
  "only emit if string changed" gate at 100 ms cadence.
- **crossterm emoji on legacy Windows conhost.** `▶ ⏸ ⟳ ▓ ░` render cleanly
  on modern Windows Terminal / PowerShell 7. Legacy `conhost.exe` may show
  boxes. Accepted: nothing breaks, ASCII fallback is a future follow-up.
- **`sink.get_pos()` drift at end of track.** Can exceed computed
  `duration` by a few ms because rodio's queue-empty fires after the last
  sample. Progress bar clamps at 100%; `sink.empty()` is the authoritative
  end signal.
- **Huge directories (10k+ tracks).** `fs::read_dir` + sort is O(n log n)
  and allocates ~800 KiB of `PathBuf`s. Fine.
- **`ropusinfo` tag pretty-print still pending.** Owned by the
  opus-tools-parity HLD (step 5). `ropusplay` does not depend on that
  landing — it parses tags independently.

## Out of scope (explicit)
