### 01:40 — Bug D root cause identified
Agent (opus) reproduced from saved input in ~60s and did the bisect.

**Surprising finding**: ALL 16 tracked SILK inter-frame state fields match
byte-for-byte between Rust and C after frame 1. The earlier hypothesis list
(gain/LTP/NLSF predictor history) was wrong — SILK itself runs identically.

**Actual root cause**: Missing SILK budget-bust handling at
`src/opus/encoder.rs:2573-2593`. C reference unconditionally checks
`ec_tell(&enc) > (max_data_bytes-1)*8` and, on overshoot, writes a 1-byte
PLC-style SILK payload then lets `opus_packet_pad` reshape to code-3 + padding.
Rust has (1) the gate condition **inverted** (`if self.mode != MODE_SILK_ONLY`
guards exactly the case that fires) and (2) an **empty body** — the corrective
action is absent.

**Why it only bites at low bitrate**: At br=6001 / 8kHz / 20ms, budget is
104 bits in a 112-bit buffer. A harsh noise burst overshoots at low
complexity. Normal fuzz bitrates (~12000 bps) never trigger the bust. Explains
why 4100+ input attempts didn't hit it.

Fix is clear, small, and localized. Dispatching fix agent with explicit spec
from the investigation notes.
