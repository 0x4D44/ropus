## 2026-05-04 22:24 BST

Root cause for the six single-stream `fuzz_encode_multiframe` artifacts is identified and fixed in the representative repro.

Path to root cause:

- Resampler scalar state was not the cause; active resampler fields matched through the last good frame. I did fix one real parity defect while there: Rust now computes `inv_ratio_q16` for all resampler modes like C does, including copy mode.
- Adding a pre-LP trace point showed the main-frame input buffer matched before `silk_lp_variable_cutoff`, but the LP filter memory diverged before that main frame.
- Tracing the prefill LP call showed the prefill input/state matched at entry, but the LP output differed.
- The bug was `TRANSITION_FRAMES`: Rust used `80`, while C derives `TRANSITION_FRAMES = TRANSITION_TIME_MS / MAX_FRAME_LENGTH_MS = 5120 / 20 = 256`. Rust was therefore interpolating the wrong LP transition filter taps during CELT-to-SILK/SILK bandwidth prefill.

After changing `TRANSITION_FRAMES` to 256, the representative artifact `crash-c6df51d92a801467b128d92806de9657bdb1af9c` runs all five frames byte-identical. The only remaining diagnostic differences are inactive C union views of the resampler FIR state, not byte-affecting state.

Emotional temperature: relieved. This was exactly the kind of subtle constant drift that looks like a state-management bug until the trace is tight enough.

## 2026-05-04 23:17 BST

Root cause for the `fuzz_multistream` artifact is identified and fixed.

The remaining divergence was in surround masking, not the multistream packet framing. C's `surround_analysis` helper uses `logSum` with return type `opus_val16`; in the fixed-point build that truncates the Q24 log accumulator to 16 bits before storing it back into the `celt_glog` mask accumulator. Rust returned the full `i32`, which looked more sensible but over-reduced the SILK target rate. That made Rust enter SILK at `targetRate_bps=10000` while C used `13500`.

Patched Rust `log_sum` to preserve the C `opus_val16` truncation. The multistream crash artifact now passes the original fuzz target: Rust and C both emit the same 109-byte packet.

Emotional temperature: pleased, with a rueful note. This was a classic "cleaner than C" trap: the math was nicer, but the port needs the fixed-point truncation because it is observable codec behavior.
