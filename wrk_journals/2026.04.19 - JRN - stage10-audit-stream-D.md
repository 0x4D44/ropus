**Role.** Adversarial auditor. Scope: gaps between stages, not within. Siblings A (1-5) and C (8-9) already landed row-by-row compliance; B (6-7) pending.
**Inputs.** HLDs, journals, live source + test runs, commit range through `cfb9e23`.

## Summary

Stage 8.10 close-out is substantively complete and internally defensible. The cooperative streams will score each stage green against its HLD. What they won't flag, because the work wasn't scoped to one stage, is the **integration gap**: no test anywhere runs the full analysis + DNN + DRED pipeline end-to-end and decodes to PCM, because the last-mile `OpusDREDDecoder::decode*` methods still return `OPUS_UNIMPLEMENTED` and no journal tracks who inherits that follow-up now that Stage 7 FARGAN landed (for PLC only, not for DRED reconstruction). Several smaller concerns cluster around documentation drift (top-level `README.md` and `ropus/README.md` both still list DRED/DNN/Analysis as "Deferred" post-Stage-6/7/8 ship) and the one pre-existing `prop_sequence_invariants` failure that every stage inherits and no one owns. No material regressions introduced by late stages.

## Concerns

### 1. No end-to-end analysis + DNN + DRED integration test (HIGH)

- **Concern.** Stage 6 (analysis), Stage 7b.1.5 (DNN weight population + PLC wiring), and Stage 8 (DRED encoder + `OpusDREDDecoder::parse`/`process`) each ship with their own oracle. No single test drives a WAV through encode with analysis + DRED active and decodes to PCM via the full pipeline. The closest thing, `harness-deep-plc/tests/dred_integrated_encode.rs::rust_encoder_decoder_dred_roundtrip`, stops at `OpusDREDDecoder::process` (stage 2 of the FARGAN pipeline) and never calls `decode`/`decode24`/`decode_float` — because all three return `OPUS_UNIMPLEMENTED` (see `C:\language\mdopus\ropus\src\opus\dred.rs:228-278`).
- **Evidence.** Rust integrated test at `C:\language\mdopus\harness-deep-plc\tests\dred_integrated_encode.rs:307-349`. The three stubs at `ropus/src/opus/dred.rs:228`, `:249`, `:270`.
- **Material impact.** High for integration confidence; Low for shipped functionality if callers only need `parse`/`process` today. The claim "Stages 6-8 ship a working DRED-enhanced codec" is weaker than it appears — you can emit DRED payloads and parse them; you cannot yet use them to reconstruct PCM without falling through to FARGAN, which ropus has for PLC but hasn't wired for DRED.
- **What resolves it.** The intended Stage 7 joint-commit relocating `decode*` from `OpusDREDDecoder` onto `OpusDecoder` with a real FARGAN-synthesis body (documented in `stage8-supervisor.md:105` and `opus/dred.rs:205-216`). Schedule as "Stage 7c" or "Stage 8.11"; without it, Stage 8 ships a format-only DRED. Trip-wire tests at `opus/dred.rs:527, 550, 559, 567` will catch if this regresses.
