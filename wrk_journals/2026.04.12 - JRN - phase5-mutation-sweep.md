## Approach

### Prior Art (lessons from Phases 1-2)
1. **Filter table data** — fft.rs had 395/606 (65%) table data mutations. Exclude with `--exclude-re`.
2. **Triage categories**: dead code, equivalent mutants, no-test (live), weak-test
3. **Pinning pattern**: exact-value `assert_eq!` with C-reference-computed values
4. **Target**: >85% live-code kill rate per module

### Workflow per module
1. Agent reads module + existing tests
2. Lists mutants, designs filtering strategy
3. Runs filtered mutation test (~200-400 mutants, not full 1000+)
4. Triages survivors
5. Writes HLD for pinning tests → returns for review
6. **Team lead reviews HLD, iterates until approved**
7. Agent implements tests
8. Agent runs /code-reviewer on their own work
9. Agent applies feedback, returns final

### ~14:55 — Plan adjustment: sequential execution

**Root cause of contention:** cargo-mutants copies the entire repo into a temp dir and builds from scratch. With 3 worktree agents + main repo all running cargo concurrently, the build cache and linker compete. On Windows this is especially painful (no reflink, falls back to full copy).

Attempted clean mutation run on opus/decoder.rs while agents running — listed 680 mutants but 0 executed. Copy phase alone took 22s and didn't complete within timeout.

**Revised plan:**
1. Wait for batch 1 agents (celt/decoder, silk/decoder) to finish
2. Once worktrees are cleaned up, run mutations **sequentially from main repo**
3. One module at a time, with filtering, saving results to logs/
4. After each module, triage survivors myself (team lead does triage with data)
5. Write consolidated HLD per module
6. Dispatch implementation agents one at a time in worktrees
7. Code review each implementation

This is slower on paper but actually faster because there's zero contention. Estimated times:
- opus/decoder: ~680 mutants, ~15 min
- celt/decoder: ~1114 mutants, ~25 min
- silk/decoder: ~1435 mutants, ~30 min
- opus/encoder: ~1854 mutants, ~40 min
- celt/encoder: ~2429 mutants, ~50 min
Total: ~160 min (~2.7 hours) for mutation runs

Can cut significantly by filtering more aggressively.

Inverting the process:
1. Use the 2 existing HLDs (opus/decoder, celt/decoder) as-is
