# 2026.05.08 — Fuzz expansion campaign

## Context
Following the 2026-05-07 overnight 8h campaign (commit `bc446cb`):
- 9/10 fuzz targets came back clean across ~957 M executions
- `fuzz_encode_multiframe` found **2 real C-vs-Rust parity bugs** in
  the first ~2 minutes (commits `9b8c552` PLC delay_buffer; `55e224c`
  SILK LBRR ec_prev). Both bugs are the same shape: C mutates state
  in place, Rust takes `&State` and compensates at call sites, one
  call site got missed.
- 3 `fuzz_decode_safety` slow-units flagged (non-fatal). Untriaged.

The fast TTF on multiframe + repeating defect class strongly suggests
more bugs are reachable — the campaign caught the easy ones, not the
last ones. 8 of the 10 targets look saturated; running them again
mostly burns compute.

## Goal
One more focused round of fuzzing on the encoder, designed to surface
the *next* multiframe-class bug (not retest saturated targets).

2. Iterate on HLDs via SendMessage until the design is right.

4. I (team lead) maintain this journal across the campaign.
5. Final phase: a fresh focused fuzz run against the new/broadened
   targets to validate they actually find what they're designed to find

### 09:05 BST — Stream B triage landed; harness fix green-lit

Single root cause across all three slow-units:
- Decoder opens at low sample rate (≤16 kHz).
- `OpusDecoder::new` auto-loads embedded DEEP_PLC weight blob.
- Setter shuffle leaves `complexity ≥ 5` so `enable_deep_plc=true`,
  no `reset()` in any of the three shuffles.
- Fuzz target's FEC line passes `MAX_FRAME=5760` regardless of API
  sample rate. At 8 kHz this requests 720 ms of phantom PLC per FEC
  call. Recursive PLC chunker splits into 33-36 SILK PLC chunks of
  20 ms each. Each chunk runs the full FARGAN synthesizer at ~50 ms.
- Total: 1.7 s per FEC call × 2 FEC calls per iteration ≈ 3-7 s.

Math checks out vs observed wall-time (6.5/4.6/3.4 s on warm cache).

**Not a decoder bug.** C reference would do the same recursion if
DEEP_PLC sources were compiled in (harness omits them, so apples-to-
apples comparison wasn't possible — but the recursion shape in
`opus_decoder.c:716-768` is identical to ours).

Fix: clip the FEC `frame_size` argument in `fuzz_decode_safety.rs` to
the packet's actual sample count via `opus_packet_get_nb_samples`,
matching real-caller behavior. Single-file harness change; no library
edits.

Round-1 feedback: (1) acknowledge coverage delta on the FEC+prefix-
PLC composition path explicitly in Risks, (2) sanity-check other
fuzz targets for the same `MAX_FRAME`+`fec=true` shape before
committing.

Implementation workflow same as Stream A: implement → validate
(slow-unit replays + 30-min re-fuzz) → /code-reviewer →
/code-review-apply-feedback → re-verify → report. Don't commit.

### 13:30 BST — Stream E diagnosis landed; HIGH confidence; fix green-lit

Diagnosis at `wrk_journals/2026.05.08 - JRN - mode-transition crash
diagnosis.md`. Packet-layout math conclusive: 1 (TOC) + 41 (main
CELT) + 26 (redundancy) = 68 bytes; divergence at byte 42 = exact
start of CELT->SILK redundancy region. C↔Rust line-pair citations
direct. Empirical patch verification: gated SetPrediction(2) before
redundancy makes the crash artifact replay clean; reverting the gate
reproduces.

**Mechanism**: `CELT_SET_PREDICTION(v)` writes `disable_pf = v<=1`
and `force_intra = v==0`. Both are read during a CELT encode:
`disable_pf` gates pitch-prefilter (`celt/encoder.rs:2452`),
`force_intra` flows into `quant_coarse_energy`
(`celt/encoder.rs:2831`). C calls `SetPrediction(celt_pred)` BEFORE
the CELT→SILK redundancy frame (`opus_encoder.c:2288-2295`); Rust
calls it AFTER (`opus/encoder.rs:3097-3107`). When frame N+1 has
`prev_mode==CELT_ONLY && mode==HYBRID` (the crash config), the
redundancy encode inherits stale `disable_pf=1, force_intra=1` from
the prior frame's prefill `SetPrediction(0)` — producing 26 bytes
different across the redundancy region.

Why prior fuzz didn't find: `fuzz_encode_multiframe`'s uniform
bitrate distribution didn't reach the `prev_mode==CELT_ONLY ->
mode==HYBRID` adjacency that requires `redundancy && celt_to_silk`.
Stream D's boundary-biased grammar over `(forced_mode, max_bw,
signal, bitrate)` reaches it routinely.

Decisions on open questions:
1. **Delete existing SetPrediction block at 3097-3107** (cleaner C
   mirror; redundant prefill block at 3133-3143 unchanged).
2. **Patch-bump ropus/Cargo.toml** (matches 2026-05-07 fix pattern).
3. **Skip 15-min pre-commit fuzz**; campaign-final phase covers it.

Stream E green-lit for implementation. Required validation: crash
replay clean + prior known-failure replays clean + lib tests
1868/1868 + param-grid 17/17. /code-reviewer + apply-feedback
workflow. Don't commit (supervisor reviews diff).
