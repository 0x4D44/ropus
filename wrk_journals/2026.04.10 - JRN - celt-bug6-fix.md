## New findings (for future investigation)

1. **DTX transition has a separate step-1 bug.** The DTX on->off->on test
   still fails at step 1 after the Bug 6 fix. The HLD was optimistic in
   expecting this to shift to step 2. Root cause investigation needed —
   DTX changes both `use_dtx` and bitrate/application simultaneously,
   so the divergence may be in the DTX state machine or the combined
   mode-change path, not in `patch_transient_decision`.

2. **`patch_transient_decision` is effectively dead code.** The code
   reviewer and DA both observed that after i16 truncation, the maximum
   possible `mean_diff` after division is 32767, while the threshold is
   16777216. The function can NEVER return `true` in the C reference.
   This is a known upstream bug in xiph/opus, not our problem, but worth
   a code comment for future maintainers.

3. **HLD hex value is wrong.** The HLD states `52363264 = 0x031E9A00` but
   the correct hex is `0x031F0000`. The downstream truncation explanation
   in the HLD is also affected. Low priority — the HLD is a design
   artifact, not production code.
