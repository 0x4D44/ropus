# JRN: full-test Phase 3

**Date**: 2026-04-19
**Scope**: Stage 3 (ambisonics roundtrip) + Stage 4 (native benchmark sweep) for
the `full-test` runner, plus review-feedback application (CR verdict
`approve-with-nits`).

## What shipped (before review)

Two new modules + envelope/CLI wiring. Covered by 34 new unit tests.

| File | Purpose |
|---|---|
| `ambisonics.rs` | Stage 3 dispatcher. Spawns `cargo run --release -p ropus-harness --bin projection_roundtrip`, scrapes the per-order + trailing `PASS: all fixtures ...` marker. Distinguishes cargo-101 (build fail) from runtime-1 (fixture mismatch). |
| `bench.rs`      | Stage 4 dispatcher. One-time prebuilds `ropus-compare`, loops the 10-vector set from `tools/bench_sweep.sh`, parses the Unicode-box-drawn report table, computes encode/decode ratios. |

Four files modified:
- `full-test/src/main.rs`: Stage 3 + Stage 4 dispatch, chained skip on upstream
  build failure.
- `full-test/src/report.rs`: envelope extended with `stages.ambisonics` and
  `stages.bench` via the splice-serde-value pattern established in Phase 2.
- `full-test/src/cli.rs`: help text bumped to Phase 3.
- `full-test/Cargo.toml`: 0.2.0 -> 0.3.0.

## Review feedback applied (2026-04-19 apply-changes pass)

Based on `wrk_journals/2026.04.19 - CR - full-test-phase3.md` (`approve-with-nits`)
and `wrk_journals/2026.04.19 - DA - full-test-phase3.md`.

1. **Bench prebuild on-disk check** (CR Important #1). After `cargo build` returns
   `status.success()`, `bench.rs::run()` now verifies the `ropus-compare` binary
   is actually on disk. If not, stage-level `build_failed = true` with reason
   `"ropus-compare binary not on disk after build"` — no per-vector spawn storm
   on stale locks / concurrent deletes / exotic `CARGO_TARGET_DIR`.

2. **`CARGO_TERM_COLOR=never` + `--color=never`** (CR Important #2). Set
   `CARGO_TERM_COLOR=never` on all three `Command` invocations in Stage 3 and
   Stage 4 (`ambisonics::run`, `bench::prebuild_ropus_compare`,
   `bench::run_one_vector`). Additionally pass `--color=never` to
   `ropus-compare bench ...` so the Unicode-box-drawing parser is robust
   against future terminal-detection logic in the harness.

3. **Crash vs skip split** (DA #3, CR scoped). `VectorBench` gained
   `crashed: bool` + `crash_reason: Option<String>`. The `skipped` path is now
   reserved exclusively for missing fixtures (`skip_reason: "fixture missing at
   <path>"`). The crash path — non-zero exit, parse failure, spawn error —
   flips `crashed` and leaves `skipped=false`. `BenchResult::all_passed()` now
   returns `false` when any vector has `crashed==true` or when
   `build_failed==true`; missing fixtures keep the stage green.

   This lets Phase 4's banner logic surface crashes as WARN (not FAIL — bench
   stays WARN-only per HLD § PASS/FAIL/WARN) without masking them as
   "fixture missing".

4. **HLD citation on `all_passed()`** (CR Nit #7). Added a doc comment citing
   `HLD § PASS/FAIL/WARN` explicitly, with the `"bench is WARN-only on
   regressions; only crashes elevate"` quote.

5. **Stale docstring** (CR Nit #11). `parse_summary`'s doc comment no longer
   claims to parse a `=== SUMMARY ===` block header (it doesn't — it scans
   for `PASS:` markers and per-order lines over the whole stream). Rewrote
   to "per-order + overall marker lines in the stdout stream" with a note
   about header-independence.

## Deliberately NOT applied (supervisor / future phase scope)

- **Vector-table shared source of truth** (DA #1). Real risk: `bench.rs`
  `VECTORS[]` is a fork of `tools/bench_sweep.sh` with only first/last labels
  guarded by a pin test. Adding an 11th vector to the shell without the Rust
  table drifts silently. **Follow-up**: extract to `tests/vectors/bench_manifest.toml`
  read by both, or add a test that parses the shell's `VECTORS=(...)` and
  diffs. Out of Phase 3 apply-changes scope; flagged for a future phase.

- **Hand-crafted-fixture drift** (CR Important #3). Mitigated with a `TODO`
  comment on `build_vector_row` pointing at `harness::print_bench_report`
  (`harness/src/main.rs`) as the source-of-truth for the column widths and
  Unicode box chars. No fixture capture change in this pass.

- **Three-builds architecture** (DA #2). Stages 2/3/4 do independent cargo
  builds with different profiles. Phase 5 integration will measure the real
  wallclock impact; fixing this speculatively is premature.

- **Stage 3 `cargo run` asymmetry with Stage 4 build+exec** (CR Important #4).
  HLD § Stages:66 specifies `cargo run --release` for Stage 3 verbatim;
  deviation would require an HLD amendment, not an apply-changes pass.

- **LOC ceiling** (CR nit / DA #9). HLD § Success criteria:179 cap is ≤1200;
  Phase 3 total is ~2118. Supervisor amends HLD separately.

## Hints for Phase 4 (HTML report) agent

- `VectorBench` now has four mutually-exclusive mode flags: `skipped`
  (fixture missing), `crashed` (invocation failed), neither (clean row).
  Phase 4's row rendering must branch on all three.
- `BenchResult::build_failed` is new. Banner logic needs to treat it like
  `TestsResult::build_failed`: stage is red, downstream propagation unchanged.
- Bench crashes flip `BenchResult::all_passed()` to false but the HLD's
  exit-code rule (0 on WARN) still applies — Phase 4 must distinguish
  "stage non-green" from "overall exit non-zero". A crashed vector should
  surface as WARN in the banner, not FAIL, and must not set exit_code=1.
