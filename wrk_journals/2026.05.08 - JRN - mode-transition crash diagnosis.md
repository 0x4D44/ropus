## Hypothesis chain

1. **Initial hypothesis (2026-05-07 family)**: state-carry across frames where Rust's `&State` borrow misses a mutation. The 42-byte-prefix-match shape fits.
2. **Refinement 1 from `ROPUS_FUZZ_DUMP=1`**: only frame 2 differs; frames 0,1,3,4,5 match byte-for-byte. This is much stronger than "drift": the bug is **localized** to frame 2's encode and the encoder state converges back by end-of-frame-2.
3. **Refinement 2 from packet-layout reasoning**: with `nb_compr_bytes=41` and `redundancy_bytes=26`, packet layout is `TOC(1) + main_CELT(41) + redundancy(26) = 68`. The byte-42 divergence point is exactly the **start of the CELT->SILK redundancy frame**. Main CELT (bytes 1..42) matches — both encoders produce identical main-CELT output.
4. **Refinement 3**: frame 2 satisfies `prev_mode==CELT_ONLY && mode==HYBRID`, so `was_celt && !is_celt` at `encoder.rs:1928-1942` fires `redundancy=true, celt_to_silk=true`. The CELT->SILK 5 ms redundancy frame (`encoder.rs:3058-3084`) is the divergent code path.

The smoking gun: at the entry to the CELT->SILK redundancy encode the CELT state has
`disable_pf == 1` and `force_intra == 1`, both inherited from the prior frame's CELT
prefill block (`encoder.rs:3144`, `SetPrediction(0)`). These flags are read by the
CELT encoder during pitch prefilter selection (`celt/encoder.rs:2452`) and by
`quant_coarse_energy` (`celt/encoder.rs:2831`), so they materially change the encoded
bytes.

## Root cause

`SetPrediction(2)` is sequenced AFTER the CELT->SILK redundancy frame in Rust, but BEFORE it in the C reference.

C reference (`reference/src/opus_encoder.c:2284-2295`):

```c
celt_encoder_ctl(celt_enc, CELT_SET_END_BAND(endband));
celt_encoder_ctl(celt_enc, CELT_SET_CHANNELS(st->stream_channels));
celt_encoder_ctl(celt_enc, OPUS_SET_BITRATE(OPUS_BITRATE_MAX));
}
if (st->mode != MODE_SILK_ONLY)
{
    opus_val32 celt_pred=2;
    if (st->silk_mode.reducedDependency)
       celt_pred = 0;
    celt_encoder_ctl(celt_enc, CELT_SET_PREDICTION(celt_pred));   // <-- BEFORE redundancy
}
... tmp_prefill setup ...
... delay buffer / gain_fade / stereo_fade ...
... redundancy bits ...
if (redundancy && celt_to_silk) {                                  // line 2428
    celt_encoder_ctl(celt_enc, CELT_SET_START_BAND(0));
    celt_encoder_ctl(celt_enc, OPUS_SET_VBR(0));
    celt_encoder_ctl(celt_enc, OPUS_SET_BITRATE(OPUS_BITRATE_MAX));
    err = celt_encode_with_ec(celt_enc, pcm_buf, st->Fs/200,        // 5ms redundancy encode
                              data+nb_compr_bytes, redundancy_bytes, NULL);
    ...
    celt_encoder_ctl(celt_enc, OPUS_RESET_STATE);
}
```

Rust (`ropus/src/opus/encoder.rs:2880-2891` then `:3058-3113`):

```rust
celt.ctl(CeltEncoderCtl::SetEndBand(endband));
celt.ctl(CeltEncoderCtl::SetChannels(self.stream_channels));
celt.ctl(CeltEncoderCtl::SetBitrate(OPUS_BITRATE_MAX));
// (no SetPrediction here)
... delay buffer / gain_fade / stereo_fade / SetAnalysis / SetSilkInfo ...
// --- CELT->SILK redundancy frame ---  (line 3058)
if redundancy && celt_to_silk {
    celt.ctl(CeltEncoderCtl::SetStartBand(0));
    celt.ctl(CeltEncoderCtl::SetVbr(0));
    celt.ctl(CeltEncoderCtl::SetBitrate(OPUS_BITRATE_MAX));
    celt_encode_with_ec(celt, &pcm_buf, self.fs / 200, &mut redundancy_frame, ...);  // <-- with stale disable_pf/force_intra
    redundant_rng = celt.rng;
    celt.ctl(CeltEncoderCtl::ResetState);
}
celt.ctl(CeltEncoderCtl::SetStartBand(start_band));
if self.mode != MODE_SILK_ONLY {
    let celt_pred = if self.silk_mode.reduced_dependency != 0 { 0 } else { 2 };
    celt.ctl(CeltEncoderCtl::SetPrediction(celt_pred));   // <-- AFTER redundancy
}
```

Mechanism: `CELT_SET_PREDICTION(v)` writes two CELT state fields:

```rust
self.disable_pf  = (v <= 1) as i32;
self.force_intra = (v == 0) as i32;
```

(`ropus/src/celt/encoder.rs:3464-3470`). Both are read during a CELT encode:
- `disable_pf` gates the pitch-prefilter at `celt/encoder.rs:2452`.
- `force_intra` is passed to `quant_coarse_energy` at `celt/encoder.rs:2831` and changes the energy-quantisation bitstream.

Frame 2 enters with `disable_pf=1, force_intra=1` because of the prior frame's
prefill SetPrediction(0) (`encoder.rs:3144`). C clobbers them back to `0,0` at
line 2294 BEFORE the redundancy encode. Rust does not — the redundancy encode
runs with the wrong flags, producing the observed 26 different bytes (the entire
CELT->SILK redundancy frame region of the packet).

After the redundancy encode, `OPUS_RESET_STATE` (line 3082 / C 2441) wipes
`disable_pf` and `force_intra` back to 0 in both implementations, and the main
CELT encode that follows ends up with the same `disable_pf=1, force_intra=1`
state on both sides (Rust via prefill `SetPrediction(0)` at line 3150, C via the
same call at C 2485). That is why **only the redundancy frame diverges and
frames 3..5 reconverge**.

## Why this only fires now

The auto/forced mode distribution for `fuzz_encode_multiframe` did not push
through `CELT_ONLY -> HYBRID` on a frame that requires CELT->SILK redundancy.
Stream D's boundary-biased grammar over `(forced_mode, max_bandwidth, signal,
bitrate)` reaches this transition pair routinely, and the fuzzer found it in
~55 seconds. The same defect class as the 2026-05-07 wins — a control-flow
divergence between the C reference's per-frame setter ordering and Rust's,
exposed only by a specific mode-transition adjacency.

## Proposed fix

**Surgical fix (recommended)**: move the `SetPrediction(celt_pred)` call up to
the same point in the encode sequence as the C reference, i.e. immediately after
`SetEndBand`/`SetChannels`/`SetBitrate(MAX)` and BEFORE the CELT->SILK
redundancy block.

- File: `ropus/src/opus/encoder.rs`
- Approximate line range: insert ~5 lines around line 2891 (right after the
  existing `celt.ctl(CeltEncoderCtl::SetBitrate(OPUS_BITRATE_MAX))`), and
  delete the existing block at lines 3097-3107 (or comment-cite that it has
  been moved).
- New block, with C-cite:

```rust
// C: opus_encoder.c:2288-2295 — set CELT prediction BEFORE the CELT->SILK
// redundancy frame so that 5 ms redundancy encode sees the right
// disable_pf/force_intra. Without this, the redundancy frame inherits the
// stale prefill SetPrediction(0) from the prior frame's mode transition.
if mode != MODE_SILK_ONLY {
    if let Some(ref mut celt) = self.celt_enc {
        let celt_pred = if self.silk_mode.reduced_dependency != 0 { 0 } else { 2 };
        celt.ctl(CeltEncoderCtl::SetPrediction(celt_pred));
    }
}
```

Tradeoff: the `if self.mode != MODE_SILK_ONLY { SetPrediction }` block at
lines 3097-3107 should be removed (it's now redundant with the new earlier
call) — the prefill block at 3133-3143 still does its own `SetPrediction(0)`
after `ResetState`, which is correct.

**Confidence (empirical)**: I added an env-var-gated SetPrediction(celt_pred)
right before the redundancy block (matching the C order); the crash artifact
replays clean (`Executed ... in 55 ms` instead of `byte mismatch`). Then
removed the gate to confirm the original failure (which still reproduces
without the fix). This is a definite, surgical 1-block fix.

## Risks / open questions

- **Risk**: very low. The new ordering matches the C reference exactly, and the
  prefill block at lines 3133-3143 still runs its `ResetState + tmp_prefill
  encode + SetPrediction(0)` sequence inside the main CELT encode path —
  unchanged. The CELT->SILK redundancy block already does its own
  `ResetState` at line 3082, so any `disable_pf/force_intra` we set just
  before don't survive into the main encode anyway except via the prefill
  re-set.
- **Open question**: should the second `SetPrediction` block (currently lines
  3097-3107) be deleted entirely, or kept as a redundant safety call? C only
  has the one call (line 2294). Recommendation: delete it (cleaner,
  truly mirrors C).
- **Open question (minor)**: are there other reorder-class divergences hiding
  on the SILK_ONLY -> HYBRID -> CELT_ONLY edge that the boundary-biased
  fuzzer would surface in the post-fix re-run? My instrumentation only probed
  CELT-side state. If post-fix fuzzing finds another artifact in the same
  region within minutes, treat that as evidence of a sibling bug; otherwise
  this is the only one on this surface.
