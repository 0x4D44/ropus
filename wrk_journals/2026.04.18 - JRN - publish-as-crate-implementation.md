## Orchestration model

Per stage:
1. **Implement** — one (or parallel) implementation agents. Each gets the HLD + scoped task.
2. **Review in parallel** — `code-reviewer` agent + `devils-advocate` agent. Independent reads.
3. **Apply feedback** — if reviews raise issues, launch an apply-feedback agent with explicit file:line references.
4. **Supervisor gate** — read the diff, decide go/no-go.
5. **Commit** atomic per stage.

## Stages

1. Workspace skeleton + file moves
2. Library cleanup (features, env var, main.rs)
3. Peripheral migration (build.rs paths, fuzz, tooling)
4. Public API (re-exports, stability contract)
5. Metadata, LICENSE, README
6. Verification (MSRV, test, doc, dry-run publish)
7. Completeness audit vs HLD
