### Triage
Traced the call graph for each unused symbol:
- `banner::classify` — only the `#[cfg(test)] mod tests` inside the same
  file calls it. Production main.rs calls `classify_with_platform`. So
  `classify` is a test-only convenience wrapper that pre-dates the platform
  parameter being added.
- `platform::Outcome::not_claimed()` — only consumer is `banner::classify`,
  which is itself test-only. Cascaded dead code.
- `platform::outcome_without_claim()` — called by `not_claimed` (test-only
  cascade) plus `not_claimed_for_tests` and `failing_for_tests`, both
  already `#[cfg(test)]`. So in non-test builds it has zero callers.
- `BenchResult::skipped` — every caller is inside a `#[cfg(test)] mod tests`
  block (banner.rs, bench.rs, html.rs, report.rs). Production callers use
  `BenchResult::skipped_with_profile(...)` directly. Test-only convenience.
- `BenchProfile` import in banner.rs — only referenced inside the
  `#[cfg(test)]` test module of the same file.

Plan: gate each test-only helper with `#[cfg(test)]` rather than deleting,
because the tests genuinely use them. This is the minimal, honest fix —
no `#[allow(dead_code)]`, no resurrected callers that don't belong.
