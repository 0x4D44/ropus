## Lessons learned

1. **Multi-agent parallel races are real.** Stages 1+2 had no conflict (different files);
   Stages 6a+6b both touched `fuzz_repro_diff.rs` and Stage 6b's b120 trace was
   overwritten. For future parallel dispatches: explicitly partition file ownership
   between agents, or run sequentially.

2. **Hypothesis-attribution can be wrong even when the trace is right.** The Stage 4 impl
   agent labeled all 4 repros as G4 because the diff tool's hypothesis mapping was keyed
   off `boundary_id` blindly. The reviewers caught this; the apply-feedback agent fixed
   the call-out to be sub-field-aware. **Devil's-advocate caught critical bug that
   reviewer might have rubber-stamped.**

3. **The bug was upstream of the entire investigation.** V1 localized to b6 (post
   silk_encode_frame_FIX). V2 Phase C localized to b103 sub-field. Stage 6 Phase D probe
   showed Gains_Q16 already wrong at function entry. Final root cause was OUTSIDE the
   codec entirely — in the fuzz harness's modulus computation. **Maximum-skepticism
   review prevented a false codec "fix"** that would have masked the harness asymmetry
   while leaving real cx=10 testing gaps unexplored.

4. **The permanent diagnostic infrastructure has standalone value** even though the bug
   it was built for turned out to be elsewhere. The Phase C trace will be useful for any
   future bug inside `silk_encode_frame_FIX`.

5. **HLD V2 §3.1 multi-bug risk was correctly anticipated.** The 5 repros' divergence
   shape variation (Δtell=0/1/4/12) hinted at multiple root causes; Phase C confirmed two
   distinct hypothesis labels (G4 + G5). Both turned out to share the upstream harness
   cause, but the V2 protocol was correctly conservative.
