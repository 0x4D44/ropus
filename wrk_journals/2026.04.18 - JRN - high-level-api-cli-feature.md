## Stage 1d — Devil's advocate (completed)
DA verdict: **FIX-FIRST-THEN-COMMIT**. Three concrete risks ranked, three decisions to reverse, eight items "accepted too easily."

**Top three risks identified:**
1. **`OPUS_BITRATE_MAX = -1` collides with `OPUS_BAD_ARG = -1`** in `from_code` macro. Currently safe (no -1 fed to from_code in the live code path) but will silently misclassify if a future CTL constant equals an error code. Surgical fix: guard `if code >= 0 { return Self::Unknown(code); }` at the top of from_code.
2. **Zero behavioral test coverage of `api.rs`** — `cargo check` was the green light; no tests of `as_c_int` mappings, error round-trips, or end-to-end pipeline. DA also found a real inconsistency: `frame_size_from_pcm_len` validates `output.len() >= channels` for decoder (api.rs:516) but not encoder (api.rs:345). Zero-length encoder input would silently pass `frame_size = 0` to inner.
3. **`EncoderBuildError` may silently drop `OPUS_INTERNAL_ERROR`** from CELT init (encoder.rs:1742) into `Unknown(i32)`. Audit needed.

**Reversed (applied in fix round):**
- Macro retained but with negative-code guard.
- Behavioral tests added.
- Variant audit performed.
- `get_channels()` deletion contingent on grep result.
- Doc comments on identical error types.

**Rejected (devil's advocate did not win):**
- "Reverse the macro entirely" — keep, the negative-code guard fixes the actual bug; losing 70 LoC of saved boilerplate just for IDE go-to-def isn't worth it.
- "Rename `FrameDuration::Argument`" — keep; C parity wins for the porting story (will add doc comment).
