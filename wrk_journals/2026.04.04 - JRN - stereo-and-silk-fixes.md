## Key Findings

### Devil's Advocate: theta_rdo is the Real Stereo Blocker
- `theta_rdo = false` hardcoded in quant_all_bands (C sets true for stereo+complexity>=8)
- This disables encoder-side spectral reconstruction (resynth=false)
- ALL 30 stereo non-silence failures trace to this single hardcoded boolean
- Fix requires ~100 lines: save/restore encoder state, try two theta roundings, pick lower distortion
- Also found `fold_start` off-by-one (masked while resynth=false)

### SILK: Rate Control Loop Iteration Difference
- silk_gains_quant itself is CORRECT — identical outputs for identical inputs
- The encode_frame rate control loop iterates different number of times
- C: 6 iterations for frame 1, Rust: 5 iterations (both converge to same result)
- C: 2 iterations for frame 2, Rust: 3 iterations (DIFFERENT result due to extra iteration)
- Root cause is deep in the SILK rate control — different nBits counting or gainMult convergence
- Also found: TargetRate_bps computation completely wrong (passes raw bitrate)
- Also found: missing tot_blocks/curr_block maxBits frame splitting
- Also found: missing inDTX=useDTX assignment

### PVQ Search: Type Width Mismatch
- y[] is Vec<i32> in Rust, celt_norm (i16) in C
- Values can exceed i16 range after y[j]*=2 or y[j]+=2
- Added i16 truncation at 3 key assignment points
- Also found `extend32` was changed to truncate globally — REVERTED (breaks 34 call sites)
- Fix didn't immediately flip tests (values may not overflow in practice for these signals)
