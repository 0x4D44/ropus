## Critical Findings

### 1. Zeroed Y Buffer (devil's advocate — SHOWSTOPPER)
The Rust encoder passes a freshly-allocated **zero buffer** as the Y (second channel) data to `quant_all_bands`, instead of the second channel from `x_norm[n..]`. This means all stereo non-silence is encoded with a zeroed second channel. **This is the actual root cause of ALL stereo failures, independent of theta_rdo.**

One-line fix: `x_norm.split_at_mut(n)` to pass the real second channel.

### 2. Buffer byte save/restore (impl reviewer)
- `save_snapshot()` saves `buf[..offs]` (front) but C saves `buf[offs..storage]` (dirty region)
- `snapshot()` / `restore()` save only scalars (documented: "buffer NOT restored")
- V2 design correctly uses `snapshot()`/`restore()` for scalars + manual `buffer()` copy for bytes

### 3. BandCtx cannot be cloned (impl reviewer)
Holds `&mut EC`. V2 design saves/restores only scalar fields (seed, remaining_bits).

### 4. Missing nBitsExceeded update (SILK reviewer)
Was in V1 risks section, promoted to required in V2. Without it, bit reservoir feedback is permanently zero.

### 5. BITRESERVOIR_DECAY_TIME_MS missing (SILK reviewer)
Constant not defined in Rust. Must add: `const BITRESERVOIR_DECAY_TIME_MS: i32 = 500;`

### 6. fold_start do-while (impl reviewer + devil's advocate)
The fold accumulation loop in Rust uses `for fold_i in fold_start..fold_end` which may be zero iterations. C uses do-while (always at least 1 iteration). Currently dormant but activates with resynth=true.

## Design Decisions Made

| Decision | Rationale |
|----------|-----------|
| Fix zeroed Y buffer first, before theta_rdo | It's the root cause; theta_rdo is secondary |
| Use `snapshot()`/`restore()` for scalars + manual byte copy | Matches C exactly; existing methods support this |
| Add trait methods with `unreachable!()` defaults | Consistent with existing EcCoder pattern |
| Don't clone BandCtx; save/restore scalar fields only | `&mut EC` prevents clone; only scalars change |
| nBitsExceeded update is required, not optional | Without it, SILK target rate drifts silently |
