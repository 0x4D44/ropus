## M4b — collapse C++ shell + Rust FFI into a single DLL (`foo_ropus.dll`)

**Trigger:** user review of HLD after M4 landed. Collapse to one.

**Scope (this cycle):**

**Open risks:**

## M5 — wire the Rust FFI through the stub `input_impl`

**Trigger:** M4b proved the single-DLL build shape + symbol availability. M5 fills in the stub decode path so fb2k actually plays `.opus` files through ropus.

**Scope:**
1. `fb2k-ropus/src/input_ropus_opus.cpp` — replace stub bodies with FFI calls through `ropus_fb2k.h`. Keep the input_singletrack_impl surface unchanged; only bodies change.
2. `fb2k-ropus/src/dllmain.cpp` — bump C++ component version `0.1.0` → `0.2.0`; rewrite the description string (drop "M4 scaffold" language).

4. **No Rust side changes** — ropus-fb2k is stable; M5 is pure C++ wiring.

**Design (fixed by HLD §4.2 / §5.2 / §5.3 + `ropus_fb2k.h`):**

- Class member `RopusFb2kReader* m_reader = nullptr`; closed in destructor with null-guard.
- `IoCtx` struct (file ptr + abort cb ptr + `std::exception_ptr pending`). Lives as a class member; `abort` pointer refreshed before every FFI call.
- Four C callbacks wrap `file::read / seek / get_size / abort_callback::is_aborting`. Each catches any C++ exception, stores it in `IoCtx::pending`, returns -1. Post-FFI the C++ side checks `pending` first (rethrow original type), then `ropus_fb2k_last_error_code()` (map + throw).
- `open(info_read)` passes `ROPUS_FB2K_OPEN_INFO_ONLY`; `open(decode)` passes 0. Open failures map to `exception_io_unsupported_format` so fb2k falls through to the next decoder.
- `decode_run` allocates a stack/member buffer ≥ 5760×2 floats; `ropus_fb2k_decode_next` → populate `audio_chunk::set_data(buf, samples, channels, 48000)`.
- `decode_seek` seconds × 48000 → u64 samples → `ropus_fb2k_seek`.
- `decode_can_seek` returns `true` iff `m_file->can_seek()` AND `m_info.total_samples > 0`.
- `get_info` populates file_info (sample_rate, channels, duration, bitrate) + tags via `ropus_fb2k_read_tags` callback into `file_info::meta_add` + ReplayGain fields from `RopusFb2kInfo` into `replaygain_info`.
- `g_is_low_merit()` explicit override returning `false` (M4 relied on input_stubs default; HLD §5.3 wants explicit).

**Error mapping (HLD §5.2):**
- -1 BAD_ARG → `pfc::exception_bug_check_v2` (shouldn't happen post-validation)
- -2 IO → `exception_io(msg)`
- -3 ABORTED → `exception_aborted()`
- -4 INVALID_STREAM → `exception_io_unsupported_format(msg)` during open; `exception_io_data(msg)` otherwise
- -5 UNSUPPORTED → `exception_io_unsupported_format(msg)` during open; `exception_io_data(msg)` otherwise
- -6 INTERNAL → `pfc::exception(msg)` (Rust panic caught)
