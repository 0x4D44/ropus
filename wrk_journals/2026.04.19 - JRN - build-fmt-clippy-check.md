#### 2a. `[profile.test]` ignored
Cargo only honours `panic` on `dev`/`release`, not on `test`. The block was a
no-op that Cargo rightly complained about. Removed outright — test inherits from
`dev` (`panic = "unwind"`), so behavior is unchanged.

#### 2b. CState dead_code in c_ref_differential
`preemph_mem_e`, `tapset_decision`, `rng` were populated in both the C and Rust
snapshots but never compared. `preemph_mem_e` is *intentionally* excluded per a
documented representation difference (Rust stores `x`, C stores `coef0*x/2^15`).
`tapset_decision` and `rng` were placeholders — the C-side trace helper doesn't
surface them, so both snap paths hard-coded `0`, making any comparison vacuous.
Fix: remove the three dead fields; collapse their snap lines; add a `let _ =
(pme0, pme1);` to silence the FFI out-pointer values we still need to provide but
don't want to compare. Left a struct comment pointing to the omission so the next
person who wants coverage knows to extend the C helper.

#### 2c. unused_mut on `s2` in encoder.rs
The local `let mut s2: Vec<i16>` is populated via `collect()` and only read. The
`mut` was copy-paste from the sibling `s` binding (which does use `mut`). Dropped
the `mut`.
