#### 2a. `[profile.test]` ignored
Cargo only honours `panic` on `dev`/`release`, not on `test`. The block was a
no-op that Cargo rightly complained about. Removed outright — test inherits from
`dev` (`panic = "unwind"`), so behavior is unchanged.

#### 2b. CState dead_code in c_ref_differential
`preemph_mem_e`, `tapset_decision`, `rng` were populated in both the C and Rust
snapshots but never compared. `preemph_mem_e` is *intentionally* excluded per a
documented representation difference (Rust stores `x`, C stores `coef0*x/2^15`).
`tapset_decision` and `rng` were placeholders — the C-side trace helper doesn't
surface them, so both snap paths hard-coded `0`, making any comparison vacuous.
Fix: remove the three dead fields; collapse their snap lines; add a `let _ =
(pme0, pme1);` to silence the FFI out-pointer values we still need to provide but
don't want to compare. Left a struct comment pointing to the omission so the next
person who wants coverage knows to extend the C helper.

#### 2c. unused_mut on `s2` in encoder.rs
The local `let mut s2: Vec<i16>` is populated via `collect()` and only read. The
`mut` was copy-paste from the sibling `s` binding (which does use `mut`). Dropped
the `mut`.

### 5. Clippy
Starting count: 160+ warnings (including 102 near-identical missing-safety-doc on
capi C ABI functions) and 3 errors.

Errors fixed:
- `fargan.rs` test asserted `s >= i16::MIN && s <= i16::MAX` on an `[i16]`
  slice — a tautology that clippy correctly rejects. Rewrote to check the
  *real* invariant (the C reference clamps to `[-32767, 32767]`, deliberately
  one short of `i16::MIN`), which is the contract `synthesize_int` actually
  enforces.
- `coverage_tests.rs` had `assert!(dtx_found || true)` — an identity `true`
  assertion dressed up with a comment. Removed both the assertion and the
  `dtx_found` bookkeeping; the loop's `.unwrap()` already enforces "encoding
  does not panic".

Warning fixes (non-suppression, where feasible):
- `assert_eq!(x, true/false)` → `assert!(x)` / `assert!(!x)` across
  encoder/decoder/api tests.
- `a < lo || a > hi` → `!(lo..=hi).contains(&a)` in capi validation and
  one harness binary.
- `data.offset(off as isize)` → `data.add(off)` in `packet_parse.rs`
  (pad_info.offset is already `usize`).
- Manual memcpy loop → `copy_from_slice` in `interframe.rs`.
- `iter + index` loops → `iter().enumerate()` / `iter_mut()` across several
  files; preserved cursor-and-write patterns where needed.
- `match { Ok => … Err => {} }` → `if let Ok(…) = …` in a proptest assert.
- `std::iter::repeat(x).take(n)` → `std::iter::repeat_n(x, n)`.
- `x % 2 == 0` → `x.is_multiple_of(2)`.
- `if x >= N { x - N } else { 0 }` → `x.saturating_sub(N)`.
- `&vec![...; N]` → `&[...; N]` for a fixed-size test slice.
- `y > MAX { y = MAX } if y < MIN { y = MIN }` → `y.clamp(MIN, MAX)`.
- Extracted a long `(i32, …, Option<i32>)` tuple into a local `type` alias in
  `coverage_tests.rs` (clippy's `type_complexity`).
- Collapsed several `if let X { if cond { … } }` patterns into let-chain
  `if let X && cond { … }`.
- Promoted the `field_reassign_with_default` allow to the enclosing function
  in `interframe.rs` (~40-field snapshot; inlining every field into the
  struct literal would make it far less readable).
- Rewrote a doc comment paragraph in `tests/conformance/tests/encode.rs` to
  avoid clippy interpreting "+" mid-sentence as a list continuation.

Suppressions (with reasoning):
- Crate-level `#![allow(clippy::missing_safety_doc)]` on `capi/src/lib.rs`:
  every `pub unsafe extern "C" fn` in the crate shares the same safety
  contract (valid pointers, correctly-sized buffers, single-threaded handle
  use) already documented in xiph/opus headers.
- `#[allow(clippy::type_complexity)]` on the `Vec<(&str, Box<dyn Fn(usize) ->
  i16>)>` in `harness/src/main.rs`: a named `type PcmFn = Box<dyn Fn(...)>`
  alias changes the `'static` inference so the closures that capture
  `sample_rate` fail to satisfy the bound. The inline form keeps the
  closures' lifetimes flexible; keeping the clippy suppression local.
