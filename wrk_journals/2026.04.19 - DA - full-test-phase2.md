# DA: full-test Phase 2

## Top 3 challenges (supervisor should decide)

1. **Workspace self-measurement, unverified on Windows.** `cargo llvm-cov test
   --workspace` will instrument AND run `full-test`'s own 56 unit tests
   (`Cargo.toml:15` lists it as a member). `is_workspace_path`
   (`llvm_cov_parse.rs:140`) does not exclude `full-test/`, so its 1335 prod LOC
   contaminates the rollup. Nothing scrubs `LLVM_PROFILE_FILE` / `RUSTFLAGS` /
   `CARGO_LLVM_COV_*` before spawn ‚ÄĒ `cargo llvm-cov run --bin full-test` gets a
   profraw collision with the inner test run. Impl agent admits "Did NOT run Stage 2
   end-to-end" (journal ¬ß Verification); **coverage numbers this phase produces are
   unverified**. Fix: `--workspace --exclude full-test`, exclude `full-test/` from
   `is_workspace_path`, scrub the env vars before spawn.

2. **`--skip testvector` deviation: right direction, wrong evidence.** `cargo test -p
   conformance -- --list` returns exactly **30** tests (24 testvectors + 6 category
   probes: api/decode/encode/extensions/padding/projection). Impl agent's journal
   claims "~60" ‚ÄĒ off by 2x. Real number still argues for the deviation (the 6 probes
   matter), but amend the HLD so the justification isn't inflated. Secondary risk:
   libtest `--skip` matches **substrings**, not prefixes. A future test named
   `does_not_leak_testvector_path` or similar gets silently dropped. Cheap fix: use
   `--skip ::testvector` (matches path component only) or enumerate
   `--skip testvector0 --skip testvector1 --skip testvector2`.

3. **Partial-fetch blind spot.** `setup::ietf_vectors_present`
   (`full-test/src/setup.rs:102`) probes only `testvector01.bit`. An interrupted fetch
   delivering 1-10 but not 11-24 yields `present=true`, Stage 2 runs, tests 11-24
   panic at `File::open`, report red with no actionable signal. Two-line fix: probe
   all 24 `.bit` files (the harness also needs `.dec` / `m.dec`).

## Other pokes

4. **LOC ceiling blown.** Prod-only (excluding `#[cfg(test)]`): Phase 1 = 541, Phase 2
   adds 724 = **1265 prod LOC** already. HLD ceiling is ‚Č§1200 **total** with 4 phases
   to go. Either rebase the ceiling to ~1800 or declare the metric covers prod-only
   and accept we're 5% over. Silent drift is the worst option.

5. **`Outcome::skipped()` behind `#[cfg(test)]`** (`tests.rs:50-62`) exists only for
   `report.rs` test fixtures. A literal struct-init at the two call sites would be
   smaller. Delete in Phase 4 if still unused.

6. **`tempfile` dep:** defensible ‚ÄĒ already in `Cargo.lock` via fetch-assets, zero
   new transitive cost. But the handle is used as a path generator, not for its
   real value (secure-create, atomic unlink). `env::temp_dir().join(format!("ropus-cov-{}.json",
   std::process::id()))` is equivalent. Minor.

7. **`per_test` Vec ~1000 entries √ó ~150 bytes = ~150 KB live, ~500 KB serialised
   per report.** Fine for memory, but `ignored_test_names` + `failed_test_names` +
   `per_test[].fqn` triple-stores every FQN string. Fold to indices for ~30%
   savings. Phase 4 decision, not a blocker.

8. **Clippy 3-round iteration:** `module_inception`, `field_reassign_with_default`,
   `doc_overindented_list_items`. All cosmetic; none caught real bugs. Not sloppiness,
   not value ‚ÄĒ just fresh-crate noise. Not a red flag.

9. **Category-summary feeder data missing.** HLD ¬ß Output point 2 wants CELT / SILK /
   Opus / DNN roll-ups. Phase 2 only stores `fqn` and `binary`. Phase 4 HTML will
   re-derive by string-matching FQNs. Push the mapping into `cargo_parse` now (~20
   LOC helper) to avoid a Phase 4 detour.

10. **Coverage-skip reason lost in build-fail path** (`tests.rs:204-215`). Sets
    `coverage=None` silently. Stage 0's IETF-missing has a nice reason text; llvm-cov
    failure has none. HTML can't explain why coverage is blank.

11. **Test count ‚ÄĒ "11 per module" is close but not quite.** Real: cargo_parse 12,
    llvm_cov_parse 7, tests.rs 10. None are tautological ‚ÄĒ they encode the cargo
    output-format contract (real fixtures, not `strip_prefix` self-tests). Maintenance
    cost is low; keep.

## Defensible choices

- `--skip testvector` over `--exclude conformance` ‚ÄĒ direction correct; patch HLD.
- Parser tolerance (empty-on-malformed, no panics) ‚ÄĒ right call.
- `tempfile` dep ‚ÄĒ already in lockfile, zero new cost.
- `test result:` summary as source of truth ‚ÄĒ correct; matches human-reading of logs.
- Binary stem + 16-hex-hash strip for `BinaryResult.name` ‚ÄĒ stable across runs.
- Regex cache via `OnceLock` ‚ÄĒ idiomatic.
- No `--include-ignored` ‚ÄĒ matches HLD; don't relitigate.
- 1 MiB stderr cap shared from Phase 1 ‚ÄĒ good reuse.
