## Bugs Found

### Bug 1: `silk_add_pos_sat32` integer overflow — FIXED

**File**: `c:\language\mdopus\src\silk\common.rs:1041`
**Symptom**: `attempt to add with overflow` on u32 addition
**Root cause**: `a as u32 + b as u32` panics in fuzz builds (overflow checks enabled). C reference uses wrapping unsigned arithmetic.
**Fix**: Changed to `(a as u32).wrapping_add(b as u32)` — matches C behavior exactly.
**Found by**: fuzz_encode_multiframe within first 30 seconds of running.

### Bug 2: SILK encoder state divergence on multi-frame encoding — KNOWN/UNFIXED

**Config**: sr=24000, ch=2, app=VOIP, br=24000, cx=7
**Symptom**: Frame 0 matches byte-for-byte. Frame 1 diverges (first 3 compressed bytes match, then drift).
**Root cause**: Encoder internal state (prediction, gains) diverges between Rust and C after first frame. Classic state accumulation bug.
**Found by**: fuzz_encode_multiframe — validates this target's entire purpose.

### Bug 3: SILK decode PCM divergence on rich seeds — KNOWN/UNFIXED

**Config**: sr=24000, ch=1, pkt_len=31
**Symptom**: First ~480 of 960 decoded samples match, then drift. Typical SILK numerical drift pattern.
**Found by**: Enriched decode corpus (structurally valid TOC bytes with random payloads hit SILK decode paths that synthetic sine/noise seeds never reached).
