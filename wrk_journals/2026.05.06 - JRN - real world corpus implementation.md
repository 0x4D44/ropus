# Journal - Real World Corpus Implementation

Date: 2026.05.06

## Implementation Notes

- Read the signed HLD, HLD signoff, and corpus assessment before changing code.
- Kept the slice narrow: no codec implementation changes, no WebM parsing, no
  player/container semantics, and no committed media binaries.
- Hardened `corpus_diff` accounting so a candidate set with only skipped files
  or zero decoded audio exits non-green. Empty candidate directories still exit
  `2`; mismatches and panics still exit `1`.
- Added a machine-readable `CORPUS_DIFF_SUMMARY` line for `full-test` to parse.
- Added a `full-test` corpus stage. Default and quick profiles report status
  without claiming corpus coverage; non-quick `--release-preflight` generates a
  temporary FFmpeg-native `opus` smoke from `tests/vectors/48k_sine1k_loud.wav`
  and requires `corpus_diff` exact PCM parity against the C reference.
- Added `tests/vectors/real_world/corpus_manifest.toml` as tracked metadata for
  the first gateable generated non-reference smoke. Generated `.opus` media is
  still local/temp/ignored only.
- Updated full-test JSON/HTML/help/preflight wording so quick explicitly makes
  no corpus claim and non-quick release-preflight advertises the generated
  corpus claim.
- Updated README and the oracle matrix with the limited oracle caveat: exact PCM
  parity versus C reference for supported Ogg family-0 packet decode only; no
  WebM/player semantics, seek, output gain, pre-skip, granule, FEC, or PLC.

## Testing Intent

- Harness unit tests cover skipped-only, zero-audio-only, nonzero supported
  match, mismatch/panic, and empty candidate exit semantics without needing
  media fixtures.
- Full-test unit tests cover default/quick/non-quick policy, manifest parsing,
  missing FFmpeg/native-opus behavior, corpus summary parsing, JSON surface, and
  HTML wording.
- The actual generated-smoke command will be run if FFmpeg with native `opus` is
  available locally.

## Emotional Temperature

This is a good kind of testing work: the implementation is mostly about making
claims falsifiable. The main friction is avoiding overclaiming; it is tempting
to call the generated file "real-world", but it is more honest to call it a
generated non-reference smoke and leave curated corpus breadth for a later
asset/provenance decision.
