## Critical Bug Found: dynalloc_logp decrement

Through surgical bit-position tracing (C tell vs RS tell after each encode step), I found that the dynalloc encoding loop used 8 extra bits per frame.

**Root cause**: `dynalloc_logp = dynalloc_logp.max(2) - 1` (clamp then subtract) instead of `(dynalloc_logp - 1).max(2)` (subtract then clamp). When logp=2, Rust dropped to 1 while C stayed at 2. Each "no" flag with logp=1 uses ~1 bit vs ~0.42 bits at logp=2, totaling ~8 extra bits across 14 remaining bands.

**Impact**: This single-character fix (`max(2) - 1` → `(- 1).max(2)`) flipped 14 tests from FAIL to PASS.
