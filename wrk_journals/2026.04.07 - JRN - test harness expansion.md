## Bugs Discovered
- 102 sweep failures across VBR, VOIP, non-20ms frames, SILK, bandwidth forcing
- PLC output diverges at both 48kHz (CELT) and 8kHz (SILK)
- FEC encode/decode differs in VOIP mode
- DTX: Rust produces 0 DTX frames vs C's 39 on silence (DTX detection not ported)
- set_bitrate() was rejecting values that C clamps (fixed in Phase 6)

## Review Findings Applied
- Phase 1: Named constants instead of magic numbers, CTL return value checking,
  5 additional EncodeConfig fields (max_bandwidth, lsb_depth, prediction_disabled,
  phase_inversion_disabled, force_mode)
- Phase 2: DTX tests use generated noise/silence signals, 12kHz label fix,
  15 additional sweep cases (MEDIUMBAND, 16kHz long frames, FEC+stereo, LOWDELAY 8kHz)
- Phase 12: Exit code consistency fixes (fec, dtx, longsoak), test-all command
