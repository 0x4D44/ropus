# Multiframe Encoder State Divergence — Investigation & Fix

**Date:** 2026-04-12
**Origin:** Fuzz Campaign #4, Finding #3
**Bug:** Frame 1/5 Hybrid encoder byte mismatch (state accumulation divergence)

## Bug Summary

- **Trigger:** `mf5_24k_stereo_*_br24000.bin` seeds in `fuzz_encode_multiframe`
- **Config:** sr=24000, ch=2, app=AUDIO(2048), br=24000, cx=7
- **Mode:** Hybrid Wideband (TOC config 13)
- **Symptom:** Frame 0 bit-exact, Frame 1 diverges at byte 4 of 60 (Rust=0xA5, C=0xA6)
- **Root cause suspects** (from campaign investigation, 70% confidence):
  1. `silk_gains_quant` hysteresis chain (primary)
  2. `silk_hp_variable_cutoff` activation on frame 1 (secondary)
  3. `silk_encode_frame_fix` post-loop state updates (tertiary)

## Approach

**Phase 1: Reproduce & Pinpoint** — Build a state-comparison tool that encodes 2 frames through both Rust and C, then dumps and diffs key SILK encoder state after each frame. This tells us exactly which variable diverges first.

**Phase 2: Audit & Fix** — Line-by-line audit of the specific divergent code path against C reference, then fix.
