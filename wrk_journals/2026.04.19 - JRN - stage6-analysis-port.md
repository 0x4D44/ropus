**Code reviewer.** Ships with nits. Verified all 14 items:
  - Four flagged deviations each matched to specific C file:line.
  - `MLP` chain (LAYER0 → LAYER1 → LAYER2) walked line-for-line.

  add `new_boxed()`.

**Devils-advocate.** One FAIL (MUST-FIX) caught:
  - **M_PI truncation on MSVC.** C `analysis.c:49-50` provides the
    fallback `#define M_PI 3.141592653` (10 digits). MSVC's
    `<math.h>` doesn't define `M_PI` without `_USE_MATH_DEFINES`
    (which the harness config doesn't set), so C compiles with the
    truncated value. The Rust port used `core::f32::consts::PI`
    (full precision). DA identified this as a deterministic tier-1
    breaker.
  Three WEAKs: smoke-test numerical coverage is zero (recommended
  promoting to MUST-FIX); FFT bit-exactness is asserted-not-proven;
  reset uses explicit field assignments (struct-change fragile).
  Five PASSes (MLP chain, ring buffer, downmix type erasure,
  FIXED_POINT choice, log/log10 promotion).

**Feedback application.** All five fixes applied plus one
surprising find:
  - **M_PI fix was subtler than the brief specified.** The literal
    `3.141592653_f32` actually rounds to the SAME bits (0x40490FDB)
    as full-precision PI, so a straight constant swap buys zero
    ULPs. The drift comes from the computation path — C does
    `pi4 = M_PI^4` and `.5/M_PI` in double (because `M_PI` is a bare
    double literal in C), then casts to f32 at the end. Rust was
    doing it entirely in f32. Fixed by expressing the chain in f64
    and rounding once at the final cast.
  - Golden-vector differential test `test_analysis_run_matches_c_reference`
    added to `harness/tests/c_ref_differential.rs` (+530 lines). Live
    FFI differential — calls both C and Rust `run_analysis` with
    matching inputs, compares 5 `AnalysisInfo` scalars bit-exactly.
    Required widening `ropus::opus::analysis` to `#[doc(hidden)] pub`
    and bumping a handful of items from `pub(crate)` to `pub`. Added
    `run_analysis`, `tonality_analysis_init`, `tonality_get_info` FFI
    bindings to `harness/src/bindings.rs` (+16 lines).
  - `#[repr(C)]` added to `AnalysisInfo`.
  - `celt_mode` kept with explicit TODO comment about multi-rate
    support.
  - `new_boxed()` added using `MaybeUninit::new_uninit` + `write_bytes(0)`
    + init dance (MaybeUninit::assume_init is still unstable).

**Outcome — mixed.**
  Positive: M_PI issue fixed correctly. Three should-fixes applied.
  `new_boxed` ready for encoder embedding. Full ropus lib suite

  Negative: **the golden-vector differential test FAILS** on the

## Sub-stage 6.3b — diagnostic + fix — DONE

**Diagnostic agent.** Used the `#[repr(C)]` struct-layout-equality
trick: run both C and Rust `run_analysis` on one frame, then
byte-compare the full `TonalityAnalysisState`. First differing
field localizes the drift stage.

**Finding.** First divergence at pipeline stage 1
(`downmix_and_resample`). The `inmem` buffer on the Rust side was
**4096x smaller** than on C — exactly `1 << SIG_SHIFT` (SIG_SHIFT=12).
Smoking gun.

**Root cause — NOT the port.** The bug was in the test harness's
caller-supplied `DownmixFunc` callback
(`harness/tests/c_ref_differential.rs::rust_downmix_float`
and the sibling `test_downmix_float` in `analysis.rs`). Both
callbacks used the **non-FIXED_POINT** form of `FLOAT2SIG`:
`x * 32768.0 as i32`. But the harness builds with `FIXED_POINT 1`
(see `harness/config.h:8`), so C uses the FIXED_POINT inline function
at `reference/celt/float_cast.h:166-172`:

```
FLOAT2SIG(x) = { x *= 32768 << SIG_SHIFT; clamp; round-half-even; }
```

Missing the `<<SIG_SHIFT` factor, the clamp, and the rounding mode
produces exactly the observed ~4096x scale gap plus ULP drift.
Cascaded through FFT → band energies → BFCC → MLP = ~60 ULPs of
drift in `music_prob`. The `ropus/src/opus/analysis.rs` port
itself was correct all along.

**Fix.** Both test-harness downmix callbacks now implement the
FIXED_POINT `FLOAT2SIG` chain exactly: multiply by `134_217_728.0`
(= `32768 << 12`), clamp to `+/-(65536 << 12)`, `round_ties_even()
as i32`. Non-production test-only code; zero impact on the shipping
analysis module.

**Validation.**
- `test_analysis_run_matches_c_reference` passes bit-exactly.
- `#[ignore]` attribute removed.
- Full state byte-comparison confirms byte-equality after frame 1,
  excluding only the `arch` metadata tag (0 vs 4 for scalar vs SSE —
  no computational effect on x86 since `opus_fft` only SIMD-dispatches
  on ARM NE10).
- Added two `#[ignore]`d diagnostic helpers for future use.
- `cargo test -p ropus-harness --release --test c_ref_differential
  -- --include-ignored`: 19/19 pass.
- `cargo test -p ropus --lib --release`: 1804/1 pre-existing.

**Implication for Stage 6.4 (wiring).** The encoder's real downmix
callback must apply FIXED_POINT `FLOAT2SIG` to match
`reference/src/opus_encoder.c:748-778`. Exact scaling of downmix
output into `inmem` is load-bearing.

**Outcome.** Stage 6.3 closes tier-1 at module level. Ready for
Stage 6.4 wiring.
