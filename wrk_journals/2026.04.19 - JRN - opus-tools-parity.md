- **Implementer** agents (general-purpose, one per step) — write code + tests
- **Code-reviewer** agent (code-reviewer skill) — review the diff

  reviewer misses (different lens: what could go wrong, what's being
  silently assumed, what's out of spec)

Supervisor (me) stays hands-off on code; only reads summaries, routes
feedback, commits at stage boundaries.

### 2026-04-19 — step 4 done
- Every ropusdec flag wired except stdin/stdout: `--float`, `--rate`,
  `--gain`, `--no-dither`, `--raw`, `--packet-loss`.
- Resampler generalised from `resample_to_48k` → `resample(from, to)`.
- Float WAV writer (format code 3, fact chunk, fmt-18).
- TPDF dither (xorshift32, fixed seed).
- **Devil's-advocate caught THREE blockers the reviewer missed:**
  1. PLC on lost packets filled full 120 ms buffer regardless of
     stream's real frame duration — every simulated loss inflated
     output by up to 6×. Fixed: track `last_frame_samples` from
     the previous real decode; pass sized slice to PLC.
  2. Default path silently shifted every i16 sample ±1 LSB — the
     pipeline decoded via `decoder.decode` (i16) then roundtripped
     through f32 for dither. Fixed: decode_path-selection logic —
     i16 decoder only used for `--no-dither` without
     `--float`/`--rate`, everything else uses `decode_float` so
     dither applies to true f32 precision.
  3. Dither truncated (`as i16`) instead of rounding, biasing
     negatives +0.5 LSB. Fixed: `.clamp(...).round() as i16`.
- Also applied: short-stream resample padding; NaN guard on
  `--gain`; `--rate 48` "did you mean 48000" nudge.
- 66 lib tests + 10 integration tests all pass.
- Concurrent workstream committed as 7d64c30 between my Step 3 and
  Step 4. I diff-ed carefully and staged only my files.
