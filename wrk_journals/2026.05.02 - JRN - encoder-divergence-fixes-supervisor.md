# Encoder divergence fixes — supervisor journal

Follow-up to the 24h fuzz campaign (closed 2026-05-02 21:36 BST). Goal:
close the 4 real codec bugs surfaced during the campaign by dispatching
two HLD-first investigation agents, one per root-cause cluster.

## Mandate from Arthur

- Each agent writes an HLD up-front; supervisor reviews and signs off
  before any code is written. Agents iterate on the design until
  supervisor is happy.
- Agents follow good practice: use `/code-reviewer` skill and
  `/code-review-apply-feedback` skill during implementation.
- Maintain this journal; update regularly to disk.

## Workstreams

### Cluster A — Encoder state-accumulation
- Findings #7 `multiframe-cbr-cross-frame-divergence` (4 repros, MED)
- Findings #8 `multistream-encode-bytes-divergence` (4 repros, MED)
- Hypothesis: single root cause. multistream family=0 ch=1 is a thin
  wrapper over the regular encoder, so the divergence is plausibly the
  same cross-frame state-update mismatch that multiframe surfaces.
- Single-frame fuzzing is bit-exact today, so the bug is purely in the
  per-frame state update path.

### Cluster B — Float-PCM ingest
- Findings #9 `roundtrip-float-12k-cx7-vbr-divergence` (2 repros, MED)
- Findings #10 `encode-float-lowdelay-8k-divergence` (2 repros, MED)
- Hypothesis: shared root cause. Both go through `encode_float`. Likely
  candidates: `float2int16` rounding, float-domain analysis path,
  float-PCM resampling pipeline.
- Single-frame i16 fuzzing has been bit-exact since Stream A; only the
  float-PCM path diverges.

## Protocol

1. Each agent produces an HLD in `wrk_docs/2026.05.02 - HLD - <slug>.md`
   covering: scope, hypothesised root cause, instrumentation plan, fix
   plan, test plan, rollback. **No code in HLD pass.**
2. Supervisor reviews. Iterates until satisfied (correctness,
   tractability, test coverage, blast-radius).
3. On sign-off: agent implements, runs `/code-reviewer` on the diff,
   then `/code-review-apply-feedback` to action the review.
4. Final gate: harness `roundtrip` + relevant fuzz repros pass; full
   `cargo build` + `cargo test` clean; agent updates the
   `known_failures/` README to mark the class fixed.
