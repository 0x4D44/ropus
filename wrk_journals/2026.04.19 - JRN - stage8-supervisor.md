## Supervisor rules

1. Never read/write implementation source directly. Only read journals,
   HLD, review outputs, git log, test output.
2. Commit at end of each sub-stage (only after devil's-advocate has had
   its say and apply-feedback has run).
3. If a sub-stage fails tier 1, record the drop to tier 2 in the
   journal and keep going — do not redesign mid-stage.
4. If an agent returns "done" but tests weren't run, reject and send
   back. Trust-but-verify via `cargo test` exit status in their report.
5. Stage 7 owns DNN weight plumbing. We publish signatures; they wire.
   Do not cross that seam.
