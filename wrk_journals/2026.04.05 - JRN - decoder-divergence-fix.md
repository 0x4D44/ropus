## Decoder Bugs Found and Fixed (6 total, 3 commits)

### Bug 1: MDCT shift for transient frames
- **File**: `src/celt/decoder.rs` celt_synthesis()
- `shift = mode.max_lm - lm` unconditionally → should be `mode.max_lm` for transient
- Caused array-out-of-bounds panic on transient frames

### Bug 2-5: Dynalloc loop (4 sub-bugs)
- **File**: `src/celt/decoder.rs` dynalloc section
- Used integer `tell` instead of `tell_frac` (fractional bits)
- Missing `total_bits -= quanta` decrement after boost allocation
- Spurious `+1` in while condition
- Modified outer `dynalloc_logp` instead of local copy

### Bug 6: total_bits_alloc missing BITRES shift
- **File**: `src/celt/decoder.rs` quant_all_bands call
- Passed `data_len * 8` (plain bits) instead of `data_len * (8 << BITRES) - anti_collapse_rsv`
- Reduced max_diff from 54,809 → 370

### Bug 7: Anti-collapse bit read wrong function
- **File**: `src/celt/decoder.rs` anti-collapse section
- Used `decode_bit_logp(1)` (arithmetic coder) instead of `decode_bits(1)` (raw bits)
- Corrupted range coder state, causing rng/oldBandE divergence on frame 1

### Bug 8: Energy finalise ordering
- **File**: `src/celt/decoder.rs`
- Rust had: anti_collapse → unquant_energy_finalise
- C has: unquant_energy_finalise → anti_collapse (finalise modifies oldBandE used by anti_collapse)
