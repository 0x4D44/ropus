# 2026.05.06 - JRN - quick smoke stage2 timeout diagnosis

Arthur asked for diagnosis only: no source edits. The current patch appends
`--skip extensions` to Stage 2 for `--quick --release-preflight`, but the
plain Stage 2 workspace command still timed out at 300s.

Emotional temperature: mildly frustrated that the first fix optimized around a
single known slow conformance test while the command itself still means "run
nearly everything." That is an understandable local fix, but it was probably
too narrow for a smoke path.

What I checked:

- `full-test/src/tests.rs` still builds one Stage 2 cargo invocation:
  `cargo test --workspace --lib --tests --no-fail-fast -- --test-threads=1`
  with optional libtest `--skip` filters.
- `cargo test --workspace --lib --tests --no-fail-fast -- --test-threads=1 --skip extensions --list`
  completes quickly and shows that only the `tests/conformance/tests/extensions.rs`
  binary is filtered out. The rest of the workspace remains scheduled,
  including `ropus`'s 1850 library tests, conformance decode/encode, and
  deep-PLC harness tests.

Conclusion: Stage 2 after `--skip extensions` is spending time across several
heavy buckets, not one test. The `ropus` lib suite is about 91s by itself,
conformance decode and encode each exceed 60s, and deep-PLC/control takes about
62s and currently has a failing Tier 2 SNR test. More global libtest skip
filters would be brittle because names like `decode` and `encode` match huge
parts of the ordinary `ropus` suite too.

Recommendation: for `--quick --release-preflight`, change Stage 2 from a full
workspace invocation with skips to a narrower package profile. Keep full Stage
2 unchanged for non-quick validation. The minimal bounded profile should
exclude `conformance`, `ropus-harness-control`, and `ropus-harness-deep-plc`
from the quick-preflight Stage 2 workspace command. This preserves broad normal
Rust unit/integration coverage while removing the known exhaustive and unstable
harness suites from the smoke path.

Residual concern: even with those exclusions, `ropus --lib` is still about 91s
serially. That may be acceptable for "quick release preflight" if the target is
under 300s, but not if the target is a sub-minute smoke. If Arthur wants
sub-minute, Stage 2 needs an explicit smoke test list rather than a workspace
profile.
