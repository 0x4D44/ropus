# DA: full-test Phase 3

Scope: new `ambisonics.rs` + `bench.rs`; deltas to `main.rs`/`report.rs`. 34 new tests. Stage 3/4 never run live end-to-end (per journal).

## Top 3 challenges (supervisor should decide)

1. **Vector table is a fork of the shell script, not a mirror.** `bench.rs:40-91` clones `tools/bench_sweep.sh:20-31`. The guard test pins only first and last labels. No shared source of truth — no manifest, no grep-and-diff test. Add an 11th vector to the shell, forget the Rust table, divergence goes silent for months. Fix: extract to `tests/vectors/bench_manifest.toml` read by both, or add a test that parses the shell's `VECTORS=(...)` and diffs. Today's pinning test is theatre.

2. **Three separate cargo builds per full run is architectural.** Stage 2 = instrumented (`-C instrument-coverage`). Stage 3 = `cargo run --release` (different profile, rebuilds ropus clean). Stage 4 = `cargo build --release --bin ropus-compare` (third configuration). Cold cache: ~2-3 min × 3 = ~10 min on build artefacts that could share one tree. HLD's "Stage 3 ≈ 15 sec" silently assumes warm cache that Stage 2's instrumentation just poisoned. Fix: one `cargo build --release --bin ropus-compare --bin projection_roundtrip` up front, shared by 3/4. Or gate 3/4 behind `--skip-coverage`.

3. **Bench is never FAIL — even when `ropus-compare` panics.** `BenchResult::all_passed() -> true` hardcoded (`bench.rs:143`), locked by `bench_result_always_all_passed`. A crash populates `skip_reason` with stderr line 1 and the row becomes "skipped", indistinguishable from "WAV missing". Defensible for **bench anomalies** (HLD is explicit). But it collapses two modes: (i) 10× slow regression → valid WARN vs (ii) segfault → real bug masked as WARN. Track crashes separately; they shouldn't look like missing fixtures.

## Other pokes

4. **`c > 0.0` guard swallows real bugs.** A 0.0 C baseline on 10s audio at 128k is impossible on a working build — timer or unit bug. Silent `None` hides it; log a WARN before the fall-through.

5. **`OrderOutcome::detail: String` is unbounded.** `parse_order_line` takes everything after `:` verbatim; a future hex dump lands in JSON then HTML. Cap to ~256 bytes.

6. **Upstream-build-failure chain hides bench signal.** Stage 2's llvm-cov instrumentation breaking (HLD-listed risk) skips 3/4 — but their release builds are independent. Try their build anyway; surface the real error or run bench.

7. **`prebuild_ropus_compare` greps only `error[E` / `error: `.** OOM'd rustc, link failures, OS-kills match neither; the fallback eats the real cause.

8. **7 of 34 tests re-assert `all_passed()` across shapes.** Collapse to parametrised; keep the load-bearing parser tests.

9. **LOC ceiling.** Phase 2 DA flagged it; Phase 3 blew past 1200 silently. Rebase the HLD or Phase 4 has a problem.

## Defensible choices

- **Exit-code-101 vs binary non-zero** (`ambisonics.rs:138`) distinguishes cargo compile failure from runtime fail — avoids Phase 1 DA's pattern-match poke.
- **`build_cargo_args` / `build_vector_row` factored for pure unit tests** — follows Phase 2 discipline.
- **Unicode `│` split with `cols.len() < 5` guard** — matches harness exactly, partial-table-tolerant.
- **Per-vector WAV-missing → skipped** — mirrors shell script.
- **`starts_with("tests/vectors/")` assertion** — catches copy-paste escapes cheaply.

## Phase 5 risk prediction

**Stage 3 and Stage 4 race on `target/release/libropus.rlib`.** Stage 3 does `cargo run --release -p ropus-harness --bin projection_roundtrip` — compiles `projection_roundtrip` in the `harness` crate. Stage 4 then `cargo build --release --bin ropus-compare` — also in `harness`, but a different bin with potentially different feature resolution. If cargo decides features differ, Stage 4 rebuilds **all of ropus** on what Arthur expects to be a warm tree. The 25-min bench budget becomes 50+. Silent in JSON; only wallclock shows it. Mitigation: single up-front `cargo build --release --bin ropus-compare --bin projection_roundtrip`, then Stages 3/4 spawn binaries directly.
