## Root Cause

Existing unit tests used tolerance-based assertions:
- `(reconstructed - true_energy).abs() <= true_energy` (100% tolerance!)
- `(result - expected).abs() <= 2`
- `error < 0.05` (5% relative)

These wide tolerances mean arithmetic mutations (+ ↔ -, * ↔ +, << ↔ >>) survive
because the mutated output is still "close enough" to pass the property check.
