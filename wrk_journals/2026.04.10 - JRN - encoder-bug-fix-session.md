### T+8 — Remaining issues analyzed

**Bitrate extremes (209 mismatches):** Mode selection bug. At 6kbps after
510kbps, C chooses SILK NB 10ms, Rust chooses CELT FB 20ms. The configs
use `force_mode=AUTO`, `max_bandwidth=FB`. This is a bandwidth decision
bug where the Rust encoder fails to narrow bandwidth after an extreme
bitrate drop. Different bug class from the SILK transition fixes.

**FEC (5 mismatches):** Config changes packet_loss_pct from 5% to 30%.
LBRR (SILK's FEC mechanism) configuration may differ. Down from 104
mismatches — the main transition fix resolved most of it.

**VOIP/DTX (1 decoder mismatch each):** Encoder packets match (enc_eq=true)
but decoder produces different PCM by ~4 samples. Pure decoder bug at mode
transition frames. Minor and separate from encoder fixes.

## Session Summary

### Bugs fixed (4 commits)

1. **Bug 6** — CELT `patch_transient_decision` i16 truncation (`6500be6`,
   concurrent). C's `opus_val16` implicitly truncates Q24 log-energy to i16;
   Rust was keeping full i32.

2. **Bugs 8/9/10** — SILK encoder mode transition (`1500771`). Three root
   causes: (a) missing VAD warmup during prefill, (b) missing
   `controlled_since_last_payload = 0` reset after prefill, (c) missing
   `in_dtx = use_dtx` initialization per frame. Also: LBRR flags reset on
   transition and transition detection variable.

3. **Bug 7** — SILK decoder stereo M/S delta truncation + decode_signs
   bounds check (`9a449e3`). The stereo M/S-to-LR delta computation used
   `wrapping_mul` without `silk_smulbb`'s i16 truncation. Also fixed
   `silk_decode_signs` which incorrectly bounded sign bit consumption
   against `frame_length`, desyncing the range coder for non-multiple-of-16
   frames.

   0x78 (Hybrid FB) to 0x48 (SILK WB). Correct mode now, but bandwidth
   still diverges (WB vs NB) due to `allow_bandwidth_switch` state issue.

   when packet_loss_pct changes from 5% to 30%.

3. **1-mismatch decoder issues** — encoder packets match but decoder PCM
   differs by ~4 samples at mode transition frames. Pure decoder bug.
