### PASS/FAIL/WARN semantics

`banner.rs:53-58` (PASS/WARN → 0, FAIL → 1), verified in
`banner::tests::exit_code_mapping`.

## Gaps found

All non-blocking:

1. **LOC ceiling breached (non-blocking, HLD amendment)**: prod LOC (excluding
   `#[cfg(test)]`, comments-only, blanks) is **~2116**; HLD § Success criteria
   sets ≤1200. Test-code LOC (1531) pushes total to 3647. The overrun is driven
   by HTML rendering (522 prod LOC), bench stage (338), cargo parser (208), all
   justifiable for the responsibilities. Recommend amending HLD to ≤2500 prod
   LOC.
2. **`banner` not in JSON envelope (non-blocking)**: `report::Envelope::to_json`
   (`report.rs:25-36`) emits `exit_code` but not `banner`. A downstream parser
   must reconstruct WARN vs PASS from indirect signals. Noted in Phase 4 DA.
3. **TODO comment (non-blocking)**: `bench.rs:379` has a `TODO:` pointing at
   the harness bench printer as source-of-truth. It is a cross-reference, not
   an open work item; safe to leave.

## HLD amendments recommended

1. Raise § Success criteria LOC cap from 1200 to ~2500 prod LOC, or remove the
   numeric cap entirely. The existing figure was explicitly flagged as
   aspirational in Phase 4.
2. Add optional `banner` field to JSON envelope for CI parsers (out of scope
   for this audit — noted in Phase 4 DA already).

## Targeted gap-check answers (from audit brief)

1. **Bench column layout vs README (lines 115-127)**: HTML shows 7 columns
   (Vector + 2 raw C/Rust ms + ratio, for encode AND decode); README shows 3
   columns (Vector, Encode, Decode — ratio only). The HTML is a superset —
   raw timings + ratios. Satisfies HLD "matching" (ratio column layout
   preserved). Not a gap.
2. **`--help` coverage**: Every flag documented, `--emit-json` included
   (`cli.rs:65`). Verified at runtime. Not a gap.
3. **Commit-subject XSS escaping**: `html.rs:150` routes `commit_subject`
   through `html_escape`; regression-tested by `metadata_values_escaped`
   (`html.rs:932-962`). Not a gap.
4. **JSON envelope round-trip**: Runtime check with
   `--emit-json --skip-quality --skip-coverage --skip-benchmarks
   --skip-ambisonics` produced a well-formed JSON document on current master.
   Not a gap.
5. **Missing `tests/results/`**: `write_report` (`main.rs:194-203`) calls
   `create_dir_all`, creating the directory. Not a gap.
6. **`--skip-coverage` + `--no-fail-fast` guard**: correct.
   `build_cargo_args` (`tests.rs:94-115`) uses `--no-fail-fast` on the plain
   `cargo test` path (line 95), `--ignore-run-fail` on the llvm-cov path
   (line 107). Regression-tested by `skip_coverage_builds_plain_test_command`
   and `coverage_on_excludes_explicit_no_fail_fast_flag`. Not a gap.
7. **TODO/FIXME/XXX**: one `TODO` (informational cross-reference at
   `bench.rs:379`); the `XXX` match was inside a docstring listing the regex
   pattern `E0XXX` (`issues.rs:61`) — false positive. Non-blocking.
