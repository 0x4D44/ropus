## Warnings Found & Root Causes

### DNN feature-flag unit-type pattern (8 warnings)
When `feature = "dnn"` is disabled (default), `DnnPlcArg<'_>` is `()`. This causes:

1. **`opus/decoder.rs:22`** - `celt_lpcnet_noop()` explicitly returns `()` -> `unused_unit`
2. **`opus/decoder.rs:919,972,994`** - Passing `celt_lpcnet_noop()` (= `()`) to `decode_with_ec` -> `unit_arg`
3. **`celt/decoder.rs:920`** - `let lpcnet = lpcnet;` where lpcnet is `()` -> `redundant_locals` + `let_unit_value`
4. **`celt/decoder.rs:1545`** - `let _ = lpcnet;` where lpcnet is `()` -> `let_unit_value`
5. **`silk/decoder.rs:2245,2509`** - Same `let _ = lpcnet;` pattern -> `let_unit_value`

**Fix**: `#[allow(clippy::unused_unit)]` on `celt_lpcnet_noop`, `#[allow(clippy::unit_arg)]` at call sites, `#[allow(clippy::let_unit_value)]` on `let _ = lpcnet;` statements. These are legitimate cfg-conditional patterns, not dead code.

### Complex return type (1 warning)
6. **`opus/encoder.rs:2950`** - `get_silk_state()` returned `Option<(i32 x 13)>` -> `type_complexity`

**Fix**: Introduced `SilkEncoderSnapshot` struct with named fields. Updated producer in `encoder.rs` and consumer in `tests/harness/main.rs`.

### Unneeded return (1 warning)
7. **`tests/harness/main.rs:5586`** - `return None;` at end of cfg block -> `needless_return`

**Fix**: Changed to implicit `None` in both Windows and Linux cfg blocks.

## Iteration Notes

- `#[allow(clippy::let_unit_value)] let _ = lpcnet;` is the only clean approach for consuming a `()` parameter
