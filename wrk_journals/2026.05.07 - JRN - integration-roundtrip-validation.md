### Real find — 5.1 multistream LFE byte divergence
- **Cell**: 6-channel mapping_family=1, 4 streams, 2 coupled (L+R, Ls+Rs)
  + 2 mono (C, LFE).
- **Symptom**: frame 0 packet differs. Rust 1740 bytes, C 1742 bytes.
  Common prefix ~1729 bytes (99.4%). Divergence in the last ~13 bytes —
  that's the tail stream's payload, i.e. the LFE mono stream.
- **Hypothesis** from implementation agent: per-stream rate allocation /
  CTL fan-out for LFE differs by ~16 bits, suggesting
  `OpusMSEncoder::set_bitrate` LFE bit-budget split.
- **Why this matters**: this is exactly the May 4 multistream analysis-
  plumbing-fix area, validated only against 2-stream dual-mono. 5.1 was
  uncovered. We just closed that gap and immediately found the bug it was
  hiding.
- **Determinism**: frame 0, no state warm-up, signal-dependent. Easy
  bisect target.
