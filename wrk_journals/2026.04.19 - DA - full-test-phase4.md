# DA: full-test Phase 4

Scope: `html.rs` (1212/584 test) + `banner.rs` (447/334 test) + `main.rs`/`Cargo.toml` deltas. Uncommitted.

## Top 3 challenges

1. **Category bucket fallback silently pollutes "Types."** `html.rs:260` routes every unknown FQN head into Types. A future `ropus-fb2k` test, a `harness::` integration binary, or any lone-fn FQN lands in Types with no audit trail — the bucket becomes "I don't know + Types." Fix: an explicit `Other` bucket rendered only when populated, or an eprintln when the default branch fires. Miscategorising silently is strictly worse than "no data."

2. **The `<meta name="bench_ratios">` format is a write-only homegrown DSL.** Escaping IS implemented (`html.rs:609`) and tested (`"A|0.8500|1.1000;B__escape|-|-"`, line 1167). But (i) no parser exists — the regression-vs-previous feature promised in HLD § Decisions is absent; (ii) `|`/`;` stripping is lossy — `"SILK;WB"` and `"SILK|WB"` collide; (iii) code writes `-`, docstring says `—` (em-dash). Fix: `serde_json::to_string(&Vec<(label, enc, dec)>)`. Same tag, escape-safe, parser is `from_str`. Ship before the format calcifies.

3. **`BENCH_WARN_RATIO = 1.15` is a guess, not evidence.** Hardcoded, strict-greater (`bench_at_threshold_is_not_warn` locks it). No measurement of `--iters=30` variance floor — plausible σ could exceed 0.05, making the threshold both below the noise floor AND blind to a 1.00→1.14 regression. Either (a) pick from three baseline runs' observed σ × 3, (b) expose as `--bench-warn-ratio`, or (c) defer until the previous-report comparison (challenge 2) exists. Today WARN is cosmetic.

## Other pokes

4. **Exit-0 WARN is silent in CI.** Stdout prefixes `"WARN —"`, nothing parses it. `--emit-json` carries `exit_code` but not `banner`. Add `banner: "WARN"` to the envelope.

5. **Coverage "unavailable" reason renders verbatim.** `html.rs:468` prints whatever `coverage_skip_reason` holds — Arthur sees `"malformed_json: expected , at line 12"` accurate but useless. Collapse the three known reasons into enum cases with actionable text.

6. **Banner logic is explicit and correct.** `banner.rs:81-86` early-returns FAIL on three predicates; ordering is not match-arm-dependent. The `build_failed=true && all_passed()=true` case is trivially FAIL via `||`. Nothing to challenge.

7. **Module breakdown scale fears are unfounded.** ropus has ~30-50 modules — scrollable alphabetical, not 2000 rows. Revisit post-1.0.

8. **Banner test LOC is heavy.** 113 prod / 334 test — 3:1. Seven of thirteen `#[test]`s are one-liner `assert_eq!(classify(...), Variant)`. Collapse with a table. html.rs at ~1:1 earns its keep via XSS tests; banner's don't.

9. **`chrono` re-add costs nothing.** `default-features=false, features=["clock"]` pulls only `num-traits` + (Windows) `windows-targets`, already in `Cargo.lock`. Not the scope creep Phase 1 CR feared.

10. **Flag-note HTML is trusted-literal.** `html.rs:177` bypasses `html_escape` because strings contain `<code>`. Fine today; a trap for a future `format!("Flag: <code>{}</code>", user_supplied)`. Add `// INVARIANT: static literals only`.

## Defensible choices

- Category render order matches HLD list, not BTreeMap alphabetical (`html.rs:219`).
- `html_escape` on every variable including commit SHAs — `metadata_values_escaped` is load-bearing.
- Stage 4 rendered with `warn` class, not `fail` — matches HLD.
- `colspan="6"` for skipped/crashed rows; the crashed-vs-skipped distinction from Phase 3 DA survives into rendering.
- `resolve_commit_subject()` returns `"(no subject available)"` outside a git checkout — report always renders.

## Phase 5 failure prediction

**Three independent cargo builds blow the 75-min budget past 90+ on cold Windows.** Phase 3 DA flagged this; Phase 4 didn't fix it. Stage 2 = `cargo llvm-cov test --workspace` (instrumented profile); Stage 3 = `cargo run --release -p ropus-harness --bin projection_roundtrip` (different profile, full ropus rebuild); Stage 4 = `cargo build --release --bin ropus-compare` (feature resolution may differ from Stage 3's bin → another rebuild). HLD's "Stage 3 ≈ 15 sec" assumes a warm cache Stage 2's `-C instrument-coverage` just poisoned. First real run: Arthur sees 95-min wallclock with the HTML reporting 45-min tests + 25-min bench + 15-min of invisible rebuilding between stages. Visible only in wallclock; absent from HTML and JSON. Fix (unchanged from Phase 3 DA): one up-front `cargo build --release --bin ropus-compare --bin projection_roundtrip` after Stage 2, share the tree, spawn binaries directly.
