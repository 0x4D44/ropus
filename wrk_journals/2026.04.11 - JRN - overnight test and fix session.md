### 22:15 - Investigation Results & Fixes

**Failure #2 root cause:** Feature-gate issue. Test expects PitchDNN panic when no model loaded, but without `--features dnn`, a no-op stub runs. Test needs `#[cfg(feature = "dnn")]`.

**Failure #3 root cause:** Two bugs:
- `PitchDNNState` stub (no-dnn) unconditionally returns 0, ignoring invalid input
- `LPCNetPLCState::load_model` silently discards fargan error, returns success code

**Fix agent (fix-lpcnet)** applied 4 changes to `src/dnn/lpcnet.rs`:
1. Added `#[cfg(feature = "dnn")]` to `test_compute_frame_features_updates_state_before_pitchdnn_panic`
2. Also gated the `use std::panic::...` import used only by that test
3. PitchDNN stub `load_model` now validates via `parse_weights(data)` instead of returning 0
4. `LPCNetPLCState::load_model` now propagates fargan error (`return ret2`)
5. Updated `test_plc_load_model_accepts_encoder_blob_but_not_fargan_blob` expectation to match fix

All 135 DNN tests pass after fix.

**Failure #1 resolution:** Opus encoder test now passes consistently (5/5 runs). Likely test-ordering dependency in the initial parallel run. Resolved by recompilation after lpcnet fix.
