## 22:05 BST - Root cause isolated

Ran `fuzz_repro_diff` against all seven current artifacts. Four were already byte-exact, three still diverged on frame 1: `crash-55d77303fe6edf000bcc3c3ad99a94d586023e08`, `crash-654607c9e9877057a733e156a8624db30b5eb10f`, and `crash-audio-cx9.bin`.

First-divergent-state evidence on `crash-55...`:
- Frame 0 bytes matched, but frame 1 diverged.
- Phase B showed `n_bits_exceeded`, LBRR counters, and target rate matched until after stereo LR-to-MS.
- Phase C first divergent boundary was b100, immediately after `silk_hp_variable_cutoff`; Rust's SILK input buffer started with prior stereo memory while C's started with zeros.
- C `enc_API.c` copies channel 0 resampler state into channel 1 after `silk_control_encoder` for mono-internal to stereo-internal transitions (`nPrevChannelsInternal == 1 && id == 0`). Rust only copied during transition setup, before `silk_control_encoder` could rewrite resampler state.

Fix applied in `ropus/src/silk/encoder.rs`: mirror the C post-control resampler-state copy immediately before stereo input resampling.

Emotional temperature: better. This is the kind of small, parity-shaped fix I wanted; the diagnostic traces kept this away from speculative rate-control changes.

## 22:42 BST - Coverage and post-skip artifact

Migrated all seven current known-failure crash files to `tests/fuzz/seeds/fuzz_encode_multiframe/` and removed the broad `sr=8000 && ch=2` filter. Re-ran the seven migrated seeds through `cargo +nightly fuzz run --fuzz-dir tests/fuzz fuzz_encode_multiframe <seed> -- -runs=1`; all passed, including `crash-audio-cx9.bin` as `regression-vbr-fec-dtx-audio-cx9.bin`.

A 60-second fuzz smoke then found a new unmasked artifact, `crash-09964c78518daec23e18bf4fa69764095aba4893`, with a CBR SILK-to-CELT transition returning Rust=97 bytes vs C=92 bytes. First-divergent-state evidence showed `n_bits_exceeded`, LBRR counters, and range-coder trace matched through SILK, but Rust kept too many SILK-only redundancy bytes. Root cause: Rust clamped `redundancy_bytes` only in HYBRID mode, while C clamps it for both HYBRID and SILK-only using `max_redundancy = (max_data_bytes - 1) - ((ec_tell + 7) >> 3)` in SILK-only. Added the missing SILK-only clamp and added the repro as `regression-vbr-fec-dtx-09964c78.bin`.

Also aligned standard NSQ final `sLTP_*_buf_idx` state with C by not resetting those indices after the final buffer memmove. That removed the diagnostic extended-state mismatch on the smoke artifact; the byte fix was the redundancy clamp.
