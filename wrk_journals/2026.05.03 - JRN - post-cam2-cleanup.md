## Stage 1b Worker C Results

Worker C closed the `crash-02af2a155f9ad590f9ecae56fff5c752c8e848a4` artifact.

Root cause:

- The packet symptom looked like CBR padding/framing because C produced a shorter SILK payload then padded to target bytes, while Rust spent one extra SILK byte.
- The upstream cause was `silk_setup_resamplers`: C initializes the temporary internal-rate-to-API-rate correction resampler with `silk_resampler_init(..., 0)`, i.e. decoder delay semantics. Rust used the encoder wrapper, forcing `for_enc=true`, which chose the wrong delay matrix during a 12 kHz internal to 8 kHz internal rate switch.
- Worker C changed only the temporary correction resampler to `for_enc=false`; the actual API-to-new-internal encoder resampler remains encoder-side.
