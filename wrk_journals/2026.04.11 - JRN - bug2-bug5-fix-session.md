### T+23min — Bug #5 FIXED

**Root cause confirmed:** `silk_inner_prod_aligned_scale` at `src/silk/encoder.rs:3113` used i64 accumulation. C reference at `reference/silk/inner_prod_aligned.c:42` uses i32 with wrapping.

**Fix applied** (4 lines changed in `src/silk/encoder.rs:3113-3118`):
- `i64` accumulator → `i32`
- `(x[i] as i64 * y[i] as i64) >> scale` → `(x[i] as i32 * y[i] as i32) >> scale`
- `sum += ...` → `sum = sum.wrapping_add(...)`

**Regression test added** at `src/silk/encoder.rs:9677`: `test_bug5_encode_8khz_stereo_71kbps_matches_c_reference` — encodes 10 frames of LCG-generated stereo PCM at exact fuzz config, compares byte-for-byte with C.

### T+35min — Bug #2 FIXED

**Reproducing packet:** TOC `0x60` + 255 bytes of `0xFF` at sr=48000, ch=1.

**Root cause:** Missing ICDF table offset in `silk_decode_pulses` at `src/silk/decoder.rs:688`. The C reference at `silk/decode_pulses.c:75-76`:
```c
sum_pulses[i] = ec_dec_icdf(psRangeDec,
    silk_pulses_per_block_iCDF[N_RATE_LEVELS-1] + (nLshifts[i] == 10), 8);
```
The `+ (nLshifts[i] == 10)` offsets the ICDF table by 1 byte after 10 LSB shifts, preventing the overflow symbol from being decoded again. The Rust code was missing this offset, causing the decoder to consume vastly different numbers of bits from the range coder (6514 vs 3508 bits), which cascaded into completely different CELT decode input.

**Primary fix** (3 lines in `src/silk/decoder.rs:688`):
```rust
let table_offset = if n_lshifts[i] == 10 { 1 } else { 0 };
sum_pulses[i] = rc.decode_icdf(&SILK_PULSES_PER_BLOCK_ICDF[N_RATE_LEVELS - 1][table_offset..], 8);
```

**Secondary fix** (2 lines in `src/silk/common.rs:533,542`):
- Changed `silk_add_sat32(x, -y)` → `silk_sub_sat32(x, y)` in `silk_lpc_inverse_pred_gain`
- Fixes edge case when `y == i32::MIN` (negation wraps to `i32::MIN`, giving wrong result)
- C reference uses `silk_SUB_SAT32(x, y)` which handles this correctly

**Tests added:**
- `test_garbage_hybrid_swb_decode_matches_c_reference` — decodes garbage hybrid SWB packet, compares PCM
- `test_lpc_inverse_pred_gain_matches_c_reference` — differential test with 1000+ random coefficient patterns

**Code review:** Passed. Fix verified correct against C reference.
