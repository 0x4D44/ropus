#### Pre-existing flake — `opus::analysis::tests::test_new_boxed_is_equivalent_to_new`

The test compares `TonalityAnalysisState::new()` (stack alloc) against
`TonalityAnalysisState::new_boxed()` (heap alloc via `alloc_zeroed`).
It passes when run alone, fails when run after other tests in the
suite. Even with `--test-threads=1` it fails. The most likely
explanation: `new()` returns a state with non-zero stack-frame
padding bytes that `alloc_zeroed` doesn't carry. Pre-existing —
none of my edits touch `analysis.rs` or `TonalityAnalysisState`.
Suggest a separate fix-it ticket; out of scope for this PR.
