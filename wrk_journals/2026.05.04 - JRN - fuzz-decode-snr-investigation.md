# 2026.05.04 - JRN - fuzz decode SNR investigation

## 12:05 - Start

Arthur asked me to investigate a reproducible `fuzz_decode` smoke failure:
`Float SILK/Hybrid decode SNR 35.99 dB < 50 dB`, artifact
`tests/fuzz/artifacts/fuzz_decode/crash-cee7d75d4bcc68daf712a9c721dc36c0b1b2ad2f`.

Initial observations:
- Workspace instructions read from `AGENTS.md`.
- Existing dirty files are journal-only and pre-existing; I will not touch them.
- The artifact is 130 bytes.
- Prior project notes mention the SILK/Hybrid SNR oracle as a known fragile/possibly over-broad fuzz oracle, so I need to avoid assuming this is a codec regression before replaying and classifying it.

Emotional temperature: cautious and slightly wary. The failure smells like a known oracle-calibration class, but I do not want to hand-wave a real decode divergence away.

## 12:22 - Reproduction and diagnosis

Replayed the artifact with:

```bash
timeout 180s cargo +nightly fuzz run --fuzz-dir tests/fuzz fuzz_decode \
  tests/fuzz/artifacts/fuzz_decode/crash-cee7d75d4bcc68daf712a9c721dc36c0b1b2ad2f -- -runs=1
```

Confirmed the original panic:

```text
Float SILK/Hybrid decode SNR 35.99 dB < 50 dB (sr=48000, ch=2, pkt_len=128, samples=960)
```

Diagnostic findings:
- Packet parses as Hybrid SWB, code 3 CBR, two frames of 63 bytes each.
- Both Rust and C return `Ok`, both return 960 samples/channel, and final range matches.
- The i16 path has the same 35.99 dB SNR as the float path, so this is not float conversion.
- Mono decode also diverges, so it is not stereo expansion.
- The same packet is bit-exact at 8, 12, and 16 kHz, but diverges at 24 and 48 kHz.
- A SILK-WB reinterpretation of the first frame is bit-exact at 16, 24, and 48 kHz, so the SILK low-band and SILK resampler path are not the trigger.

Conclusion: this is not the earlier zero-length recovery/DTX class. It is arbitrary Hybrid high-band synthesis drift on attacker-controlled input. That is still useful to know, but it is not a good standalone 50 dB SNR oracle because the comparable SILK low-band path is already exact and the drift only appears when CELT high-band arbitrary payload is included.

Emotional temperature: calmer. The important part was not pretending this was the same recovery class. It is a different oracle boundary, and the evidence is specific enough to justify a narrow classifier change rather than random decoder surgery.

## 12:37 - Fix and validation

SILK-only packets and low-rate Hybrid packets can still use the 50 dB SNR oracle; recovery/DTX packets keep their existing sample-count-only classification; roundtrip paths still use the existing well-formed packet gate and are not weakened by this classifier.
