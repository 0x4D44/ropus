### `cargo clippy --workspace --all-targets`

Initial run reported 9 unique lints (some duplicated across the
lib/test/bin facets of the same crate). Each was investigated for
root cause; none indicated missing code, latent defects, or a bad
merge. Inventory and resolution:

| # | Lint | Site | Verdict |
|---|---|---|---|
| 1 | `mixed_attributes_style` | `ropus/src/lib.rs:100` (silk_trace mod) | Style. Doc text was duplicated as both outer `///` on the `pub mod` and inner `//!` inside it. Consolidated to a single inner block. |
| 2 | `useless_comparison` (a.k.a. saturating_sub never < 0) | `ropus/src/silk/decoder.rs:1164` | Dead defensiveness. `nb_subfr` is `usize`, so `saturating_sub(2).max(0)` is `saturating_sub(2)`. The next line already does `nb_subfr - 1` with the same precondition. Replaced with plain `nb_subfr - 2`, which also matches C reference `silk/PLC.c:186` exactly. |
| 3 | `manual_checked_ops` | `ropus-tools-core/src/commands/encode.rs:330` | New lint in Rust 1.95. The existing `if samples_written > 0` guard was correct but stylistically dated. Rewrote as `if let Some(avg_bps) = …checked_div(samples_written)`. |
| 4 | `unnecessary_cast` (i32→i32) | `harness/src/bin_inner/projection_roundtrip.rs:321` | Stale cast — `c_int` is `i32` on Linux. Dropped. |
| 5 | `doc_lazy_continuation` | `harness/src/bin_inner/fuzz_repro_diff.rs:5..=7` | The line `//! + 4 multistream)` started with `+ ` after the comment marker, so the doc parser saw a markdown list item. Reworded `+` to `and`. |
| 6 | `manual_is_multiple_of` ×3 | fuzz_repro_diff.rs:1187, 1251, 1277 | Idiom drift after `is_multiple_of` stabilised in 1.94. Pure rewrite. |
| 7 | `type_complexity` | fuzz_repro_diff.rs:1702 | Function-pointer tuple type was inlined. Added `type BaselineSetter` alias. |
| 8 | `unnecessary_cast` (i32→i32) | fuzz_repro_diff.rs:1905 | `streams: i32` from `new_surround`; `4000 * streams.max(1)` is already i32. Dropped. |
| 9 | `field_reassign_with_default` | fuzz_repro_diff.rs:2204 | The fn mixes scalar assigns and `for` loops over array fields, so a single struct-literal can't replace it cleanly. Used struct-literal init with `..Default::default()` for the scalar prefix; the loops then mutate the result, which doesn't trigger the lint. |

No `#[allow]` suppressions were added. Every lint had a real fix.

The `nb_subfr` finding is the most interesting one: it's a port-quirk
artefact (Rust ported `nb_subfr` as `usize` for indexing safety, but
the C reference uses signed `opus_int`, so the `.max(0)` was a
belt-and-braces guard added during port that became unreachable).
Behaviour is unchanged — the SILK spec guarantees `nb_subfr ∈ {2, 4}`
and the next line already relies on that.
